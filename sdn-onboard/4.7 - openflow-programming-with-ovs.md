## Dàn bài

## 4.7.1 Flow grammar recap và `-O` flag

`ovs-ofctl add-flow <bridge> <flow-spec>` với flow-spec = comma-separated list matches + `actions=` + action list. Selectors áp dụng cho flow itself: `table=N`, `priority=N` (0-65535, default 32768), `cookie=HEX`, `idle_timeout=N`, `hard_timeout=N`, `importance=N`, `send_flow_rem`, `check_overlap`.

Quan trọng: luôn pass `-O OpenFlow13` (hoặc cao hơn) khi dùng multi-table, group, meter, default OVS là OF 1.0 không có các feature này.

## 4.7.2 Match fields: 12-tuple + NXM/OXM extensions

Layer 2: `in_port`, `dl_src`, `dl_dst`, `dl_type`, `dl_vlan`, `dl_vlan_pcp`.

Layer 3 IPv4: `nw_src`, `nw_dst`, `nw_proto`, `nw_tos`, `nw_ttl`.

Layer 4: `tp_src`, `tp_dst`, `tcp_flags`, `icmp_type`, `icmp_code`.

Tunnel metadata: `tun_id`, `tun_src`, `tun_dst`, `tun_flags`.

Conntrack: `ct_state` bitfield (`+trk`, `+new`, `+est`, `+rel`, `+inv`, `+rpl`, `+snat`, `+dnat`), `ct_zone`, `ct_mark`, `ct_label`.

Registers: `reg0..reg15`, `xreg0..xreg7`, `xxreg0..xxreg3`, `metadata`.

Masked matches: CIDR (`nw_src=10.0.0.0/8`) hoặc slash-mask (`dl_src=00:11:22:00:00:00/ff:ff:ff:00:00:00`).

## 4.7.3 Actions: 8 cơ bản + extensions

Output: `output:N`, `NORMAL`, `FLOOD`, `ALL`, `LOCAL`, `IN_PORT`, `CONTROLLER[:max_len]`, `drop`.

Header mod: `mod_vlan_vid:N`, `strip_vlan`, `push_vlan:0x8100`, `pop_vlan`, `mod_dl_src`, `mod_dl_dst`, `mod_nw_src`, `mod_nw_dst`, `dec_ttl`.

General field ops: `set_field:VALUE->FIELD`, `load:VALUE->FIELD`, `move:SRC->DST`.

Pipeline: `resubmit([PORT],[TABLE])`, `goto_table:N`, `group:N`, `meter:N`.

Stateful: `learn(...)`, `ct(commit,zone=N,nat(src=IP),table=N,exec(...))`, `ct_clear`.

Advanced: `conjunction(id,k/n)`, `clone(...)`, `encap(...)`/`decap()`, `check_pkt_larger`, `truncate`.

## 4.7.4 Multi-table pipeline recipe: 3-stage L3 routing (UofSC Lab 6 absorb)

Table 0 classifier. ARP đi `NORMAL`, IP → Table 1:
```
ovs-ofctl add-flow s1 "table=0,arp,action=normal"
ovs-ofctl add-flow s1 "table=0,ip,action=goto_table:1"
```

Table 1 L3. MAC rewrite + TTL dec + goto Table 2:
```
ovs-ofctl add-flow s1 "table=1,ip,nw_dst=192.168.2.0/24,\
 action=mod_dl_src=00:00:00:00:00:01,\
 mod_dl_dst=00:00:00:00:00:05,dec_ttl,goto_table:2"
```

Table 2 L2, output theo destination MAC (learned hoặc static).

## 4.7.5 Conntrack recipe: 5-flow stateful firewall (Compass Ch 9 + UofSC Lab 8)

```
# 1. Send new outbound TCP vào conntrack
ovs-ofctl add-flow br0 "table=0,priority=50,ct_state=-trk,tcp,\
 in_port=veth_l0,action=ct(table=0)"
# 2. New + tracked → commit + forward
ovs-ofctl add-flow br0 "table=0,priority=50,ct_state=+trk+new,tcp,\
 in_port=veth_l0,action=ct(commit),veth_r0"
# 3. Return traffic → ct lookup
ovs-ofctl add-flow br0 "table=0,priority=50,ct_state=-trk,tcp,\
 in_port=veth_r0,action=ct(table=0)"
# 4. Established reply → forward
ovs-ofctl add-flow br0 "table=0,priority=50,ct_state=+trk+est,tcp,\
 in_port=veth_r0,action=veth_l0"
# 5. Established outbound → forward
ovs-ofctl add-flow br0 "table=0,priority=50,ct_state=+trk+est,tcp,\
 in_port=veth_l0,action=veth_r0"
```

Verify: `ovs-appctl dpctl/dump-conntrack`. NAT: `ct(commit,nat(src=192.168.1.1))`. Zone: `ct(zone=N)` isolate overlapping ranges.

## 4.7.6 Groups + meters (OF 1.1+ và 1.3+)

Group: `ovs-ofctl -O OpenFlow13 add-group br0 "group_id=1,type=select,bucket=weight=50,output:1,bucket=weight=50,output:2"`. Types: `all` (multicast), `select` (ECMP), `indirect` (pointer), `fast_failover` (live/dead).

Meter: `ovs-ofctl -O OpenFlow13 add-meter br0 "meter=1,kbps,band=type=drop,rate=1000,burst_size=100"`. Reference: `actions=meter:1,NORMAL`.

## 4.7.7 Flow hygiene: monitor, replace-flows, diff-flows

`ovs-ofctl monitor br0 watch:` stream live flow mod → stdout. Leave chạy trong khi debug controller app.

`ovs-ofctl replace-flows br0 new.flows` atomic swap toàn bộ flow table (khác `add-flows` = append thuần).

`ovs-ofctl diff-flows br0 new.flows` preview thay đổi KHÔNG apply. **Luôn chạy trước `replace-flows` trong production.**

## 4.7.8 Advanced tutorial: 5-table MAC learning (Compass Ch 8)

Canonical exercise xây VLAN-aware MAC learning switch qua 5 stage (admission → VLAN input → MAC+VLAN learn → destination lookup → output). Dùng `learn(...)` action với NXM register cho dynamic flow installation. Sẽ viết chi tiết trong Phase B, đây là exercise pedagogically valuable nhất trong toàn OVS curriculum.
