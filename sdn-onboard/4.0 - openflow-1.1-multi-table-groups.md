## Dàn bài

## 4.0.1 OF 1.1 release 28/02/2011: pre-ONF spec còn shepherd bởi Stanford

Giới thiệu OpenFlow 1.1.0 công bố 28/02/2011, cách ONF thành lập đúng 3 tuần (21/03/2011). Nêu vị trí lịch sử: spec cuối cùng do Stanford Clean Slate Lab shepherd trước khi ownership chuyển sang ONF. Phân tích motivation: single-table của 1.0 gây cross-product explosion (đã bàn tại 3.1.7), 1.1 giải bằng pipeline nhiều bảng nối tiếp. Pointer đến URL archive.org snapshot 03/2011 và current mirror opennetworking.org.

## 4.0.2 Multi-table pipeline semantics

Mô tả flow mới 1.1: packet đi qua Table 0, 1, 2, … theo thứ tự tăng dần; mỗi table match → execute instructions; GOTO_TABLE instruction bắt packet nhảy sang table cao hơn (không bao giờ thấp hơn, chống loop). Nêu khái niệm action set (tích lũy xuyên pipeline, execute một lần ở cuối) vs action list (execute ngay trong table đó). Cross-ref forward đến 4.1.2 khi 1.2 bổ sung OXM để identify field trong action.

## 4.0.3 Instructions vs actions, key distinction từ 1.1

Phân biệt hai khái niệm mới được tách trong 1.1: **instructions** là container-level (WRITE_ACTIONS, APPLY_ACTIONS, WRITE_METADATA, CLEAR_ACTIONS, GOTO_TABLE), quy định điều gì xảy ra với packet sau khi match; **actions** là low-level operation (OUTPUT, SET_VLAN_VID, PUSH_MPLS...) được đóng gói trong instruction. Nêu ordering semantic: instructions thực thi theo thứ tự cố định trong spec (apply before write; goto cuối cùng), còn actions trong cùng instruction thực thi tuần tự theo thứ tự xuất hiện.

## 4.0.4 Group table: ALL, SELECT, INDIRECT, FAST_FAILOVER

Phân loại bốn group type theo spec 1.1 §5.6.1. **ALL** (multicast/flooding, execute all buckets); **SELECT** (load balance across buckets, thuật toán implementation-defined, thường là hash-based); **INDIRECT** (single bucket, dùng để chia sẻ action set giữa nhiều flow); **FAST_FAILOVER** (execute bucket đầu tiên có liveness UP, watch_port hoặc watch_group). Nêu ý nghĩa FAST_FAILOVER: switch tự switch bucket không cần controller can thiệp → reroute sub-10ms khi port down, bypass RTT controller.

## 4.0.5 MPLS support native

Liệt kê actions MPLS mới trong 1.1: PUSH_MPLS, POP_MPLS, SET_MPLS_TTL, DEC_MPLS_TTL, COPY_TTL_IN/OUT. Nêu match fields mới: MPLS label, MPLS traffic class, MPLS BoS (bottom of stack). Phân tích ý nghĩa: 1.1 chính thức mở cửa OpenFlow vào WAN/carrier network, nơi MPLS là de-facto transport (RFC 3031). Forward reference đến 4.2.4 (PBB 802.1ah trong 1.3) và Block XVI SDN WAN.

## 4.0.6 Use case: multi-tenant classifier → tunnel

Mô tả pattern điển hình triển khai multi-table: Table 0 classify theo ingress port + VLAN (tenant identification) → Table 1 set tunnel id (metadata hoặc register) → Table 2 forward based on tunnel id + destination. So sánh với cách implement cùng use case bằng single-table 1.0: phải flatten thành N_tenant × M_dest entries. Đây là minh chứng cụ thể multi-table giảm entry count từ O(N·M) xuống O(N+M).
