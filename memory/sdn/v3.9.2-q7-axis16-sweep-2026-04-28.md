## Phase R2 fix scope (preliminary)

Based on the audit so far, the R2 fix commit will need to:

1. **9.1 axis-16 section #2** (line 801): drop the 4 fabricated function names and correct 1 wrong name. Specifically:
   - Drop `dpif_netlink_flow_put`. The flow-put path is dispatched through the `dpif_class->flow_put` callback in the vtable, not as a top-level `dpif_netlink_*` function.
   - Drop `dpif_netdev_recv`. The userspace netdev datapath uses PMD threads, not a recv function. Reception happens through the class vtable.
