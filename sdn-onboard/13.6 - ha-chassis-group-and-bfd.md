## 13.6.1 Vấn đề: gateway chassis SPOF

Part 13.5 chassisredirect chỉ định một chassis làm active gateway. Nếu chassis chết, north-south traffic down đến khi admin manual failover.

**HA_Chassis_Group** giải quyết: group nhiều chassis với priority, OVN tự động failover.

---

## 13.6.2 HA_Chassis_Group schema

NBDB:

```
HA_Chassis_Group:
  name           — group identifier
  ha_chassis     — set<HA_Chassis UUID>

HA_Chassis:
  chassis_name   — reference Chassis.name trong SB
  priority       — 0-32767, higher = preferred
```

Cấu hình:

```bash
[admin]# ovn-nbctl ha-chassis-group-add hcg-external
[admin]# ovn-nbctl ha-chassis-group-add-chassis hcg-external chassis-1 100
[admin]# ovn-nbctl ha-chassis-group-add-chassis hcg-external chassis-2 50
[admin]# ovn-nbctl ha-chassis-group-add-chassis hcg-external chassis-3 30
[admin]# ovn-nbctl lrp-set-gateway-chassis lrp-external chassis-1 100
```

Priority: chassis-1 (100) = active, chassis-2 (50) = standby, chassis-3 (30) = standby 2.

## 13.6.3 BFD (RFC 5880) detection

BFD packet: small UDP packet (port 3784) exchange giữa chassis. Mỗi bên gửi BFD mỗi X ms. Không nhận N consecutive → declare peer DOWN.

Default OVN BFD:
- **Tx/Rx interval**: 500ms.
- **Detection multiplier**: 3 (peer down sau 1.5s).

Fast-timer: Tx/Rx 100ms, multiplier 3 → detection 300ms. BFD traffic tăng.

BFD session auto-setup giữa HA_Chassis members khi configure.

Monitor:

```bash
[chassis]# ovs-appctl bfd/show
```

## 13.6.4 Failover workflow

chassis-1 (active) crash:

1. chassis-2 stop nhận BFD từ chassis-1.
2. Sau 1.5s, chassis-2 declare DOWN.
3. ovn-controller update `is_active=true` cho chassis-2 trong HA_Chassis_Group.
4. Traffic north-bound mới redirect đến chassis-2.
5. Tenant VM thấy ~1.5-2s outage.

chassis-1 recovery:

1. Re-register với SBDB.
2. BFD khôi phục.
3. Nếu priority cao hơn + preemption=true (default), fail-back.
4. Disable preemption (`ha_chassis_group_options:preemption=false`) tránh flapping.

---

## 13.6.5 Best practices production

1. **3 HA_Chassis members tối thiểu**.
2. **Disable preemption** trong production.
3. **Dedicate gateway chassis** — không mix workload VM (tránh CPU contention).
4. **Monitor BFD state** qua Prometheus exporter.
5. **Quarterly failover test**: kill ovn-controller chassis-1, verify migrate <2s.

### ▶ Capstone Block XIII: Full OVN lab 3-chassis HA

**Outcomes:** Deploy 3-chassis OVN với HA_Chassis_Group, test failover end-to-end.

**Before You Begin:** 3 Ubuntu VM với OVN 22.03.8 (chassis-1/2/3 + central + SB cluster).

**Instructions:**

**1.** HA_Chassis_Group:

```bash
[central]# ovn-nbctl ha-chassis-group-add hcg-gw
[central]# ovn-nbctl ha-chassis-group-add-chassis hcg-gw chassis-1 100
[central]# ovn-nbctl ha-chassis-group-add-chassis hcg-gw chassis-2 50
[central]# ovn-nbctl ha-chassis-group-add-chassis hcg-gw chassis-3 30
```

**2.** Gateway router:

```bash
[central]# ovn-nbctl lr-add lr-tenant
[central]# ovn-nbctl lrp-add lr-tenant lrp-external 40:44:00:00:ff:01 192.0.2.1/24
[central]# ovn-nbctl lrp-set-ha-chassis-group lrp-external hcg-gw
```

**3.** Verify active:

```bash
[central]# ovn-sbctl list Port_Binding cr-lrp-external
# chassis: <chassis-1 UUID>
```

**4.** Generate traffic:

```bash
[vm-in-tenant]# ping -i 0.1 8.8.8.8 &
```

**5.** Failover — kill ovn-controller chassis-1:

```bash
[chassis-1]# systemctl stop ovn-controller
```

Monitor:
- ~1-2s packet loss.
- `ovn-sbctl list Port_Binding cr-lrp-external` → chassis-2.

**6.** Recovery: `systemctl start ovn-controller` trên chassis-1.

**7.** Cleanup.

## Điểm cốt lõi cần nhớ

Thứ nhất, HA_Chassis_Group là OVN abstraction cho gateway failover. Apply trên LRP distributed gateway.

Thứ hai, BFD (RFC 5880) detect sub-second với 500ms interval + multiplier 3. Fast-timer cho critical deployment.

Thứ ba, preemption có thể flapping. Production disable.

Thứ tư, dedicate gateway chassis + monitor BFD + quarterly failover test — 5-nine availability best practice.
