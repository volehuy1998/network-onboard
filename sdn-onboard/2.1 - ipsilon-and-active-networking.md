## Dàn bài

## 2.1.1 Ipsilon IP Switching: GSMP biến thể + flow classifier

Giới thiệu Ipsilon Networks (Peter Newman et al., 1996) và bài toán tận dụng ATM hardware fast-switching cho IP traffic. Mô tả mô hình hai pha: pha đầu router xử lý packet bằng phần mềm, pha hai (sau khi nhận diện flow dài) cài cell-relay path qua ATM switch và bypass IP layer. Pointer đến RFC 1953 (IFMP) và RFC 1987 (GSMP v1.1) là hai protocol Ipsilon publish năm 1996. Phần này KHÔNG lặp lại GSMP message detail từ Part 2.0 §2.0.3, chỉ phân tích vai trò Ipsilon là người dùng đầu tiên biến GSMP thành commercial product.

## 2.1.2 Ipsilon → MPLS lineage

Phân tích con đường ý tưởng "label-switching" lan từ Ipsilon (1996) qua Cisco Tag Switching (1996-1997) đến IETF MPLS Working Group (1997, RFC 3031 ban hành 2001). Nêu acquisition Ipsilon bởi Nokia năm 1997 và lý do MPLS thay thế hoàn toàn IP Switching: MPLS không cần ATM, hoạt động trên Ethernet/PPP/Frame Relay, và label-stacking cho phép VPN/TE. Mục đích historical: cho học viên thấy ý tưởng tốt + timing không phù hợp dẫn đến market failure dù technology đúng hướng.

## 2.1.3 Active Networking: DARPA funding 1996-2001

Đặt bối cảnh DARPA Active Networks Program (PI: David Tennenhouse, MIT) như nỗ lực tách network forwarding khỏi giao thức cố định. Liệt kê 5 nhóm tham gia chính: MIT (ANTS), Univ. of Pennsylvania (SwitchWare/PLAN), Columbia (NetScript), Princeton (RCANE), BBN (Smart Packets). Pointer đến hai bài seminal: Tennenhouse & Wetherall "Towards an Active Network Architecture" (CCR 1996) và Calvert et al. "Architectural Framework for Active Networks" (CCR 1999).

## 2.1.4 Capsules vs programmable switches

So sánh hai trường phái Active Networking: (a) capsule approach, code đính kèm packet, switch execute on-the-fly (đại diện: ANTS); (b) programmable switch approach, code install trước qua management channel, packet chỉ trigger code (đại diện: SwitchWare). Phân tích trade-off: capsule linh hoạt hơn nhưng overhead per-packet cao và security khó kiểm soát; programmable switch giống mô hình SDN sau này hơn (control out-of-band, data-plane fast-path).

## 2.1.5 Tại sao Active Networking không thương mại hóa được

Phân tích ba nguyên nhân theo Wetherall retrospective (HotNets 2007): (a) performance overhead khi execute Java/PLAN code per packet không scale với line-rate Gigabit, (b) security model chưa giải quyết được sandboxing đủ mạnh để vận hành liên-AS, (c) thiếu killer application, operator không thấy use case kinh doanh đủ thuyết phục để nâng cấp infrastructure. Liên hệ với bài học SDN sau này: cần chia rõ control vs data plane, không trộn execution logic vào fast path.

## 2.1.6 Di sản đến P4 (Block VI)

Tổng kết tư tưởng programmable data plane từ Active Networking đến P4 (Bosshart et al., CCR 2014). Nêu khác biệt cốt lõi: P4 là Domain-Specific Language compile-time (không inline execution), runtime chỉ load match-action table, giải được cả ba vấn đề performance/security/business case của AN. Forward reference đến Part 6.x P4 + PISA architecture và Part 6.x Tofino/eBPF/XDP.
