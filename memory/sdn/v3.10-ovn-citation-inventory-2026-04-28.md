## 1. Methodology

This inventory is the canonical superset. Each row is a citation-candidate that R1 audits per §3.3 verification primitives. Some rows are noise (e.g., a `Table 18` mention that is part of a verbatim quote, not a free-standing claim). R1 disambiguates per-row.

**Citation kinds detected (10 categories):**

| Kind | Pattern | R1 verification primitive |
|---|---|---|
| `URL_OVN` | `https://github.com/ovn-org/ovn/...` | Primitive B (URL ref segment check) |
| `URL_OVS` | `https://github.com/openvswitch/ovs/...` | Primitive B against OVS repo |
| `BRANCH_TAG` | `branch-22.03`, `v22.03.8`, etc. | Primitive B context check |
| `INLINE_FILE` | `` `controller/binding.c` `` and similar | Primitive A path-existence check |
| `INLINE_FUNC` | `` `function_name()` `` (inline backticks) | Primitive A function-name check |
| `COMMIT_SHA` | 8-40 char hex token | Primitive E commit verification |
| `SCHEMA_FILE` | `ovn-sb.ovsschema`, `ovn-nb.ovsschema` | Primitive D schema parser |
| `SCHEMA_TABLE` | backticked table name from the 31+27 known list | Primitive D table existence |
| `PIPELINE_STAGE` | `ls_in_*`, `ls_out_*`, `lr_in_*`, `lr_out_*` | Primitive C against `northd/northd.c` PIPELINE_STAGE block |
| `TABLE_ID` | `Table 18`, `table 26`, etc. | Primitive C cross-check with PIPELINE_STAGE for the relevant pipeline |

Some rows fall in multiple kinds (e.g., a URL with a branch tag in its path). The script emits one row per (kind, target) pair, so a single curriculum line may produce multiple rows. R1 deduplicates to unique-position tuples.

**Caveats.**

- The script does not parse fenced code blocks separately; some rows inside ``` blocks are CLI examples, not curriculum claims. R1 per-row review filters.
- The `COMMIT_SHA` pattern matches any 8-40 char hex string with both letters and digits; some rows are MAC addresses or hash tokens unrelated to git commits. R1 filters.
- The `INLINE_FUNC` pattern matches any backticked `name()`; some rows are CLI commands (`ovn-nbctl()`) or shell snippets. R1 filters.
- The `TABLE_ID` pattern is an over-collector for §0.4 v3.10 sample finding type WRONG_FACT. R1 confirms whether each match is a free-standing claim or a verbatim quote of source.
