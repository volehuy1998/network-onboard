## Dàn bài

## 4.1.1 OF 1.2 release 05/12/2011, spec đầu tiên do ONF công bố

Giới thiệu OpenFlow 1.2.0 (05/12/2011), spec đầu tiên ONF phát hành sau khi nhận ownership từ Stanford tháng 03/2011 (đã bàn tại 3.2.4). Nêu ý nghĩa tổ chức: đây là proof-of-concept rằng ONF vận hành được spec process (WG → TSC → member vote → publish) chứ không chỉ là rebrand của Stanford. Pointer đến current URL opennetworking.org/sdn-standards/openflow/.

## 4.1.2 OXM TLV format

Mô tả OXM header 32-bit theo spec 1.2 §7.2.3: **class** 16-bit (OFPXMC_OPENFLOW_BASIC=0x8000, vendor space > 0xFFFF), **field** 7-bit, **HM** (has-mask) 1-bit, **length** 8-bit (length of payload). TLV payload theo sau chứa value (và mask nếu HM=1). Nêu origin: OXM format kế thừa gần nguyên vẹn từ NXM (Nicira Extensible Match) đã có trong Open vSwitch từ 2009, backward compat intentional để vendor đã implement OVS không phải rewrite matching engine.

## 4.1.3 Match field extensions: ARP, ICMP, TCP flags, metadata

Liệt kê các match field extensions chính trong 1.2 §A.2.3 beyond 12-tuple của 1.0: **ARP detail** (ARP_OP opcode request/reply, ARP_SPA sender protocol address, ARP_TPA target protocol address, ARP_SHA sender hardware address, ARP_THA target hardware address) cho phép firewall SDN lọc ARP storm; **ICMP detail** (ICMP_TYPE, ICMP_CODE) cho rate limiting ICMP; **TCP flags** (URG/ACK/PSH/RST/SYN/FIN bits riêng rẽ) cho SYN flood detection; **Metadata** (64-bit register cross-table, bổ sung từ 1.1). Phân tích ý nghĩa: 1.0 dùng 12-tuple cứng chỉ cover generic packet filtering; 1.2 mở rộng để handle protocol-specific patterns như ARP flood protection, ICMP rate limiting, stateful TCP flag-based policy. Forward reference đến Part 4.7 (OF programming) với recipe concrete dùng TCP flags cho firewall.

## 4.1.4 Controller roles: EQUAL, MASTER, SLAVE

Mô tả RBAC model cho multi-controller setup (§6.3.5): **EQUAL** (default, all controllers read/write ngang hàng, mỗi nhận async event), **MASTER** (exactly one per switch, full read/write, nhận tất cả async event), **SLAVE** (read-only + configuration monitoring, không nhận PACKET_IN/FLOW_REMOVED). Nêu protocol: OFPT_ROLE_REQUEST/REPLY với generation_id để tránh race condition khi 2 controller cùng claim MASTER. Forward reference đến Block VII (controllers) khi bàn distributed controller architecture (ODL/ONOS).

## 4.1.5 Migration OF 1.0 → 1.2: không backward compat wire

Phân tích break change: 1.2 wire format KHÔNG backward compat với 1.0 (header version khác: 0x01 vs 0x03; ofp_match struct hoàn toàn khác, fixed 12-tuple vs variable OXM TLV). Nêu cơ chế negotiate: OFPT_HELLO exchange ở đầu connection, switch và controller thoả thuận highest common version. Thực tế deployment: OVS giai đoạn 2012-2014 hỗ trợ đồng thời 1.0 và 1.3, bỏ qua 1.1/1.2 vì vendor adoption thấp (cross-ref 4.2.6).
