---

## Dàn bài

## 14.1.1 Integration bridge br-int role

*Sẽ phát triển: br-int là anchor của OVN logical pipeline; tất cả logical flow deployed ở đây; VM tap port attach; managed hoàn toàn bởi ovn-controller.*

## 14.1.2 External bridge br-ex — NAT/floating-IP exit

*Sẽ phát triển: br-ex cho north-south traffic; floating IP exit; kết nối physical uplink NIC; mapping qua ovn-bridge-mappings.*

## 14.1.3 Provider bridges br-provider0, br-provider1 — physnet

*Sẽ phát triển: provider network physnet mapping; multiple physical networks; VLAN trunk configuration; `ovn-bridge-mappings` key in Open_vSwitch external_ids.*

## 14.1.4 Patch ports giữa bridges

*Sẽ phát triển: OVS patch port mechanism; giả kết nối không qua kernel hoặc driver; installed automatically by ovn-controller based on bridge mappings.*

## 14.1.5 Topology map qua ovs-vsctl show

*Sẽ phát triển: walk-through output của `ovs-vsctl show`; identify br-int, br-ex, br-provider-*; trace patch ports; port types (internal, patch, tap, dpdk).*
