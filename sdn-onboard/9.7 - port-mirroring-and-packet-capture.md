## Dàn bài

## 9.7.1 Mirror table trong OVSDB: reference từ Bridge.mirrors

Mirror là first-class row trong OVSDB table `Mirror`, được reference bởi `Bridge.mirrors` column (kiểu set). Columns chính: `name` (identifier), `select_src_port` / `select_dst_port` / `select_vlan` (source filters, có thể kết hợp), `select_all` (wildcard match), `output_port` (local SPAN destination), `output_vlan` (RSPAN mirror tag).

## 9.7.2 Canonical atomic idiom với --id=@ reference

Tạo mirror yêu cầu reference nhiều Port UUID, lý tưởng để làm một OVSDB transaction atomic. Idiom chuẩn:

```
ovs-vsctl -- set Bridge br0 mirrors=@m \
 -- --id=@eth0 get Port eth0 \
 -- --id=@eth1 get Port eth1 \
 -- --id=@mtap get Port mirror-tap \
 -- --id=@m create Mirror name=mymirror \
 select-dst-port=@eth0,@eth1 \
 select-src-port=@eth0,@eth1 \
 output-port=@mtap
```

Nếu bất kỳ bước nào fail, cả transaction rollback, tránh half-configured mirror với orphan Mirror row.

## 9.7.3 Local SPAN (output_port): mirror ra internal port

Output vào một port thuộc cùng bridge (thường là `type=internal` để host kernel thấy được). Dùng `tcpdump -i <tap>` hoặc feed Wireshark/Suricata từ interface đó. Cần `ip link set <tap> promisc on` nếu muốn bắt mọi frame.

## 9.7.4 Remote SPAN (output_vlan): mirror tag + ship qua trunk

Thay vì output port, mirror wrap frame với outer VLAN tag rồi output ra trunk port. Nhận bên đầu xa qua `ovs-vsctl add-port br0 rspan-in tag=<mirror-vlan>`. Overhead: VLAN tag 4 byte + có thể drop do MTU mismatch nếu inner frame gần 1500.

## 9.7.5 Cleanup: garbage-collection vs explicit destroy

`ovs-vsctl clear Bridge br0 mirrors`: remove reference từ Bridge.mirrors. Mirror row KHÔNG còn referenced → OVSDB auto-garbage-collect. Khác với `QoS` + `Queue` (Part 9.9) là không auto-GC, cần `destroy` thủ công.
