## 7.5.1 Kiến trúc event system của Ryu

Ryu xây dựng trên mô hình event-driven: mọi OpenFlow message nhận từ switch đều được ánh xạ sang một Python event object, rồi dispatch đến tất cả handler đã đăng ký cho event type đó. Cơ chế đăng ký handler dùng decorator `@set_ev_cls`:

```python
from ryu.base import app_manager
from ryu.controller import ofp_event
from ryu.controller.handler import CONFIG_DISPATCHER, MAIN_DISPATCHER
from ryu.controller.handler import set_ev_cls
from ryu.ofproto import ofproto_v1_3

class MySwitch(app_manager.RyuApp):
    OFP_VERSIONS = [ofproto_v1_3.OFP_VERSION]

    @set_ev_cls(ofp_event.EventOFPSwitchFeatures, CONFIG_DISPATCHER)
    def switch_features_handler(self, ev):
        # Gọi khi switch kết nối lần đầu (handshake xong)
        datapath = ev.msg.datapath
        self.install_table_miss_flow(datapath)

    @set_ev_cls(ofp_event.EventOFPPacketIn, MAIN_DISPATCHER)
    def packet_in_handler(self, ev):
        # Gọi cho mỗi packet_in message từ switch
        msg = ev.msg
        datapath = msg.datapath
        in_port = msg.match['in_port']
        # ... xử lý packet ...
```

Tham số thứ hai của `@set_ev_cls` là dispatcher state — Ryu chỉ deliver event đến handler khi switch đang ở đúng trạng thái:

| Dispatcher | Ý nghĩa |
|------------|---------|
| `CONFIG_DISPATCHER` | Ngay sau khi switch kết nối, trước khi sẵn sàng forward |
| `MAIN_DISPATCHER` | Switch đang hoạt động bình thường — hầu hết event dùng state này |
| `DEAD_DISPATCHER` | Switch đã disconnect |

> **Chủ đề trọng tâm:** `EventDispatcher` của Ryu là single-threaded — handler phải hoàn thành nhanh và không blocking. Nếu handler gọi `time.sleep()` hoặc blocking I/O (database query, DNS lookup), toàn bộ controller bị chặn trong thời gian đó. Khi cần blocking operation, dùng `hub.spawn()` của Ryu để tạo greenlet riêng. Đây là nguyên nhân phổ biến nhất khiến Ryu controller đột ngột trở nên chậm dưới tải cao trong benchmark.

## 7.5.2 OFPFlowMod: cài đặt và xóa flow entry

`OFPFlowMod` là message OpenFlow dùng để thêm, sửa, hoặc xóa flow entry trong flow table của switch — là message quan trọng nhất mà controller gửi đến switch.

### 7.5.2.1 Cài đặt flow entry (OFPFC_ADD)

```python
def add_flow(self, datapath, priority, match, actions,
             idle_timeout=0, hard_timeout=0, cookie=0):
    ofproto = datapath.ofproto
    parser = datapath.ofproto_parser

    # Instruction: áp dụng actions ngay lập tức
    inst = [parser.OFPInstructionActions(
        ofproto.OFPIT_APPLY_ACTIONS, actions
    )]

    mod = parser.OFPFlowMod(
        datapath=datapath,
        cookie=cookie,
        priority=priority,
        idle_timeout=idle_timeout,   # 0 = không expire theo inactivity
        hard_timeout=hard_timeout,   # 0 = không expire theo thời gian
        match=match,
        instructions=inst
    )
    datapath.send_msg(mod)
```

Ví dụ cài đặt flow forward gói tin ra port 2 khi đến từ port 1 với MAC cụ thể:

```python
parser = datapath.ofproto_parser

match = parser.OFPMatch(
    in_port=1,
    eth_dst='fa:16:3e:11:22:33'
)
actions = [parser.OFPActionOutput(2)]
self.add_flow(datapath, priority=10, match=match, actions=actions,
              idle_timeout=30)   # Tự xóa sau 30 giây không có traffic
```

### 7.5.2.2 Xóa flow entry (OFPFC_DELETE)

```python
def delete_flow(self, datapath, match):
    ofproto = datapath.ofproto
    parser = datapath.ofproto_parser

    mod = parser.OFPFlowMod(
        datapath=datapath,
        command=ofproto.OFPFC_DELETE,
        out_port=ofproto.OFPP_ANY,    # Khớp bất kỳ output port
        out_group=ofproto.OFPG_ANY,
        match=match
    )
    datapath.send_msg(mod)
```

### 7.5.2.3 Table-miss flow entry

Table-miss là flow entry priority 0 với empty match (khớp mọi gói tin). Đây là entry bắt buộc phải cài đặt ngay khi switch kết nối — nếu không, gói tin không khớp flow nào bị switch drop silently thay vì gửi `packet_in` lên controller:

```python
def install_table_miss_flow(self, datapath):
    ofproto = datapath.ofproto
    parser = datapath.ofproto_parser

    match = parser.OFPMatch()   # Match rỗng = khớp mọi packet
    actions = [parser.OFPActionOutput(
        ofproto.OFPP_CONTROLLER,
        ofproto.OFPCML_NO_BUFFER   # Gửi toàn bộ packet lên controller
    )]
    self.add_flow(datapath, priority=0, match=match, actions=actions)
```

> **Hiểu sai:** "`OFPFC_DELETE` xóa toàn bộ flow table khi empty match." Thực tế: với `OFPFC_DELETE`, empty match trong Ryu không tương đương "khớp tất cả" vì `out_port=OFPP_ANY` và `out_group=OFPG_ANY` phải được set rõ ràng mới xóa không phân biệt output. Nếu thiếu hai field này, switch có thể từ chối delete hoặc chỉ xóa entry có output port khớp chính xác.

## 7.5.3 REST API pattern trong Ryu

Ryu tích hợp sẵn WSGI server dựa trên `eventlet`. Developer có thể expose HTTP endpoint để điều khiển controller từ bên ngoài — ví dụ CI/CD pipeline thêm flow rule khi deploy service mới, hoặc network management UI query flow table.

Pattern chuẩn dùng `WSGIApplication` và `ControllerBase`:

```python
from ryu.app.wsgi import ControllerBase, WSGIApplication, route
from ryu.base import app_manager
from webob import Response
import json

class FlowManager(app_manager.RyuApp):
    _CONTEXTS = {'wsgi': WSGIApplication}

    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        wsgi = kwargs['wsgi']
        wsgi.register(FlowRestController,
                      {'flow_manager_app': self})
        self.datapaths = {}   # dp_id → datapath object

    @set_ev_cls(ofp_event.EventOFPStateChange,
                [MAIN_DISPATCHER, DEAD_DISPATCHER])
    def state_change_handler(self, ev):
        datapath = ev.datapath
        if ev.state == MAIN_DISPATCHER:
            self.datapaths[datapath.id] = datapath
        elif ev.state == DEAD_DISPATCHER:
            self.datapaths.pop(datapath.id, None)


class FlowRestController(ControllerBase):
    def __init__(self, req, link, data, **config):
        super().__init__(req, link, data, **config)
        self.app = data['flow_manager_app']

    @route('flow', '/v1/switches', methods=['GET'])
    def list_switches(self, req, **kwargs):
        switches = list(self.app.datapaths.keys())
        body = json.dumps({'switches': [hex(s) for s in switches]})
        return Response(content_type='application/json', body=body)
```

Khởi động Ryu với REST endpoint:

```bash
[controller]$ ryu-manager flow_manager.py --wsapi-port 8080
[controller]$ curl http://localhost:8080/v1/switches
{"switches": ["0x1"]}
```

## 7.5.4 Thu thập traffic statistics

Ryu có thể request statistics từ switch định kỳ và xử lý reply để xây dựng visibility layer. Đây là cách Ryu được dùng trong nghiên cứu traffic engineering: poll flow stats mỗi 10 giây, phát hiện elephant flow (flow chiếm >80% bandwidth), kích hoạt rerouting bằng cách cài flow entry priority cao hơn.

**Port statistics** (tx/rx bytes, gói tin, drops):

```python
def request_port_stats(self, datapath):
    ofproto = datapath.ofproto
    parser = datapath.ofproto_parser
    req = parser.OFPPortStatsRequest(datapath, 0, ofproto.OFPP_ANY)
    datapath.send_msg(req)

@set_ev_cls(ofp_event.EventOFPPortStatsReply, MAIN_DISPATCHER)
def port_stats_reply_handler(self, ev):
    for stat in ev.msg.body:
        self.logger.info(
            'Port %d: rx_pkts=%d tx_pkts=%d rx_bytes=%d tx_bytes=%d '
            'rx_drops=%d tx_drops=%d',
            stat.port_no,
            stat.rx_packets, stat.tx_packets,
            stat.rx_bytes, stat.tx_bytes,
            stat.rx_dropped, stat.tx_dropped
        )
```

**Flow statistics** (gói tin count, byte count, duration per entry):

```python
def request_flow_stats(self, datapath):
    parser = datapath.ofproto_parser
    req = parser.OFPFlowStatsRequest(datapath)
    datapath.send_msg(req)

@set_ev_cls(ofp_event.EventOFPFlowStatsReply, MAIN_DISPATCHER)
def flow_stats_reply_handler(self, ev):
    for stat in ev.msg.body:
        self.logger.info(
            'Table %d priority %d: packets=%d bytes=%d duration=%ds',
            stat.table_id, stat.priority,
            stat.packet_count, stat.byte_count,
            stat.duration_sec
        )
```

- Expose REST GET `/stats/packet_in` trả về counter dạng JSON

**Before You Begin:**
Ryu cài đặt (`pip install ryu`). OVS đang chạy. Mininet tùy chọn cho topology test.

**Instructions:**

**1.** Tạo file `mac_learning_rest.py`:

```python
from ryu.base import app_manager
from ryu.controller import ofp_event
from ryu.controller.handler import CONFIG_DISPATCHER, MAIN_DISPATCHER
from ryu.controller.handler import set_ev_cls
from ryu.ofproto import ofproto_v1_3
from ryu.lib.packet import packet, ethernet
from ryu.app.wsgi import ControllerBase, WSGIApplication, route
from webob import Response
import json

class MacLearningRest(app_manager.RyuApp):
    OFP_VERSIONS = [ofproto_v1_3.OFP_VERSION]
    _CONTEXTS = {'wsgi': WSGIApplication}

    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        wsgi = kwargs['wsgi']
        wsgi.register(StatsController, {'app': self})
        self.mac_to_port = {}
        self.packet_in_count = 0

    @set_ev_cls(ofp_event.EventOFPSwitchFeatures, CONFIG_DISPATCHER)
    def features_handler(self, ev):
        dp = ev.msg.datapath
        parser = dp.ofproto_parser
        ofproto = dp.ofproto
        match = parser.OFPMatch()
        actions = [parser.OFPActionOutput(
            ofproto.OFPP_CONTROLLER, ofproto.OFPCML_NO_BUFFER)]
        inst = [parser.OFPInstructionActions(
            ofproto.OFPIT_APPLY_ACTIONS, actions)]
        dp.send_msg(parser.OFPFlowMod(
            datapath=dp, priority=0, match=match, instructions=inst))

    @set_ev_cls(ofp_event.EventOFPPacketIn, MAIN_DISPATCHER)
    def packet_in_handler(self, ev):
        self.packet_in_count += 1
        msg = ev.msg
        dp = msg.datapath
        ofproto = dp.ofproto
        parser = dp.ofproto_parser
        in_port = msg.match['in_port']

        pkt = packet.Packet(msg.data)
        eth = pkt.get_protocol(ethernet.ethernet)
        if eth is None:
            return

        dst = eth.dst
        src = eth.src
        dpid = dp.id
        self.mac_to_port.setdefault(dpid, {})
        self.mac_to_port[dpid][src] = in_port

        out_port = self.mac_to_port[dpid].get(dst, ofproto.OFPP_FLOOD)
        actions = [parser.OFPActionOutput(out_port)]

        if out_port != ofproto.OFPP_FLOOD:
            match = parser.OFPMatch(in_port=in_port, eth_dst=dst)
            inst = [parser.OFPInstructionActions(
                ofproto.OFPIT_APPLY_ACTIONS, actions)]
            dp.send_msg(parser.OFPFlowMod(
                datapath=dp, priority=1, idle_timeout=30,
                match=match, instructions=inst))

        data = msg.data if msg.buffer_id == ofproto.OFP_NO_BUFFER else None
        dp.send_msg(parser.OFPPacketOut(
            datapath=dp, buffer_id=msg.buffer_id,
            in_port=in_port, actions=actions, data=data))


class StatsController(ControllerBase):
    def __init__(self, req, link, data, **config):
        super().__init__(req, link, data, **config)
        self.app = data['app']

    @route('stats', '/stats/packet_in', methods=['GET'])
    def get_count(self, req, **kwargs):
        body = json.dumps({'packet_in_count': self.app.packet_in_count})
        return Response(content_type='application/json', body=body)

    @route('stats', '/stats/mac_table', methods=['GET'])
    def get_mac_table(self, req, **kwargs):
        table = {hex(k): v for k, v in self.app.mac_to_port.items()}
        return Response(content_type='application/json',
                        body=json.dumps(table))
```

## Điểm cốt lõi cần nhớ

Thứ nhất, event system single-threaded của Ryu là lưỡi dao hai mặt: đơn giản hóa lập trình bằng cách loại bỏ concurrent access vào shared state, nhưng yêu cầu mọi handler phải không blocking. Bất kỳ blocking I/O nào trong handler đều chặn toàn bộ controller.

Thứ hai, `OFPFlowMod` với `idle_timeout` là cơ chế garbage collection tự nhiên cho flow table — entry học MAC nên có `idle_timeout=30` để tự động xóa khi host không còn active, tránh chuyển tiếp table đầy với entry lỗi thời. Không đặt timeout là lỗi phổ biến khi đưa lab code vào production.

Thứ ba, REST API pattern trong Ryu tách rời mặt phẳng điều khiển (HTTP endpoint) khỏi mặt phẳng dữ liệu (OpenFlow handler). Pattern này quan trọng cho nghiên cứu: external script có thể thêm/xóa flow rule qua REST mà không cần sửa Ryu code, cho phép tự động hóa và khả năng tái tạo trong thí nghiệm.
