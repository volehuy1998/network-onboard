## 2. OF version constants

### 2.1. OVS v2.17.9 OF version representation

OVS does **not** define a single `OFP_VERSION` macro. Instead, `include/openflow/openflow-common.h` defines an enum of per-version wire-protocol numbers:

```
$ cd /c/Users/voleh/Documents/ovs && grep -n "OFP1[0-5]_VERSION" include/openflow/openflow-common.h
77:    OFP10_VERSION = 0x01,
78:    OFP11_VERSION = 0x02,
79:    OFP12_VERSION = 0x03,
80:    OFP13_VERSION = 0x04,
81:    OFP14_VERSION = 0x05,
82:    OFP15_VERSION = 0x06
```

OVS supports OF 1.0 through OF 1.5. OF 1.6 wire value (0x07) is not present in this enum at v2.17.9.

`OFP_TCP_PORT` is **not defined** anywhere in OVS `include/openflow/` at v2.17.9 (`grep -rn "OFP_TCP_PORT" include/ lib/` returns empty). The IANA-assigned port 6653 is referenced in OVS documentation and code comments rather than as a header macro.

### 2.2. OpenFlow spec1.6_rc1 OF version representation

The offline OpenFlow spec repo defines both macros at `include/openflow/openflow.h`:

```
$ cd /c/Users/voleh/Documents/openflow && grep -n "#define OFP_VERSION\|#define OFP_TCP_PORT" include/openflow/openflow.h
70:#define OFP_VERSION   0x07
76:#define OFP_TCP_PORT  6653
```

`OFP_VERSION = 0x07` corresponds to OF 1.6 (the `_rc1` draft). `OFP_TCP_PORT = 6653` is the IANA-assigned canonical TCP port for the OpenFlow control channel.

### 2.3. Cross-baseline difference

The OVS implementation supports up to OF 1.5 (`OFP15_VERSION = 0x06`). The OpenFlow spec repo at `spec1.6_rc1` declares OF 1.6 (`OFP_VERSION = 0x07`). Curriculum citations to "OF 1.6" or "1.6-rc1 TTP" are verified against the spec repo; citations to OF 1.0 through 1.5 are verified against OVS.

```
$ grep -nE "#define OFPP_LOCAL|#define OFPP_NORMAL|#define OFPP_CONTROLLER|#define OFPP_TABLE" include/openflow/openflow-1.0.h
40:#define OFPP_TABLE      OFP_PORT_C(0xfff9) /* Perform actions in flow table. */
41:#define OFPP_NORMAL     OFP_PORT_C(0xfffa) /* Process with normal L2/L3. */
45:#define OFPP_CONTROLLER OFP_PORT_C(0xfffd) /* Send to controller. */

PASS. Confirmed against curriculum claim at `4.8 - openflow-match-field-catalog.md` lines 96-98 (OFPP_LOCAL = 0xfffe, OFPP_CONTROLLER = 0xfffd, OFPP_NORMAL = 0xfffa).

### 4.2. OFPACT_OUTPUT enum at OVS v2.17.9

```
$ grep -nE "OFPACT_OUTPUT\b" include/openvswitch/ofp-actions.h | head -5
295:/* OFPACT_OUTPUT.

PASS. The OFPACT_ enum lives in `include/openvswitch/ofp-actions.h` (OVS-internal action representation), distinct from the wire-protocol OFPAT_* enum in the per-version `openflow-1.X.h` headers. This distinction is critical for Rule 14 audit decisions.
