## Dàn bài

## 9.11.1 ovs-appctl architecture: Unix socket RPC cho mọi daemon

`ovs-appctl` speak JSON-RPC qua Unix domain socket tại `/var/run/openvswitch/<daemon>.ctl`. Mỗi daemon (ovs-vswitchd, ovsdb-server, ovn-controller, ovn-northd, ovn-sbdb-monitor) register handlers riêng.

Flag `-t <daemon-name>` chọn target daemon: `ovs-appctl -t ovsdb-server version`. Default target = ovs-vswitchd.

## 9.11.2 vlog: logging control runtime (không cần restart)

`vlog/list`: list tất cả modules + destination + level.
`vlog/set <module>:<destination>:<level>`: set level runtime.
- module: `dpif`, `vconn`, `dpdk`, `upcall`, `revalidator`, hoặc `ANY`.
- destination: `console`, `syslog`, `file`.
- level: `emer`, `err`, `warn`, `info`, `dbg`.

Ví dụ turn-up dpif to debug file level:
```
ovs-appctl vlog/set dpif:file:dbg
```

`vlog/reopen`: rotate log file sau khi logrotate rename.

## 9.11.3 coverage/show: per-counter telemetry

`coverage/show` dump mọi counter named trong binary — mỗi allocation, netlink round-trip, upcall, flow install, revalidator pass đều được count. Output format:
```
Last hour: 120 / Last min: 2 / rate: 0.033/sec
```

Điểm vào (entry point) khi debug unexpected behavior: so sánh coverage snapshot trước vs sau event, tìm counter tăng bất thường.

## 9.11.4 memory/show: memory breakdown

`memory/show` split RSS theo component: `ofproto`, `ofproto_dpif`, `packets`, `handlers`, `revalidators`, `dbmon`. Debug memory leak = monitor `ofproto` size theo thời gian.

## 9.11.5 ofproto/trace: packet simulation

Command most-important cho OpenFlow debugging:
```
ovs-appctl ofproto/trace br0 "in_port=1,dl_src=aa:bb:cc:dd:ee:01,dl_dst=ff:ff:ff:ff:ff:ff"
```

Output walk từng table trong pipeline, show matched flow rules + actions executed, kết thúc với `Datapath actions: <actions>` line. Thêm flag `-generate` để thực sự execute side-effect (learn action install flow vào table).

## 9.11.6 Datapath introspection: dpctl, dpif, upcall

`dpctl/show` + `dpctl/dump-flows -m`: dump megaflow cache kernel datapath.

`dpif/show` + `dpif/show-dp-features`: datapath feature flags.

`upcall/show` + `upcall/set-flow-limit N`: upcall pressure + tune limit (default 200000, raise if flow eviction quá nhanh).

## 9.11.7 DPDK PMD telemetry

`dpif-netdev/pmd-stats-show`: idle/polling time per PMD thread.
`dpif-netdev/pmd-rxq-show`: rx queue distribution.
`dpif-netdev/pmd-rxq-rebalance`: trigger load rebalance.
`dpif-netdev/pmd-perf-show` (khi `other_config:pmd-perf-metrics=true`): detailed perf histograms.

## 9.11.8 Tunnel neighbor tables

`tnl/neigh/show`: IPv6 ND + IPv4 ARP resolution cache cho tunnel underlay.
`tnl/arp/show`, `tnl/ports/show`: complement views.
