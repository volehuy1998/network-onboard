## 12.1.1 Stack overview

A modern multi-tenant DC fabric:

```
Layer 4: Application (tenant workload)
Layer 3: Overlay VXLAN VNI per tenant
Layer 2: BGP EVPN control plane
Layer 1: Underlay BGP unnumbered or OSPF
Layer 0: Physical leaf-spine Clos
```

Each layer replaces one responsibility from the 3-tier legacy:
- Clos replaces aggregation and core.
- VXLAN replaces VLAN (it scales to 16.7 million).
- EVPN replaces flood-and-learn.
- Anycast gateway replaces single-router-as-gateway.

---

## 12.1.2 Traffic flow

**Tenant VM-to-VM in the same subnet (L2):**

1. VM-A (on leaf 1) sends a frame to VM-B (on leaf 3) in the same VLAN 10.
2. Leaf 1 looks up the MAC. The entry was pre-populated through EVPN Type 2 from Leaf 3.
3. Leaf 1 encapsulates VXLAN: outer src = Leaf 1, outer dst = Leaf 3, VNI = the VLAN 10 mapping.
4. The packet traverses the underlay through ECMP to the spine and on to Leaf 3.
5. Leaf 3 decapsulates VXLAN and forwards out the port to VM-B.

**Tenant VM-to-VM across subnets (L3):**
- The packet goes to the gateway IP (anycast on every leaf).
- Leaf 1 routes L3, looks up an EVPN Type 5 route, then encapsulates VXLAN to the destination leaf.

---

## 12.1.3 Anycast gateway

**Every leaf advertises the same default gateway IP** (10.0.1.1/24) in VLAN 10. Each VM has the default gateway 10.0.1.1, and the actual gateway is the local leaf.

Two flavors:

- **Anycast gateway MAC plus IP:** every leaf shares the same MAC and IP. MAC duplication must be handled.
- **Distributed anycast gateway** (RFC 9135, modern): each leaf has its own MAC but shares the IP. EVPN Type 2 advertises the MAC-to-IP binding.

Advantages:
- L3 traffic does not leave the leaf, so latency is optimal.
- There is no single gateway bottleneck.
- VM migration is transparent because the gateway IP does not change.

---

## 12.1.4 Underlay: BGP unnumbered versus OSPF

**BGP unnumbered** (modern): eBGP between leaves and spines without IP addresses on the point-to-point link. It uses IPv6 link-local plus BGP peering. It scales well.

**OSPF:** the traditional IGP for medium scale. It is easier to debug.

Modern DC: BGP unnumbered underlay plus iBGP EVPN overlay.

---

## 12.1.5 Multi-tenant isolation

Each tenant has its own VRF on every leaf. The VRF maps to an EVPN instance (EVI). Tenant A traffic does not mix with tenant B traffic, because isolation is enforced through separate forwarding tables.

BGP EVPN Type 5 advertises an IP prefix together with an RD and an RT. It scales to thousands of VRFs.

---

## 12.1.6 Multi-site fabric

- **DCI** (Data Center Interconnect): BGP EVPN peering between fabrics.
- **Stretched VLAN** through VXLAN-over-WAN.
- **Anycast gateway stretch:** a VM can migrate between DCs without a gateway change.

### ▶ Guided Exercise 12: Mini-fabric, 2 leaves with FRR EVPN (design)

**Outcomes:** Design a 2-leaf, 1-spine fabric and write the FRR configuration.

**Instructions:**

**1.** Topology:

```
       [Spine] loopback 10.255.255.1
         │ │
    ┌────┘ └────┐
[Leaf 1]     [Leaf 2]
  10.255.255.11 10.255.255.12
   │            │
  VM-A 10.0.10.1  VM-B 10.0.10.2
```

**2.** FRR config on Leaf 1:

```
router bgp 65011
 bgp router-id 10.255.255.11
 neighbor spine peer-group
 neighbor spine remote-as 65001
 neighbor eth0 interface peer-group spine
 !
 address-family ipv4 unicast
  neighbor spine activate
  network 10.255.255.11/32
 !
 address-family l2vpn evpn
  neighbor spine activate
  advertise-all-vni
!
```

**3.** VNI mapping:

```bash
[leaf1]# ip link add vxlan10 type vxlan id 10010 local 10.255.255.11 dstport 4789 nolearning
[leaf1]# ip link add br10 type bridge
[leaf1]# ip link set vxlan10 master br10
```

**4.** Verify:

```bash
[leaf1]# vtysh -c "show bgp l2vpn evpn summary"
[leaf1]# vtysh -c "show bgp l2vpn evpn route"
```

## Key points to remember

First, a modern DC fabric is a 5-layer stack. Each layer replaces one responsibility from the 3-tier legacy.

Second, anycast gateway means L3 traffic does not leave the leaf, scale is linear, and VM migration is transparent.

Third, BGP unnumbered underlay plus iBGP EVPN overlay is the modern best practice.

Fourth, multi-tenant isolation goes through VRF plus EVPN Type 5. Multi-site goes through DCI.
