## Dàn bài

## 3.1.1 OpenFlow 1.0 spec công bố 31/12/2009 (Version 1.0.0)

Giới thiệu timeline spec: 0.8.0 (05/2008) → 0.8.9 (12/2008) → 0.9.0 (07/2009) → 1.0.0 (31/12/2009, 42 trang, Stanford University). Nêu organizational context: lúc công bố ONF chưa thành lập (21/03/2011) nên 1.0 do Stanford Clean Slate Lab shepherd; quyền tác giả chuyển sang ONF khi tổ chức này ra đời. Pointer đến current archive tại OpenNetworking/opennetworking.org/sdn-standards/openflow/.

## 3.1.2 Wire protocol: TCP 6633 plain / TCP 6653 TLS

Mô tả transport channel: TCP port 6633 (historical, pre-IANA) cho plain connection, TCP port 6653 (IANA assigned 09/2013 qua IETF) cho TLS-encrypted channel. Nêu lý do spec 1.0 chưa bắt buộc TLS: deployment campus nội bộ không có kẻ tấn công man-in-the-middle assumed. Pointer đến security issues được raise ở Part 5.x khi bàn về SDN security model.

## 3.1.3 Message types: Hello, Features, FlowMod, PacketIn, PacketOut, FlowRemoved, Error

Phân loại message thành ba nhóm theo spec §5.1-5.4: (a) symmetric — OFPT_HELLO, OFPT_ECHO_REQUEST/REPLY (có thể khởi phát bởi một trong hai bên); (b) controller-to-switch — OFPT_FEATURES_REQUEST, OFPT_FLOW_MOD, OFPT_PACKET_OUT, OFPT_PORT_MOD (controller chủ động gửi switch); (c) async — OFPT_PACKET_IN, OFPT_FLOW_REMOVED, OFPT_PORT_STATUS, OFPT_ERROR (switch chủ động thông báo controller). Phần này chỉ liệt kê + nêu role — không đi sâu wire format binary, nội dung đó ở Part 4.x OpenFlow evolution.

## 3.1.4 12-tuple match fields — chi tiết từng field

Liệt kê 12 match fields theo struct `ofp_match` (spec 1.0 §A.2.3): ingress port, Ethernet src/dst/type/VLAN ID/VLAN PCP (6 L2), IP src/dst/ToS/protocol (4 L3), TCP/UDP src/dst port (2 L4). Nêu wildcard mask OFPFW_ALL (match bất kỳ) và individual bitmask cho mỗi field. So sánh với match field của OVS trước OpenFlow (NXM — Nicira Extensible Match): OVS đã có nhiều field hơn nhưng OF 1.0 chỉ chuẩn hóa 12. Phần này KHÔNG lặp GSMP label field từ Part 2.0 §2.0.4 — chỉ cross-ref mapping đã đề cập.

## 3.1.5 Actions list — semantics

Phân loại 8 action chính trong spec 1.0 §A.2.6: OUTPUT (forwarding với target PORT/CONTROLLER/NORMAL/FLOOD/ALL/LOCAL/TABLE/IN_PORT), SET_VLAN_VID, SET_VLAN_PCP, STRIP_VLAN, SET_DL_SRC/DST, SET_NW_SRC/DST/TOS, SET_TP_SRC/DST, ENQUEUE (QoS). Nêu nguyên tắc ordered execution: list thực thi tuần tự theo thứ tự xuất hiện; nếu action cuối không có OUTPUT thì packet bị drop implicit. Giải thích sự khác biệt OUTPUT:CONTROLLER (gửi cả packet lên controller qua PACKET_IN) vs OUTPUT:NORMAL (chuyển cho non-OF pipeline legacy của switch).

## 3.1.6 Flow entry: match + priority + counters + timeouts + cookie + actions

Mô tả anatomy flow entry theo spec §3.3: header (match fields + priority 16-bit + idle_timeout + hard_timeout + cookie 64-bit), counters (packet/byte count + duration), instructions (actions). Nêu idle_timeout (evict nếu không có packet match trong N giây) vs hard_timeout (evict bất kể traffic sau N giây). Cookie là opaque 64-bit cho controller bookkeeping — không ảnh hưởng matching/forwarding, chỉ để controller truy vết flow nào do đâu install.

## 3.1.7 Single-table limitation và workaround (chaining via FLOOD/resubmit)

Phân tích hạn chế single flow table trong OF 1.0: mọi quy tắc phải tích hợp trong một bảng duy nhất → cross-product explosion khi kết hợp nhiều dimension (ví dụ: N VLAN × M IP prefix × K ACL rule = N·M·K entries). Nêu hai workaround giai đoạn 2010-2011: (a) OVS extension `resubmit` (NXM vendor extension, không chuẩn OF), (b) controller-side logic — pre-compute cross-product rồi install. Forward reference đến OF 1.1 (02/2011) giới thiệu multi-table pipeline, và OF 1.3 (06/2012) thêm group table — nội dung này ở Part 4.x OpenFlow evolution.
