## 3. New Governance Principles (GP-6 đến GP-10)

Bổ sung vào `memory/sdn/governance-principles.md`. Binding cho Phase R2-R6.

### GP-6: Per-Keyword Commit Pattern (mandatory)

Mỗi Phase G content commit MUST có **một** trong 2 form:

**Form A (preferred):** 1 keyword full 20-axis treatment, min 50 dòng substantive content (cornerstone), 30 dòng (medium), 15 dòng (peripheral).

**Form B (acceptable):** Small group ≤ 5 keyword với explicit per-keyword section, mỗi keyword có header riêng + 20-axis explicit + min 30/20/10 dòng tier-respective.

**Form FORBIDDEN:**
- Cohort table 5 axis × N keyword (gaming pattern)
- Cosmetic cross-link consolidation
- "Group inheritance" (1 axis statement covers multiple keyword without per-keyword evidence)

### GP-7: Cohort Stamp Forbidden

Single axis statement covering > 1 keyword = counted as 0 axis per keyword (zero progress, not partial).

### GP-8: Cosmetic Stamp Forbidden

Cross-link consolidation tables marking cohort "DONE/COVERED/STAMPED" mà không add per-keyword axis content = counted as 0 progress.

### GP-9: Min Lines Per Keyword

| Tier | Min lines per keyword (substantive content) |
|------|---------------------------------------------|
| Cornerstone | 50 dòng |
| Medium | 30 dòng |
| Peripheral | 15 dòng |

Below threshold = treatment NOT counted toward tier target.

### GP-10: Pre-Commit Verification Mandatory

Audit script `scripts/anti_gaming_check.py` runs pre-commit on staged files:
- Count axis mention per keyword via regex pattern
- Flag cohort-stamp pattern (5 axis × N keyword form)
- Flag cosmetic stamp (cross-link table without content)
- Reject commit if violation detected (non-zero exit)

**Mục tiêu:** Build mechanism prevent gaming pattern at commit time, không chỉ documentation.

**Deliverables:**

1. `scripts/anti_gaming_check.py` (~300-500 dòng) — pre-commit hook script:
   - Parse staged `.md` files
   - Extract per-keyword section boundaries
