## §13.5b.1 type="" (default OVN-native)

**Khái niệm.** Default type, VM/container attached qua libvirt/CNI. ovn-controller claim by reading Open_vSwitch.external_ids:iface-id matching Logical_Switch_Port name. Loại: SBDB Port_Binding row.

**Importance.** This is the most common Port_Binding type in production OVN deployments, used for every VM and container workload attached through libvirt or a CNI plugin.

**Cơ chế hoạt động.** Claim mechanism: ovn-controller poll OVS Open_vSwitch.external_ids:iface-id, match với Logical_Switch_Port.name, update Port_Binding.chassis = local chassis UUID. Race: multiple chassis claim simultaneously cùng port → first-write-wins via OVSDB lock.

**Khác biệt giữa các phiên bản.** OVN 2.4 initial. Stable since.

**Trường hợp sự cố thực tế.** Production: VM live migration claim race causing transient connectivity loss. Fix: serialize migration step.

**Bài tập synthetic.** `ovs-vsctl set Interface tap-vm1 external_ids:iface-id=lsp-vm1` + observe Port_Binding.chassis update.

**So sánh với hệ khác.** OpenStack Neutron port binding direct analog. NSX-T VIF binding similar.

## §13.5b.2 type="patch"

**Khái niệm.** Patch port giữa LS↔LR hoặc LS↔LS internal. No physical port; logical bridge between datapath.

**Tầm quan trọng.** Tier 1. L3 routing inside OVN cần patch port LS-LR.

**Cơ chế hoạt động.** ovn-northd compile LRP Logical_Router_Port → patch port pair (1 trên LS side, 1 trên LR side). Both sides có Port_Binding.type=patch.

**Khác biệt giữa các phiên bản.** OVN 2.4 stable.

**Trường hợp sự cố thực tế.** Standard L3 routing pattern: every Logical_Router connect to Logical_Switch via patch port pair.

**Bài tập synthetic.** `ovn-nbctl lr-add lr-main; ovn-nbctl lrp-add lr-main lrp-to-ls 00:00:00:00:00:01 10.0.0.1/24` exercise.

**So sánh với hệ khác.** Linux bridge veth pair analog. Cisco SVI (Switched Virtual Interface) similar concept.

## §13.5b.3 type="localnet"

**Khái niệm.** Provider network bridge, no chassis claim, broadcast on physical bridge. Loại: SBDB Port_Binding row.

**Tầm quan trọng.** Tier 2 medium. Connect OVN logical network to physical underlay (provider network).

**Cơ chế hoạt động.** No claim — packet directly egress to physical bridge tag specified VLAN/none. Multiple chassis can simultaneously bridge.

**Khác biệt giữa các phiên bản.** OVN 2.6 introduce. 21.06 add multi-network support.

**Trường hợp sự cố thực tế.** Production: provider VLAN connect VM to external L2 (e.g., management VLAN 100).

**Bài tập synthetic.** `ovn-nbctl lsp-add ls-tenant lsp-localnet-100; ovn-nbctl lsp-set-type lsp-localnet-100 localnet; ovn-nbctl lsp-set-options lsp-localnet-100 network_name=physnet1` exercise.

**So sánh với hệ khác.** OpenStack Neutron flat/vlan provider network direct analog. NSX-T VLAN-backed segment similar.

## §13.5b.4 type="l2gateway"

**Khái niệm.** L2 bridge tới external network qua specific chassis (gateway-only).

**Tầm quan trọng.** Tier 2 medium. Rare in modern OVN (mostly replaced by localnet).

**Cơ chế hoạt động.** Claim by specific chassis (single point); broadcast forwarded only via that chassis.

**Khác biệt giữa các phiên bản.** OVN 2.4 introduce. Mostly deprecated 22.03+ in favor of localnet + HA_Chassis_Group.

**Bài tập synthetic.** Legacy config; uncommon in new deployment.

**So sánh với hệ khác.** OpenStack Neutron L2 gateway service plugin direct analog.

## §13.5b.5 type="l3gateway"

**Khái niệm.** Distributed gateway router redirect-chassis SNAT/DNAT (legacy term replaced by chassisredirect 21.06+).

**Tầm quan trọng.** Tier 2 medium (legacy).

**Khác biệt giữa các phiên bản.** Pre-OVN 21.06 distributed gateway architecture used l3gateway. 21.06+ migration to chassisredirect.

**So sánh với hệ khác.** OpenStack Neutron L3 agent direct analog (legacy).

## §13.5b.6 type="chassisredirect"

**Khái niệm.** Logical switch port virtual on redirect chassis cho DGP (Distributed Gateway Port).

**Importance.** This type is central to the distributed gateway architecture; without it, north-south traffic from a Logical_Router cannot egress through the redirect chassis.

**Cơ chế hoạt động.** Synthesized port (no real interface), exists only logically. Set redirect-chassis option on Logical_Router. Pipeline detects via REGBIT_NAT_REDIRECT (cited 13.18).

**Khác biệt giữa các phiên bản.** OVN 21.06 introduce.

**Trường hợp sự cố thực tế.** Distributed gateway pattern (cited 13.18 REGBIT_NAT_REDIRECT incident 2024-Q3).

**Bài tập synthetic.** `ovn-nbctl set Logical_Router lr-main options:redirect-chassis=chassis-1` exercise.

**So sánh với hệ khác.** Cisco HSRP gateway similar concept (active gateway designation).

## §13.5b.7 type="vtep"

**Khái niệm.** Hardware VTEP integration via VTEP schema (ovs-vtep). Bridge OVN logical L2 với hardware switch supporting VXLAN.

**Tầm quan trọng.** Tier 3 peripheral. Niche use case (HW VTEP gateway).

**Cơ chế hoạt động.** ovs-vtep daemon decode HW VTEP commands; OVN treats VTEP as remote chassis. Cross-link 9.29 vtep-ctl.

**Khác biệt giữa các phiên bản.** OVN 2.6 introduce.

**Bài tập synthetic.** `ovn-nbctl lsp-add ls-tenant lsp-vtep; ovn-nbctl lsp-set-type lsp-vtep vtep` + ovs-vtep config exercise.

**So sánh với hệ khác.** Cisco Nexus 9000 VTEP integration direct analog. Arista L3 leaf VTEP similar.

## §13.5b.8 type="external"

**Khái niệm.** External port managed via external system (e.g., direct SR-IOV passthrough).

**Tầm quan trọng.** Tier 3 peripheral. Use case: SR-IOV VF direct attach to VM.

**Khác biệt giữa các phiên bản.** OVN 21.06 introduce.

**Bài tập synthetic.** Niche; specific to SR-IOV deployment.

**So sánh với hệ khác.** OpenStack Neutron direct port binding analog. Hardware vendor-specific.
