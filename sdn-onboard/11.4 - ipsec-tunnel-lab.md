## 11.4.1 Tại sao IPsec cho overlay

Overlay encap cung cấp tenant isolation nhưng **không encrypt**. Packet tenant tồn tại plaintext trong outer IP → ai có access underlay có thể sniff + decode.

IPsec bắt buộc trong hai scenario:

1. **Cloud-hybrid**: overlay qua untrusted Internet.
2. **Multi-tenant DC**: compliance (HIPAA, PCI-DSS) yêu cầu encryption-at-wire.

IPsec encrypt outer packet với ESP (Encapsulating Security Payload).

---

## 11.4.2 ovs-monitor-ipsec daemon

Compass Ch 12: "`ovs-monitor-ipsec` watches the `Interface.options:psk` or `Interface.options:remote_cert` columns and programs strongSwan or Libreswan accordingly."

Kiến trúc:

1. Admin cấu hình tunnel + IPsec option qua `ovs-vsctl`.
2. `ovs-monitor-ipsec` daemon subscribe OVSDB, detect change.
3. Daemon generate config cho strongSwan + reload.
4. strongSwan establish IPsec SA với remote peer.
5. OVS tunnel traffic flow qua IPsec encap.

Admin không touch strongSwan config trực tiếp.

## 11.4.3 Setup với PSK

Compass Ch 12 example:

```bash
[hostA]# ovs-vsctl add-port br0 vxlan-ipsec -- \
         set Interface vxlan-ipsec type=vxlan \
         options:remote_ip=10.0.0.2 \
         options:key=flow \
         options:psk=mysecretkey123

[hostA]# systemctl enable --now openvswitch-ipsec
```

Tương tự host B với `remote_ip=10.0.0.1` và cùng PSK.

Verify:

```bash
[hostA]# ipsec statusall
# Output: Security Associations (1 up, 0 connecting)
```

Rotate PSK: `ovs-vsctl set Interface vxlan-ipsec options:psk=newsecret`. ovs-monitor-ipsec reload atomic.

## 11.4.4 Setup với certificate (production)

PSK có limitations: một key cho toàn fabric (compromise → rotate mọi peer).

Certificate-based: mỗi endpoint có cert signed bởi CA.

```bash
# CA:
[ca]# ovs-pki req+sign ipsec-node1 ipsec
[ca]# ovs-pki req+sign ipsec-node2 ipsec

# Host A:
[hostA]# cp ipsec-node1-cert.pem /etc/openvswitch/ipsec-cert.pem
[hostA]# cp ipsec-node1-privkey.pem /etc/openvswitch/ipsec-key.pem

[hostA]# ovs-vsctl add-port br0 vxlan-ipsec -- \
         set Interface vxlan-ipsec type=vxlan \
         options:remote_ip=10.0.0.2 options:key=flow \
         options:remote_cert=<host-B-cert-path>
```

Rotation: issue cert mới, không cần sync secret.

## 11.4.5 GRE over IPsec

Site-to-site VPN classic: GRE cho flexibility + IPsec encryption.

```bash
[hostA]# ovs-vsctl add-port br0 gre-ipsec -- \
         set Interface gre-ipsec type=gre \
         options:remote_ip=10.0.0.2 \
         options:key=100 \
         options:psk=sharedkey
```

### ▶ Guided Exercise 11: VXLAN over IPsec với PSK

**Outcomes:** Setup 2-host VXLAN với IPsec, verify encrypted traffic.

**Before You Begin:**

```bash
[both]# apt install openvswitch-ipsec strongswan
```

**Instructions:**

**1.** Host A (10.0.0.1):

```bash
[hostA]# ovs-vsctl add-br br-ipsec
[hostA]# ovs-vsctl add-port br-ipsec tap-a -- set Interface tap-a type=internal
[hostA]# ip addr add 192.168.1.1/24 dev tap-a && ip link set tap-a up
[hostA]# ovs-vsctl add-port br-ipsec vxlan-ipsec -- set Interface vxlan-ipsec \
         type=vxlan options:remote_ip=10.0.0.2 options:key=100 \
         options:psk=mysecretkey
[hostA]# systemctl start openvswitch-ipsec
```

**2.** Host B tương tự với `remote_ip=10.0.0.1`.

**3.** Verify SA:

```bash
[hostA]# ipsec statusall
```

**4.** Test encrypted traffic:

```bash
[hostA]# ping 192.168.1.2 &
[hostA]# tcpdump -ni eth0 -s 0 host 10.0.0.2
```

tcpdump hiện protocol ESP (IP proto 50), không UDP 4789. Payload opaque.

**5.** So sánh không IPsec:

```bash
[hostA]# ovs-vsctl remove Interface vxlan-ipsec options psk
[hostA]# tcpdump -ni eth0 udp port 4789
```

VXLAN decode clear-text.

**6.** Cleanup: `ovs-vsctl del-br br-ipsec && systemctl stop openvswitch-ipsec`.

## 11.4.6 Production considerations

**Cipher suite**: force AES-256-GCM cho modern deployment.

**Performance**: IPsec overhead 10-30% CPU. NIC IPsec offload (Intel XL710, ConnectX-6) giảm đáng kể:

```bash
[host]# ethtool -k eth0 | grep ipsec
```

**Key management**: PSK cho lab. Production: cert-based + ovs-pki hoặc Vault/step-ca.

---

## Điểm cốt lõi cần nhớ

Thứ nhất, IPsec bắt buộc cho overlay qua untrusted network + compliance requirement.

Thứ hai, `ovs-monitor-ipsec` automate strongSwan programming qua OVSDB — admin chỉ set option trên Interface.

Thứ ba, PSK cho lab, cert-based cho production. Cert-based scale tốt hơn.

Thứ tư, NIC IPsec offload (ConnectX-6, Intel XL710) giảm 10-30% overhead CPU.
