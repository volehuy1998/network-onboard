### 0.2. Why it happened (root cause)

V3.8 R5 spot-check 30/331 keyword (9% sample) random sample ÷ aggregate scorecard verification. Stratified sampling KHÔNG cover:

1. **Sub-tooling gap.** `rubric_leak_check.py` v1 regex chỉ catch `**Axis N`, `cohort C\d+`, `Phase G batch`, tier labels DEEP-N. KHÔNG catch:
   - `### \d+\. (Khái niệm|Lịch sử|Vị trí|...)` axis-numbered VN heading (9.32 §9.32.1/§9.32.2 pattern)
   - `cohort cornerstone` phrase trong heading body (9.2:870)
   - `Tier 1 cornerstone tuyệt đối` informal tier reference (9.2/9.22/9.24, ~12 instances)
   - `Phase H session S39` phase reference embedded section header (9.11)
   - `(compact treatment per cohort batch limit)` cohort-batch-stamp leftover (9.32:526)
   - `(reference, giữ tương thích content Phase [A-Z])` stale plan compatibility note (9.9:58)

2. **Manual 30/331 sampling miss.** 9% coverage. Block 10 (10.3, 10.4, 10.5) cornerstone JSON-RPC fact-error ở dạng prose 1-2 dòng giữa file 1500+ dòng. Random sampling không guarantee hit.

3. **Sibling-file numbering collision** (20.0 ↔ 20.1 ↔ 20.7) là cross-file structural bug, không expose qua per-file scorecard. R5 audit per-file, không cross-file.

4. **Systemic gap (axis 17 incident, axis 20 cross-domain)** dạng "giảm dần đều" qua nhiều file, mỗi file score axis 17 = 0.5-1 (partial credit), aggregate vẫn pass DEEP-15/DEEP-20. Pattern này hidden nếu chỉ check threshold per-keyword.

## 3. New Governance Principle GP-12

Bổ sung vào `memory/sdn/governance-principles.md` v1.2. Binding cho v3.9+ và mọi plan tương lai có tag claim "comprehensive/mastery/full".

### 3.1. Definition

**GP-12: Post-Tag Regression Audit Cadence (mandatory).**

Sau mỗi tag `v4.x-MasteryComplete` hoặc `v4.x-FullDepth` hoặc tương đương "comprehensive" claim, **bắt buộc** chu kỳ audit:

- **T+0 đến T+7 ngày:** Master block-level audit (multi-agent parallel) cho ≥1 block major (OVS hoặc OVN hoặc OF). Audit MUST cover:
  - Per-file 20-axis tier verification (random subset 20-30% file/block)
  - Cross-file structural integrity (numbering collision, broken cross-link)
  - Systemic axis under-coverage detection (axis 17 incident, axis 20 cross-domain, axis 14 version drift across block)
  - Editorial placeholder/botched-replacement scan (`XXXXXX`, `thị field`, `TODO`, `FIXME`)
  - Rule 14 source citation verification (sample SHA via MCP GitHub)

- **Findings categorize** theo severity: CRITICAL / HIGH / MEDIUM / LOW.

- **CRITICAL/HIGH findings** trigger hotfix plan (như v3.9) within 14 ngày.

- **MEDIUM/LOW findings** logged trong `memory/sdn/post-tag-audit-YYYY-MM-DD.md` cho future plan address.

### 3.2. Rationale

V3.8 R5 user spot-check 30/331 keyword (9% sample). Stratified random sampling không đủ catch:
- Sub-tooling gap (regex pattern blind spot)
- Cross-file structural collision
- Systemic giảm-dần-đều patterns (axis 17, axis 20)

Master block-level multi-agent audit có depth-per-file + breadth-cross-file + systemic-pattern detection mà spot-check không đạt được. Cadence T+7 ngày catch issues khi context còn fresh.

### 3.3. Enforcement Mechanism

- Plan tracker entry `memory/sdn/post-tag-audit-schedule.md` track cadence
- Tag annotated message reference scheduled audit date
- Audit findings file committed `memory/sdn/post-tag-audit-<tag>-<date>.md`
- CRITICAL/HIGH findings → hotfix plan within 14 ngày OR explicit deferral with justification
- Hook check (optional, future): `scripts/post_tag_audit_due.py` cảnh báo nếu tag > 7 ngày without audit file

### 3.4. Exception Policy

- **Pre-release tag (vX.Y-rcN):** không yêu cầu full block audit, chỉ spot-check
- **Hotfix tag (vX.Y.Z):** không trigger thêm cadence (đã là result của audit cycle)
- **Tag scope explicit limited** (e.g., `v3.6-AuditTooling` chỉ tooling): chỉ audit phạm vi tag claim
- **No other exceptions.** Comprehensive tag = comprehensive audit follow-up.

### 3.5. Approval

GP-12 proposed bởi Plan v3.9 Section 3. Yêu cầu user explicit approval per existing GP-amendment convention (Section 8 `governance-principles.md`). Approval message format chấp nhận: "approve GP-12" hoặc "đồng ý GP-12" trong chat hoặc commit message Phase S8.
