## Dàn bài

## 8.0.1 Linux namespaces: 7 loại

Kernel 3.8+ ship đủ 7 namespace: PID (process ID isolation), NET (network stack isolation), MNT (mount point isolation), UTS (hostname isolation), IPC (System V IPC isolation), USER (UID/GID remapping), CGROUP (cgroup hierarchy isolation). Mỗi namespace tạo qua flag trong `clone(2)` syscall (CLONE_NEWNET, CLONE_NEWPID, v.v.) hoặc `unshare(2)` cho existing process.

## 8.0.2 Network namespace: ip netns, /var/run/netns

Network namespace có network stack độc lập: interface, routing table, ARP cache, iptables rules, socket table. `ip netns add nsname` tạo persistent namespace với reference tại `/var/run/netns/nsname`. `ip netns exec nsname <command>` chạy command trong namespace, wrapper quanh `setns(2)` syscall. Đây là cơ sở cho container networking, Mininet, OVS multi-tenancy lab.

## 8.0.3 unshare + nsenter

util-linux tools: `unshare --net /bin/bash` tạo shell mới trong fresh network namespace. `nsenter -t <pid> -n /bin/bash` join vào network namespace của existing PID. Khác `ip netns` ở chỗ unshare/nsenter có thể work với non-persistent namespace (tạo ad-hoc, exit khi process kết thúc).

## 8.0.4 cgroups v1 vs v2, network bandwidth class

cgroup v1: subsystem `net_cls` gán classid cho process, `net_prio` gán priority. `tc` filter theo classid để rate limit hoặc priority queue. cgroup v2: unified hierarchy, loại bỏ net_cls/net_prio, bandwidth control chuyển sang eBPF. Distro Ubuntu 22.04+ default cgroup v2.
