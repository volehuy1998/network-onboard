## 12.1.1 Stack overview

Modern multi-tenant DC fabric:

```
Layer 4: Application (tenant workload)
Layer 3: Overlay VXLAN VNI per tenant
Layer 2: BGP EVPN control plane
Layer 1: Underlay BGP unnumbered hoặc OSPF
Layer 0: Physical leaf-spine Clos
```

Mỗi layer thay một responsibility trong 3-tier legacy:
- Clos thay aggregation/core.
- VXLAN thay VLAN (scale 16.7M).
- EVPN thay flood-and-learn.
- Anycast gateway thay single-router-as-gateway.

---

## 12.1.2 Traffic flow

**Tenant VM-to-VM cùng subnet (L2)**:

1. VM-A (leaf 1) gửi frame đến VM-B (leaf 3) cùng VLAN 10.
2. Leaf 1 lookup MAC — entry pre-populated qua EVPN Type 2 từ Leaf 3.
3. Leaf 1 encap VXLAN: outer src=Leaf 1, outer dst=Leaf 3, VNI=VLAN 10 mapping.
4. Packet traverse underlay via ECMP → spine → Leaf 3.
5. Leaf 3 decap VXLAN → forward ra port VM-B.

**Tenant VM-to-VM khác subnet (L3)**:
- Packet đến gateway IP (anycast trên mọi leaf).
- Leaf 1 route L3 → EVPN Type 5 lookup → encap VXLAN tới Leaf dst.

---

## 12.1.3 Anycast gateway

**Mọi leaf advertise cùng default gateway IP** (10.0.1.1/24) trong VLAN 10. Mỗi VM có default gateway = 10.0.1.1 — thực sự gateway là leaf local.

Hai loại:

- **Anycast gateway MAC+IP**: mọi leaf cùng MAC + IP. MAC duplication cần handle.
- **Distributed anycast gateway** (RFC 9135, modern): leaf có MAC riêng nhưng cùng IP. EVPN Type 2 advertise MAC-IP binding.

Ưu điểm:
- L3 traffic không leave leaf → latency tối ưu.
- Không single gateway điểm nghẽn.
- VM migration transparent — gateway IP không đổi.

---

## 12.1.4 Underlay: BGP unnumbered vs OSPF

**BGP unnumbered** (modern): eBGP giữa leaves + spines không cần IP trên P2P link. IPv6 link-local + BGP peering. Scale tốt.

**OSPF**: traditional IGP cho medium-scale. Easier debug.

Modern DC: BGP unnumbered underlay + iBGP EVPN overlay.

---

## 12.1.5 Multi-tenant isolation

Mỗi tenant có VRF riêng trên mọi leaf. VRF map với EVPN instance (EVI). Packet tenant A không mix với tenant B — isolation qua separate forwarding table.

BGP EVPN Type 5 advertise IP prefix với RD + RT. Scale hàng nghìn VRF.

---

## 12.1.6 Multi-site fabric

- **DCI** (Data Center Interconnect): BGP EVPN peering giữa fabrics.
- **Stretched VLAN** qua VXLAN-over-WAN.
- **Anycast gateway stretch**: VM migrate giữa DC không cần gateway change.

### ▶ Guided Exercise 12: Mini-fabric 2-leaf FRR EVPN (design)

**Outcomes:** Thiết kế 2-leaf 1-spine fabric, write FRR config.

**Instructions:**

**1.** Topology:

```
       [Spine] loopback 10.255.255.1
         │ │
    ┌────┘ └────┐
[Leaf 1]     [Leaf 2]
  10.255.255.11 10.255.255.12
   │            │
  VM-A 10.0.10.1  VM-B 10.0.10.2
```

**2.** FRR config Leaf 1:

```
router bgp 65011
 bgp router-id 10.255.255.11
 neighbor spine peer-group
 neighbor spine remote-as 65001
 neighbor eth0 interface peer-group spine
 !
 address-family ipv4 unicast
  neighbor spine activate
  network 10.255.255.11/32
 !
 address-family l2vpn evpn
  neighbor spine activate
  advertise-all-vni
!
```

**3.** VNI mapping:

```bash
[leaf1]# ip link add vxlan10 type vxlan id 10010 local 10.255.255.11 dstport 4789 nolearning
[leaf1]# ip link add br10 type bridge
[leaf1]# ip link set vxlan10 master br10
```

**4.** Verify:

```bash
[leaf1]# vtysh -c "show bgp l2vpn evpn summary"
[leaf1]# vtysh -c "show bgp l2vpn evpn route"
```

## Điểm cốt lõi cần nhớ

Thứ nhất, modern DC fabric là stack 5 layer — mỗi layer thay một responsibility 3-tier legacy.

Thứ hai, anycast gateway → L3 traffic không leave leaf, scale tuyến tính, VM migration transparent.

Thứ ba, BGP unnumbered underlay + iBGP EVPN overlay là modern best practice.

Thứ tư, multi-tenant isolation qua VRF + EVPN Type 5. Multi-site qua DCI.
