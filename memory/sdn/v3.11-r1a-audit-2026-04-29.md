## 4. New patterns observed

### 4.1. Per-version prefix drop

Multiple curriculum citations drop the OF-version-introduction prefix from
the OVS enum name. Examples:

- `OFPGT_ALL` instead of `OFPGT11_ALL`
- `OFPMF_KBPS` instead of `OFPMF13_KBPS`
- `OFPMP_METER_FEATURES` instead of `OFPMP13_METER_FEATURES_REQUEST`

This is a `WRONG_NAME` violation because the OVS source uses the version
prefix as a load-bearing convention to distinguish per-version wire-format
codes. Stripping the prefix can mislead a reader into thinking the constant
is in the umbrella `openflow.h` umbrella header when in fact it lives in
`openflow-1.X.h` per the prefix.

R1.B and beyond should expect this pattern recurring on:
- `OFPT_*` messages (OFPT_HELLO, OFPT_ECHO_REQUEST, etc.) which are OF 1.0 baseline.
- `OFPAT11_*` actions vs `OFPAT_*` action umbrella.
- `OFPIT_*` instructions (OF 1.1+).
- `OFPHET_*` hello element types (OF 1.3).

### 4.2. Curriculum mis-attributing OF-version introduction

File 3.4 line 153 says OF 1.4 introduced the band-features query. OVS source
shows `OFPMP13_` prefix indicating OF 1.3 introduction. This is a
`WRONG_VERSION_INTRO` per the new category in plan §10.

### 4.3. Class value confusion

File 3.4 line 88 says `OXM_NXM_NX` has class value 0x8001. Per `lib/meta-flow.xml`
the OFPXMC class enum is `OFPXMC_NXM_0` (0x0000), `OFPXMC_NXM_1` (0x0001),
`OFPXMC_OPENFLOW_BASIC` (0x8000), `OFPXMC_PACKET_REGS` (0x8001),
`OFPXMC_EXPERIMENTER` (0xffff). 0x8001 is `PACKET_REGS`, not the Nicira
extension class.
