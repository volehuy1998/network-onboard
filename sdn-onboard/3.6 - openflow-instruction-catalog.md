# 3.6. OpenFlow instruction catalog (6 instruction)

> **Trạng thái:** Cập nhật 2026-04-26.

## Mục tiêu bài học

Sau Phần này, học viên có thể:

1. Phân biệt instruction (set of instructions) vs action (set vs list of actions).
2. Order of evaluation pipeline: Apply-Actions → Clear-Actions → Write-Actions → Write-Metadata → Goto-Table → Meter.
3. Use case khi nào dùng instruction nào (Apply vs Write semantics).
4. Debug instruction misuse qua ofproto/trace.

## Yêu cầu kiến thức

- Phần 3.1 (OF 1.0 spec), 4.0 (OF 1.1 multi-table + groups), 4.7 (OVS programming), 9.22 (multi-table pipeline).

## §3.6.1 Apply-Actions instruction

**Khái niệm.** Instruction execute action list immediately (modify packet now). Loại: OF instruction, OF 1.1+.

**Tầm quan trọng.** Most-used instruction trong OF flow rule.

**Cơ chế hoạt động.** Apply-Actions specifies action list executed in-order on current packet copy. Modifications visible to downstream instructions in same flow rule.

**Khác biệt giữa các phiên bản.** OF 1.1 introduce. Stable since.

**Trường hợp sự cố thực tế.** Production: typical flow rule có Apply-Actions với output, set_field, dec_ttl. Common pattern.

**Bài tập synthetic.** `ovs-ofctl add-flow br0 'priority=100,ip,actions=set_field:00:11:22:33:44:55->eth_dst,output:1'` exercise.

**So sánh với hệ khác.** Cisco IOS class-map action immediate similar concept.

---

## §3.6.2 Clear-Actions instruction

**Khái niệm.** Clear accumulated action set. Loại: OF instruction, OF 1.1+.

**Tầm quan trọng.** Critical cho complex multi-table pipeline reset.

**Cơ chế hoạt động.** Reset action set to empty. Useful before re-evaluating in different table context.

**Khác biệt giữa các phiên bản.** OF 1.1 introduce.

**Trường hợp sự cố thực tế.** Pipeline reset case: table 0 set action_set, table 5 wants different action — Clear-Actions reset trước Write-Actions mới.

**Bài tập synthetic.** `actions=clear_actions,goto_table:5` exercise.

**So sánh với hệ khác.** Cisco IOS no inheritance similar concept.

---

## §3.6.3 Write-Actions instruction

**Khái niệm.** Merge action list into action set (deferred execution at pipeline end). Loại: OF instruction.

**Tầm quan trọng.** Action set deferred execution model fundamental.

**Cơ chế hoạt động.** Write-Actions merges actions into action set. Action set dedup (only 1 action per type, e.g., 1 output). Final execution after pipeline ends.

**Khác biệt giữa các phiên bản.** OF 1.1 introduce.

**Trường hợp sự cố thực tế.** Production: most flow with goto_table chain use Write-Actions để defer output until last table.

**Bài tập synthetic.** `actions=write_actions(output:1),goto_table:1` exercise.

**So sánh với hệ khác.** Cisco IOS deferred queue action analog.

---

## §3.6.4 Write-Metadata instruction

**Khái niệm.** Write to 64-bit metadata register (passes between tables). Loại: OF instruction.

**Tầm quan trọng.** Metadata persistence cross-table critical cho multi-stage pipeline.

**Cơ chế hoạt động.** Write-Metadata sets value + mask:
```
write_metadata:0x12345678/0xffff0000
```
Lower bits unchanged, upper bits set 0x1234.

**Khác biệt giữa các phiên bản.** OF 1.1 introduce.

**Trường hợp sự cố thực tế.** Production: Write-Metadata used cho tagging packet across pipeline stages (e.g., tenant ID tag).

**Bài tập synthetic.** `actions=write_metadata:0x1,goto_table:1` then table 1 match metadata=0x1.

**So sánh với hệ khác.** Cisco IOS forwarding-class similar concept.

---

## §3.6.5 Goto-Table instruction

**Khái niệm.** Continue pipeline at specified next table. Loại: OF instruction.

**Tầm quan trọng.** Multi-table pipeline foundation.

**Cơ chế hoạt động.** `goto_table:N` advance to table N. N must > current table (forward only). OVS extension `resubmit` allows backward.

**Khác biệt giữa các phiên bản.** OF 1.1 introduce. OF 1.5 packet_type-aware variant.

**Trường hợp sự cố thực tế.** Standard 5-table OVS pipeline use goto_table chain (cited 9.22).

**Bài tập synthetic.** `actions=goto_table:1` then table 1 has additional rule.

**So sánh với hệ khác.** Cisco IOS table chain via class-map similar.

---

## §3.6.6 Meter instruction

**Khái niệm.** Redirect packet to meter for rate-limit / DSCP-remark before continuing. Loại: OF instruction, OF 1.3+.

**Tầm quan trọng.** QoS/DSCP foundation.

**Cơ chế hoạt động.** `meter:N` invoke Meter table entry N. Meter has bands (DROP / DSCP_REMARK) based on rate threshold.

**Khác biệt giữa các phiên bản.** OF 1.3 introduce. OF 1.5 enhance.

**Trường hợp sự cố thực tế.** Production: rate-limit egress traffic per VM via meter.

**Bài tập synthetic.** `ovs-ofctl add-meter br0 meter=1,kbps,band=type=drop,rate=10000` + `actions=meter:1,output:1`.

**So sánh với hệ khác.** Cisco IOS policer/shaper analog. Linux tc qdisc similar concept.
