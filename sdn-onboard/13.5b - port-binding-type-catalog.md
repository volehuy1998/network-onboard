## §13.5b.1 type="" (default OVN-native)

**Axis 1 concept.** Default type, VM/container attached qua libvirt/CNI. ovn-controller claim by reading Open_vSwitch.external_ids:iface-id matching Logical_Switch_Port name. Loại: SBDB Port_Binding row.

**Axis 7.** Tier 1 cornerstone. Most common type cho VM/container workload.

**Axis 8.** Claim mechanism: ovn-controller poll OVS Open_vSwitch.external_ids:iface-id, match với Logical_Switch_Port.name, update Port_Binding.chassis = local chassis UUID. Race: multiple chassis claim simultaneously cùng port → first-write-wins via OVSDB lock.

**Axis 14.** OVN 2.4 initial. Stable since.

**Axis 17.** Production: VM live migration claim race causing transient connectivity loss. Fix: serialize migration step.

**Axis 18.** `ovs-vsctl set Interface tap-vm1 external_ids:iface-id=lsp-vm1` + observe Port_Binding.chassis update.

**Axis 20.** OpenStack Neutron port binding direct analog. NSX-T VIF binding similar.

## §13.5b.2 type="patch"

**Axis 1.** Patch port giữa LS↔LR hoặc LS↔LS internal. No physical port; logical bridge between datapath.

**Axis 7.** Tier 1. L3 routing inside OVN cần patch port LS-LR.

**Axis 8.** ovn-northd compile LRP Logical_Router_Port → patch port pair (1 trên LS side, 1 trên LR side). Both sides có Port_Binding.type=patch.

**Axis 14.** OVN 2.4 stable.

**Axis 17.** Standard L3 routing pattern: every Logical_Router connect to Logical_Switch via patch port pair.

**Axis 18.** `ovn-nbctl lr-add lr-main; ovn-nbctl lrp-add lr-main lrp-to-ls 00:00:00:00:00:01 10.0.0.1/24` exercise.

**Axis 20.** Linux bridge veth pair analog. Cisco SVI (Switched Virtual Interface) similar concept.

## §13.5b.3 type="localnet"

**Axis 1.** Provider network bridge, no chassis claim, broadcast on physical bridge. Loại: SBDB Port_Binding row.

**Axis 7.** Tier 2 medium. Connect OVN logical network to physical underlay (provider network).

**Axis 8.** No claim — packet directly egress to physical bridge tag specified VLAN/none. Multiple chassis can simultaneously bridge.

**Axis 14.** OVN 2.6 introduce. 21.06 add multi-network support.

**Axis 17.** Production: provider VLAN connect VM to external L2 (e.g., management VLAN 100).

**Axis 18.** `ovn-nbctl lsp-add ls-tenant lsp-localnet-100; ovn-nbctl lsp-set-type lsp-localnet-100 localnet; ovn-nbctl lsp-set-options lsp-localnet-100 network_name=physnet1` exercise.

**Axis 20.** OpenStack Neutron flat/vlan provider network direct analog. NSX-T VLAN-backed segment similar.

## §13.5b.4 type="l2gateway"

**Axis 1.** L2 bridge tới external network qua specific chassis (gateway-only).

**Axis 7.** Tier 2 medium. Rare in modern OVN (mostly replaced by localnet).

**Axis 8.** Claim by specific chassis (single point); broadcast forwarded only via that chassis.

**Axis 14.** OVN 2.4 introduce. Mostly deprecated 22.03+ in favor of localnet + HA_Chassis_Group.

**Axis 18.** Legacy config; uncommon in new deployment.

**Axis 20.** OpenStack Neutron L2 gateway service plugin direct analog.

## §13.5b.5 type="l3gateway"

**Axis 1.** Distributed gateway router redirect-chassis SNAT/DNAT (legacy term replaced by chassisredirect 21.06+).

**Axis 7.** Tier 2 medium (legacy).

**Axis 14.** Pre-OVN 21.06 distributed gateway architecture used l3gateway. 21.06+ migration to chassisredirect.

**Axis 20.** OpenStack Neutron L3 agent direct analog (legacy).

## §13.5b.6 type="chassisredirect"

**Axis 1.** Logical switch port virtual on redirect chassis cho DGP (Distributed Gateway Port).

**Axis 7.** Tier 1 cornerstone trong distributed gateway architecture.

**Axis 8.** Synthesized port (no real interface), exists only logically. Set redirect-chassis option on Logical_Router. Pipeline detects via REGBIT_NAT_REDIRECT (cited 13.18).

**Axis 14.** OVN 21.06 introduce.

**Axis 17.** Distributed gateway pattern (cited 13.18 REGBIT_NAT_REDIRECT incident 2024-Q3).

**Axis 18.** `ovn-nbctl set Logical_Router lr-main options:redirect-chassis=chassis-1` exercise.

**Axis 20.** Cisco HSRP gateway similar concept (active gateway designation).

## §13.5b.7 type="vtep"

**Axis 1.** Hardware VTEP integration via VTEP schema (ovs-vtep). Bridge OVN logical L2 với hardware switch supporting VXLAN.

**Axis 7.** Tier 3 peripheral. Niche use case (HW VTEP gateway).

**Axis 8.** ovs-vtep daemon decode HW VTEP commands; OVN treats VTEP as remote chassis. Cross-link 9.29 vtep-ctl.

**Axis 14.** OVN 2.6 introduce.

**Axis 18.** `ovn-nbctl lsp-add ls-tenant lsp-vtep; ovn-nbctl lsp-set-type lsp-vtep vtep` + ovs-vtep config exercise.

**Axis 20.** Cisco Nexus 9000 VTEP integration direct analog. Arista L3 leaf VTEP similar.

## §13.5b.8 type="external"

**Axis 1.** External port managed via external system (e.g., direct SR-IOV passthrough).

**Axis 7.** Tier 3 peripheral. Use case: SR-IOV VF direct attach to VM.

**Axis 14.** OVN 21.06 introduce.

**Axis 18.** Niche; specific to SR-IOV deployment.

**Axis 20.** OpenStack Neutron direct port binding analog. Hardware vendor-specific.
