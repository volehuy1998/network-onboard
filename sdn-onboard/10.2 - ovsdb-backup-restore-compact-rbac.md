## Dàn bài

## 10.2.1 Append-only log và lý do phải compact

OVSDB file `/etc/openvswitch/conf.db` (hoặc cluster DB) là append-only journal. Mỗi transaction add một entry, không rewrite toàn file. Ưu điểm: atomic write, easy replication (Raft log shipping). Nhược điểm: file size tăng tuyến tính với transaction count, không tương quan với state size.

Compact: rewrite file thành snapshot của state hiện tại, xóa transaction history cũ. Sau compact, file có thể nhỏ hơn 10-100×.

## 10.2.2 `ovsdb-tool compact` (offline) vs live compaction

Offline: `ovsdb-tool compact /etc/openvswitch/conf.db`. CHỈ chạy khi ovsdb-server KHÔNG active — nếu daemon đang write đồng thời, corruption.

Live: `ovs-appctl -t ovsdb-server ovsdb-server/compact`. An toàn với daemon đang chạy — daemon hold lock, swap file atomically.

Khuyến nghị production: dùng live, schedule cron hàng tuần hoặc khi file vượt ngưỡng (ví dụ >100 MB cho conf.db cỡ trung).

## 10.2.3 Backup + restore workflow

Snapshot current state:
```
ovsdb-client backup > /tmp/conf.db.bak
```

Restore (fresh DB, overwrite current):
```
ovsdb-client restore < /tmp/conf.db.bak
```

Lưu ý: `restore` là destructive — overwrite toàn bộ state hiện tại. Backup định kỳ (nightly cron) + retain 7-30 ngày là best practice.

## 10.2.4 Schema conversion cho upgrade

`ovsdb-tool db-version <db>` → show schema version hiện tại.

`ovsdb-tool needs-conversion <db> <new-schema-json>` → detect schema drift.

`ovsdb-tool convert <db> <new-schema-json>` → upgrade schema trong-place. Thường chạy trong 5-step upgrade choreography (xem Part 9.12).

## 10.2.5 Transaction log inspection

`ovsdb-tool show-log` → print human-readable transaction history. Debug: nếu DB nghi ngờ corrupt, `show-log` hiển thị last transactions before suspected issue.

`ovsdb-tool db-cksum` → compute checksum. So sánh giữa các node Raft cluster để detect divergence.

## 10.2.6 RBAC: `Manager.role` model

Permission model: `Manager.role=<role-name>` gán client connecting qua `ssl:` với role. Schema định nghĩa role → danh sách (table, columns, condition) client được read/write.

Ví dụ trong OVN SBDB: `ovn-controller` dùng role hạn chế — chỉ được update `Chassis` row của mình và `Port_Binding` row nó claim. Không thể write bảng khác.

Verify: `ovsdb-client -c <role-identity> transact ...` — nếu violate, server return permission error.

## 10.2.7 Cluster migration: fresh member khi leave

**Quan trọng:** Khi một server `cluster/leave` khỏi Raft cluster, nó KHÔNG thể rejoin trực tiếp. Phải:
1. Stop ovsdb-server.
2. Delete local DB file.
3. `ovsdb-tool join-cluster` với fresh node identity.

Không tuân quy tắc này = split-brain risk.
