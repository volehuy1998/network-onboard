## Quy ước (5-axis + status code + scope)

### 5-axis classification (Bucket | Context | Purpose | Activity | Mechanism)

Mỗi keyword được phân loại theo 5 chiều:
- **Bucket:** category (OVS daemon, OF action, OVN schema, etc.)
- **Context:** khi nào keyword xuất hiện (log, CLI, spec)
- **Purpose:** mục đích keyword giải quyết
- **Activity:** hành vi runtime / use case chính
- **Mechanism:** cách hoạt động bên trong

### Megaflow

**5-axis:** OVS datapath flow cache | kernel/userspace datapath | cache wildcarded entry per equivalence class thay vì microflow | on-miss userspace compute smallest mask sufficient + install megaflow | each megaflow carry mask + resolved actions, subsequent matching packets bypass userspace

Cache wildcarded flow trong kernel datapath. Mỗi megaflow đại diện 1 equivalence class packet (cùng mask, cùng action). Dramatically giảm số flow trong kernel cache so với microflow (1 entry per packet). Status: **DEEP** ✅

### Megaflow wildcarding

**5-axis:** OVS datapath optimization | userspace classifier | kernel cache nhỏ bằng mask out fields không examined | xlate record "fields touched" + unmasked field wildcarded | poorly-written rule reduce wildcarding + explode kernel flow

Optimization: chỉ ghi flow với mask trên field thực sự matched. Field không touched trong xlate được wildcarded (ignored). Rule viết kém → wildcarding kém → kernel flow count nổ. Status: **BREADTH** 🔵

### Upcall

**5-axis:** OVS datapath event | kernel→userspace when no datapath flow match | deliver unmatched packet to ovs-vswitchd → consult OpenFlow tables | handler thread dequeue + run xlate + install megaflow + re-inject | netlink (NLMSG/OVS_PACKET_CMD_MISS) cho kernel datapath, channel queue cho netdev

Cơ chế kernel báo userspace khi không match flow nào. Handler thread userspace nhận, run OpenFlow pipeline simulation, install megaflow result, re-inject packet. Tốc độ upcall = bottleneck slow path. Status: **DEEP** ✅

**5-axis:** OVS thread inside ovs-vswitchd | xử lý upcall + install datapath flow | parse upcall payload + run xlate + write back via dpif | tunable `other_config:n-handler-threads`

Thread đơn vị xử lý upcall trong ovs-vswitchd. Mỗi thread parse upcall, simulate pipeline, install megaflow rule. Số thread tunable (default ~CPU count). Bottleneck nếu upcall rate > thread capacity. Status: **DEEP** ✅

### Interface type: `patch`

**5-axis:** OVSDB Interface.type | virtual wire 2 OVS bridge không qua kernel networking stack | canonical way connect br-int ↔ br-ex / br-provider trong OVN | patch ports come in pair (peer reference via options:peer) + packet emit one port appears instantly on partner | no kernel veth/net_device

Patch port = "wire" giữa 2 OVS bridge cùng host. KHÔNG qua kernel networking stack (không có veth/net_device). Cấu hình pair: `bridge1.port1.options:peer=port2` + `bridge2.port2.options:peer=port1`. Use case canonical: OVN br-int ↔ br-ex (external network). Status: **DEEP** ✅

### VLAN bridge modes (`Port.vlan_mode`)

**5-axis:** OVSDB Port.vlan_mode column | control 802.1Q frame admit + send | provisioning + VLAN segmentation | 4 value: access/trunk/native-tagged/native-untagged

4 mode VLAN trên port:
- `access`: 1 VLAN duy nhất (`tag` column); incoming untagged → classified vào VLAN; outgoing strip tag. Tagged frame inbound → DROP (default cho VM-facing port OVN).
- `trunk`: carry VLAN trong `trunks` list (empty = all); frame must arrive + leave tagged.
- `native-tagged`: như trunk, nhưng frame VLAN = `tag` field arrive untagged + emit tagged.
- `native-untagged`: như native-tagged nhưng frame VLAN `tag` emit UNTAGGED. Cho uplink interop với access switch.

Status: **DEEP** ✅

**Curriculum:** [Phần 9.20 OVS VLAN access + trunk](9.20%20-%20ovs-vlan-access-trunk.md) (toàn bộ file 337 dòng) + [Phần 8.2 Linux VLAN+bonding](8.2%20-%20linux-vlan-bonding-team.md)

### Bond mode (`Port.bond_mode`)

**5-axis:** OVSDB Port.bond_mode column | bonded port link aggregation algorithm | provisioning + HA + bandwidth aggregation | 3 mode: active-backup/balance-slb/balance-tcp

3 mode bond:
- `active-backup`: 1 slave forward, fail switch sang slave khác. Không cần LACP switch-side.
- `balance-slb` (Source-Load-Balance): hash flow theo source MAC + VLAN, no LACP. Slave change khi link fail.
- `balance-tcp`: hash 5-tuple (cần LACP 802.3ad upstream switch). Most granular, recommended cho perf-sensitive.

Status: **DEEP** ✅

**Curriculum:** [Phần 8.2 Linux VLAN+bonding+team](8.2%20-%20linux-vlan-bonding-team.md) (Linux foundation) + [Phần 9.4 §9.4.1 OVS bond mới](9.4%20-%20ovs-cli-tools-playbook.md)

4. **AUX state** (OF 1.3+): auxiliary connection cho parallel transmission
