## Mục tiêu bài học

Sau khi hoàn thành phần này, bạn có thể:

1. **Mô tả (Understand)** DPDK Poll Mode Driver architecture: hot-loop, RX/TX queue ring, burst processing (default 32 packets), DMA descriptors.
2. **Áp dụng (Apply)** setup hugepages 2MB vs 1GB, NUMA node pinning cho workers, memory channel interleaving, mempool creation với power-of-2 sizes.
3. **Phân tích (Analyze)** CPU cache effects: cache line aligned data structures, false sharing avoidance, prefetch hints.

## 16.1.1 PMD hot loop internals

*Skeleton — nội dung sẽ bao gồm:* `rte_eth_rx_burst()` loop, polling RX queue, burst size (default 32), per-queue lock-free ring buffer implementation.

## 16.1.2 Hugepages 2MB vs 1GB

*Skeleton — nội dung sẽ bao gồm:* TLB overhead (regular 4KB page → many TLB entries), 2MB huge (1024× reduction), 1GB gigantic (for large mempools), boot-time vs runtime allocation.

## 16.1.3 Memory channels + NUMA affinity

*Skeleton — nội dung sẽ bao gồm:* Intel 2-socket systems với 4-6 memory channels per socket, interleaving vs pinning, `numactl --hardware` inspection, DPDK `--lcores` + `--socket-mem` flags.

## 16.1.4 Cache line alignment + false sharing

*Skeleton — nội dung sẽ bao gồm:* 64-byte cache line, struct padding for alignment, false sharing between PMD threads, RTE_CACHE_LINE_SIZE macro usage.

## 16.1.5 Multi-queue + RSS scaling

*Skeleton — nội dung sẽ bao gồm:* Receive Side Scaling (RSS) hash symmetric, multi-queue NIC (Intel E810 256 queues), queue-to-CPU pinning strategy, flow director for specific flow steering.
