## Dàn bài

## 8.2.1 VLAN stacking: single-tagged, QinQ, triple-tagged

Single-tagged 802.1Q (1998): 4-byte VLAN tag chèn sau MAC, ID 12-bit = 4096 VLAN. QinQ 802.1ad (2011): S-VLAN + C-VLAN stacked — service provider dùng để transport customer VLAN trong their own VLAN space. Triple-tagged: deployment niche carrier, 3 tag lồng nhau cho MEF service hierarchy.

## 8.2.2 Bonding: 7 modes

Mode 0 `balance-rr`: round-robin packet giữa slaves, cần switch hỗ trợ (static LAG).
Mode 1 `active-backup`: một slave active, còn lại standby.
Mode 2 `balance-xor`: hash (MAC src XOR MAC dst) modulo slave count.
Mode 3 `broadcast`: replicate mọi packet ra mọi slave.
Mode 4 `802.3ad` (LACP): standard dynamic aggregation, yêu cầu peer switch LACP-capable.
Mode 5 `balance-tlb`: transmit load balancing, receive dedicated.
Mode 6 `balance-alb`: adaptive LB transmit + ARP-based receive rebalance.

## 8.2.3 LACP specifics

IEEE 802.3ad LACPDU exchange: actor (our side) + partner (peer) state negotiate. Timer: fast (1 second) vs slow (30 second) — fast-timer giảm failover time xuống sub-100ms, tradeoff CPU overhead. Switch-side config: peer switch phải configure LACP port-channel tương ứng, cùng port list + LACP mode.

## 8.2.4 Team daemon: tại sao được deprecate

Team (`libteam`) ra đời 2012 thay thế bonding với userspace daemon + D-Bus API + extensible runner (activebackup, roundrobin, loadbalance, random). RHEL 7-8 ship team như alternative. RHEL 9 (2022): team deprecated, khuyến cáo dùng bonding. Lý do: bonding có tighter kernel integration, team daemon crash có thể disrupt networking, bonding không phụ thuộc userspace process.
