## 10.2.1 Database file append-only

Compass Ch M: "The DB file at `/etc/openvswitch/conf.db` is append-only: every transaction is logged."

Hệ quả:

- File tăng kích thước theo thời gian dù data state không tăng — mọi change (insert, update, delete) đều append vào log.
- Sau thời gian dài, file có thể lớn hơn data state thật sự hàng trăm lần.
- Compact cần thiết — rewrite file thành snapshot của current state.

---

## 10.2.2 Compact operations

**Offline compact** (cần stop ovsdb-server):

```bash
[host]# systemctl stop openvswitch-switch
[host]# ovsdb-tool compact /etc/openvswitch/conf.db
[host]# systemctl start openvswitch-switch
```

**Live compact** (compass Ch M khuyến nghị):

```bash
[host]# ovs-appctl -t ovsdb-server ovsdb-server/compact
```

Live compact atomic, client không thấy disruption. Nên chạy cron định kỳ (weekly hoặc khi file vượt threshold).

Kiểm tra kích thước:

```bash
[host]# du -h /etc/openvswitch/conf.db
[host]# ovsdb-tool show-log /etc/openvswitch/conf.db | wc -l
```

## 10.2.3 Backup và restore

**Backup**:

```bash
[host]# ovsdb-client backup > /tmp/conf.db.bak
```

Output là valid OVSDB database file với current state.

**Restore**:

```bash
[host]# ovsdb-client restore < /tmp/conf.db.bak
```

**Caveat quan trọng** (compass Ch M): restore KHÔNG preserve UUID của row gốc. Nếu client đang reference row qua UUID (OVN đang chạy), restore sẽ break reference — phải stop OVN, restore, restart.

Production strategy:

- Daily backup qua cron, store off-host.
- Retention 30-90 ngày cho rollback.
- Test restore trên staging trước khi rely vào backup.

## 10.2.4 Schema upgrade

Check version:

```bash
[host]# ovsdb-tool db-version /etc/openvswitch/conf.db
8.3.0
[host]# ovsdb-tool schema-version /usr/share/openvswitch/vswitch.ovsschema
8.4.0
```

Upgrade DB to match binary schema:

```bash
[host]# systemctl stop openvswitch-switch
[host]# ovsdb-tool needs-conversion /etc/openvswitch/conf.db \
         /usr/share/openvswitch/vswitch.ovsschema
yes
[host]# ovsdb-tool convert /etc/openvswitch/conf.db \
         /usr/share/openvswitch/vswitch.ovsschema
[host]# systemctl start openvswitch-switch
```

Xem transaction history:

```bash
[host]# ovsdb-tool show-log /etc/openvswitch/conf.db
```

## 10.2.5 RBAC (compass Ch O)

OVSDB RBAC (Role-Based Access Control) — giới thiệu OVS 2.8 — restrict client truy cập table/column theo role.

Cấu hình role trên Manager:

```bash
[host]# ovs-vsctl set Manager ssl:10.0.0.10:6640 role=ovn-controller
```

Role `ovn-controller` trong OVN schema — chỉ cho phép update `Chassis` row của chính nó và `Port_Binding` row thuộc về nó. Không cho phép ghi vào `Logical_Flow`, `Logical_Switch`, hay config của chassis khác.

RBAC permission model (OVN):

```json
{
  "name": "ovn-controller",
  "permissions": {
    "Chassis": {"authorization": ["name"], "insert_delete": true},
    "Port_Binding": {"authorization": ["chassis"], "update": true}
  }
}
```

Verify: `ovsdb-client` từ restricted principal cố ghi ngoài role → fail permission error.

Use case: multi-tenant cloud với nhiều chassis — mỗi chassis authenticate bằng TLS cert với CN tương ứng tên chassis, RBAC đảm bảo không thể tamper dữ liệu chassis khác.

## 10.2.6 Operational checklist

Daily:

```bash
[cron]# ovsdb-client backup > /backup/conf.db.$(date +%F).bak
```

Weekly:

```bash
[cron]# ovs-appctl -t ovsdb-server ovsdb-server/compact
```

Trước upgrade:

```bash
[admin]# ovsdb-client backup > /tmp/pre-upgrade.bak
[admin]# ovsdb-tool needs-conversion /etc/openvswitch/conf.db <new schema>
```

Monitoring alert nếu file > 1GB:

```bash
[monitor]$ du -b /etc/openvswitch/conf.db | awk '$1 > 1073741824 {print "too big"}'
```

## Điểm cốt lõi cần nhớ

Thứ nhất, OVSDB append-only — file tăng vô hạn. Compact định kỳ (weekly cron) để thu gọn.

Thứ hai, live compact là cách chuẩn production — không stop service.

Thứ ba, RBAC (OVS 2.8+) là foundation security cho multi-tenant OVN. Mỗi chassis authenticate TLS + role restricted.

Thứ tư, backup + restore không preserve UUID — caveat quan trọng. Coordinate với control plane restart khi restore.
