## Dàn bài

## 12.2.1 Micro-segmentation concept

Traditional perimeter firewall (north-south only) không defend east-west traffic — workload compromise spread freely. Micro-segmentation đặt firewall policy tại workload boundary (mỗi VM/container có policy riêng), minimize blast radius. Zero-trust principle: "never trust, always verify" applied to every connection kể cả intra-DC.

## 12.2.2 NSX Distributed Firewall example

NSX DFW implement enforcement tại hypervisor vSwitch — per-vNIC policy. Không đi qua centralized firewall appliance — throughput scale với compute capacity. Stateful inspection tại hypervisor level với kernel conntrack (ESXi VMKernel hoặc Linux netfilter). Rule set push từ NSX Manager qua NSX-T API.

## 12.2.3 OVN ACL + Port_Group (forward reference Block XIII)

OVN ACL match expression dùng OpenFlow-like syntax: `eth.src == 00:00:00:00:00:01 && ip4.dst == 10.0.0.0/24 && tcp.dst == 22`. Port_Group cho reuse: group logical ports lại thành named set, ACL reference group thay vì list port individually. Stateful default (dùng conntrack) vs stateless (stateless=true). Xem chi tiết Part 13.3.

## 12.2.4 Service function chaining RFC 7665

RFC 7665 (October 2015) định nghĩa SFC architecture: classifier (phân loại traffic vào Service Function Path), SFF (service function forwarder) route giữa SF instance, re-classification sau mỗi SF nếu cần. Use case: traffic từ web tier → WAF → IPS → firewall → origin server theo thứ tự xác định.

## 12.2.5 NSH (Network Service Header)

RFC 8300 (January 2018) NSH format: 8-byte base header + optional context metadata. Service Path Identifier (SPI) 24-bit + Service Index (SI) 8-bit identify next SF. Metadata passing giữa SF mà không cần từng SF tự parse packet. Encapsulation: NSH-over-Geneve, NSH-over-VXLAN-GPE, NSH-over-MPLS. OVS support NSH từ 2.9.
