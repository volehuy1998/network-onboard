## Dàn bài

## 7.2.1 ONOS origin: ON.Lab + AT&T, 2014

ON.Lab (Open Networking Laboratory) founded 2012 bởi Nick McKeown + Guru Parulkar tại Stanford + UC Berkeley. AT&T chính là major funder cho ONOS phát triển. Avocet first release December 2014. Goal: build controller chịu được scale service provider, khác với research-focus của Ryu hoặc umbrella của ODL.

## 7.2.2 Architecture: distributed core, Atomix consensus

Core ONOS dựa trên Atomix library (Raft-based distributed primitives) cho cluster. Multi-master — mọi node trong cluster có thể handle write, eventual consistency cho non-critical state. In-memory distributed primitives: AtomicMap, AtomicCounter, LeaderElection. Kiến trúc này cho phép horizontal scale mà không bottleneck single master.

## 7.2.3 Scale focus: 1M flows, 100ms control loop

Performance target: 1 triệu flow entries per controller cluster, 100ms control loop (time từ event → flow installed). Multi-region deployment pattern: controller cluster trải ra nhiều region, eventual consistency qua WAN, local reads low-latency. AT&T CORD (Central Office Re-architected as Datacenter) là reference deployment.

## 7.2.4 Trellis SD-Fabric stack

Trellis = ONOS + whitebox leaf-spine hardware (Broadcom Tomahawk) + OFDPA pipeline. Stack đầy đủ cho DC deployment không cần vendor controller. Production users: Comcast, Telefonica, một số European ISP. Features: underlay BGP, overlay VXLAN, multi-tenancy, ACL. Competitor trực tiếp của Cisco ACI ở tầng open source.

## 7.2.5 Current state: ONF-merged 2018

ON.Lab merged với ONF (Open Networking Foundation) tháng 10/2018. ONOS + CORD + SD-RAN (5G) consolidated dưới ONF umbrella. Governance shift: OpenFlow/P4/Stratum runtime coexist. Roadmap 2024-2026: 5G UPF virtualization, P4Runtime deeper integration. Vẫn active development (khác ODL maintenance-only).
