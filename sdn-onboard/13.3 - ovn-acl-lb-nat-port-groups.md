## 13.3.1 ACL

OVN ACL stateful — tracking connection state qua `ct()` action automatic.

```bash
ovn-nbctl acl-add <ls-or-pg> <direction> <priority> <match> <action>
```

- `<direction>`: `from-lport` (ingress từ VM) hoặc `to-lport` (egress về VM).
- `<priority>`: 0-32767. Higher wins.
- `<action>`: `allow`, `allow-related`, `drop`, `reject`.

`allow-related` = allow + auto-allow reply qua conntrack.

```bash
[admin]# ovn-nbctl acl-add ls-tenant1 from-lport 1000 \
         "ip4.src == 10.0.0.0/24 && tcp.dst == 80" allow-related
[admin]# ovn-nbctl acl-add ls-tenant1 from-lport 500 "1" drop
```

## 13.3.2 Port_Group

Port_Group nhóm LSP cho ACL apply chung:

```bash
[admin]# ovn-nbctl pg-add pg-web web-vm1 web-vm2 web-vm3
[admin]# ovn-nbctl acl-add pg-web from-lport 1000 \
         "outport == @pg-app && tcp.dst == 8080" allow-related
```

Shortcut `@pg-web` reference trong match expression.

## 13.3.3 Load Balancer

OVN L4 LB:

```bash
[admin]# ovn-nbctl lb-add lb1 10.0.100.100:80 10.0.1.1:8080,10.0.1.2:8080
[admin]# ovn-nbctl ls-lb-add ls-tenant1 lb1
```

- VIP: 10.0.100.100:80.
- Backends: 10.0.1.1:8080, 10.0.1.2:8080.

Hỗ trợ health check (OVN 22.03+). Algorithm: hash dựa 5-tuple inner — consistent hashing cho session persistence.

## 13.3.4 NAT

LR hỗ trợ 3 loại:

**SNAT** (source NAT):

```bash
[admin]# ovn-nbctl lr-nat-add lr-tenant1 snat 192.0.2.100 10.0.0.0/24
```

**DNAT_and_SNAT** (floating IP 1-to-1):

```bash
[admin]# ovn-nbctl lr-nat-add lr-tenant1 dnat_and_snat 192.0.2.101 10.0.1.10
```

**DNAT**: thường combine thành dnat_and_snat.

---

## 13.3.5 Gateway router

NAT cần gateway chassis:

```bash
[admin]# ovn-nbctl lrp-set-gateway-chassis lrp-external chassis-1 100
[admin]# ovn-nbctl lrp-set-gateway-chassis lrp-external chassis-2 50
```

Chassis-1 priority 100, chassis-2 priority 50 (failover). Active chassis handle all north-south của LR.

## 13.3.6 Workflow 3-tier ACL

```bash
# Port groups
[admin]# ovn-nbctl pg-add pg-web web-vm1 web-vm2
[admin]# ovn-nbctl pg-add pg-app app-vm1 app-vm2
[admin]# ovn-nbctl pg-add pg-db db-vm1

# Allow specific
[admin]# ovn-nbctl acl-add pg-app from-lport 1000 \
         "outport == @pg-app && tcp.dst == 8080" allow-related
[admin]# ovn-nbctl acl-add pg-db from-lport 1000 \
         "outport == @pg-db && tcp.dst == 5432" allow-related

# Default deny
[admin]# ovn-nbctl acl-add pg-web from-lport 1 "ip" drop
[admin]# ovn-nbctl acl-add pg-app from-lport 1 "ip" drop
[admin]# ovn-nbctl acl-add pg-db from-lport 1 "ip" drop
```

## Điểm cốt lõi cần nhớ

Thứ nhất, ACL stateful tự động qua `allow-related` — conntrack handle reply.

Thứ hai, Port_Group là foundation micro-seg scale. Apply ACL per-group.

Thứ ba, Load_Balancer OVN L4-only. Cho L7, dùng external (HAProxy).

Thứ tư, NAT cần gateway chassis — chokepoint north-south. Scale limited.
