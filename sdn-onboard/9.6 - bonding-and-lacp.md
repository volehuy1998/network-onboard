## 9.6.1 OVS bonding khác Linux kernel bonding

OVS implement bond driver độc lập với kernel `bonding` module. Lý do: OVS cần bond state tích hợp với OpenFlow pipeline + OVSDB transaction, không thể dựa vào opaque kernel abstraction.

Hệ quả: trên cùng host, có thể dùng OVS bond cho uplink br-int đồng thời với kernel bond cho management interface — hai stack không conflict.

---

## 9.6.2 Ba bond mode

**`active-backup`**: một member active, khác standby. Không yêu cầu peer switch support đặc biệt. Failover dựa vào carrier state + optional `bond_updelay`/`bond_downdelay`.

**`balance-slb`** (source-MAC load balancing): hash theo source MAC, rebalance định kỳ (`bond-rebalance-interval` ms). KHÔNG yêu cầu peer cooperation. Trade-off: một source MAC pin vào một member, không tận dụng nhiều link cho một flow.

**`balance-tcp`**: L2+L3+L4 hash. YÊU CẦU LACP + MLAG-capable peer (Cisco vPC, Arista MLAG, Juniper ESI). Phức tạp nhưng tận dụng bandwidth đầy đủ.

Tạo bond (compass Ch E):

```bash
[host]# ovs-vsctl add-bond br0 bond0 eth1 eth2 \
         bond_mode=balance-slb \
         lacp=active other_config:lacp-time=fast \
         bond_updelay=100 bond_downdelay=100
```

---

## 9.6.3 LACP options

- `lacp=active`: gửi LACPDU proactively (1s fast-timer, 30s slow-timer).
- `lacp=passive`: chỉ reply khi nhận LACPDU.
- `lacp=off`: tắt LACP (dùng với balance-slb hoặc active-backup).

`other_config:lacp-time=fast` — giảm LACPDU interval từ 30s xuống 1s, failover sub-100ms thay vì sub-90s.

`other_config:lacp-fallback-ab=true` — nếu LACP negotiation fail (peer không respond trong 90s), tự động fallback sang active-backup. Foot-gun-saver quan trọng khi peer switch đang upgrading.

---

## 9.6.4 Verification: bond/show và lacp/show

```bash
[host]# ovs-appctl bond/show bond0
[host]# ovs-appctl lacp/show bond0
```

`bond/show` output: `bond_mode`, `lacp_status`, active slave, `may_enable` state mỗi member, rebalance countdown.

`lacp/show` output: Actor + Partner system_id, key, priority, state bits. State bit chuẩn theo IEEE 802.1AX:

- `A`ctivity — Actor active LACP
- `C`ollecting — collecting frames
- `E`xpired — state expired
- `G`ood (synchronized)
- `I`n sync với peer
- `K`ey matched
- `N`ew connection
- `P`assive mode

"Letter salad `ACEGIKNP` trên cả hai phía" = healthy bundled link (compass Ch E).

## 9.6.5 Failover test

```bash
[host]# ip link set eth1 down          # bring down one member
[host]# ping -i 0.1 <remote>            # continuous ping
[host]# ovs-appctl bond/show bond0     # observe failover
```

Expect: active-backup sub-second reconvergence, balance-tcp với fast-timer sub-100ms.

### ▶ Guided Exercise 7: Bond + LACP với failover test

**Outcomes:** Tạo OVS bond 2 interface, enable LACP fast-timer + fallback-ab, failover test.

**Before You Begin:** Host có 2 interface eth1, eth2 link up (có thể dummy hoặc veth pair).

**Instructions:**

**1.** Tạo bond:

```bash
[host]# ovs-vsctl add-br br-bond
[host]# ovs-vsctl add-bond br-bond bond0 eth1 eth2 \
         bond_mode=balance-slb lacp=active \
         other_config:lacp-time=fast \
         other_config:lacp-fallback-ab=true
```

**2.** Verify:

```bash
[host]# ovs-appctl bond/show bond0
[host]# ovs-appctl lacp/show bond0
```

**3.** Failover — bring down eth1:

```bash
[host]# ip link set eth1 down
[host]# sleep 1
[host]# ovs-appctl bond/show bond0    # eth1 status down, eth2 active
```

**4.** Cleanup:

```bash
[host]# ovs-vsctl del-br br-bond
[host]# ip link set eth1 up
```

## Điểm cốt lõi cần nhớ

Thứ nhất, OVS bonding độc lập với Linux kernel bonding — hai stack cùng tồn tại trên một host, không conflict.

Thứ hai, ba bond mode phục vụ ba scenario: `active-backup` (simple, không cần peer support), `balance-slb` (load share per source MAC, không cần MLAG), `balance-tcp` (best throughput, cần LACP + MLAG peer).

Thứ ba, `lacp-fallback-ab=true` là foot-gun-saver bắt buộc cho production — tránh bond down hoàn toàn khi peer LACP misconfigured.

Thứ tư, "letter salad `ACEGIKNP` trên cả hai phía" là quick check healthy LACP. Thiếu letter nào → debug theo letter đó.
