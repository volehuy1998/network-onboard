### ▶ Guided Exercise 1: Compile + run L2 forwarding P4 program trên BMv2

> **Trạng thái lab verification:** Output dưới là kỳ vọng dựa trên P4 tutorials + BMv2 docs; chờ verify trên Ubuntu 22.04 thực.

**Mục đích:** Trải nghiệm end-to-end P4 dev workflow — viết P4 program, compile, deploy vào software switch, cài flow entry runtime, verify forwarding. Đây là bài học cơ bản nhất để hiểu P4 như "ngôn ngữ" thay cho OpenFlow như "giao thức cố định".

**Chuẩn bị:**
- Ubuntu 22.04 LTS VM, 4 GB RAM, 10 GB disk (Lab mode A single-node)
- Install p4c + BMv2:
  ```bash
  sudo apt install -y p4lang-p4c p4lang-bmv2
  ```
  Nếu package không có trong apt, build từ source: `git clone --recursive https://github.com/p4lang/p4c.git` + `cmake .. && make -j4 && sudo make install`.
- Verify: `p4c --version` và `simple_switch --version` đều trả về số version (expected `p4c 1.2.4+` và `BMv2 simple_switch 1.15+`).

**Mô hình lab (topology):**

```
┌────────────────────────────────────────────────┐
│          Ubuntu 22.04 VM                       │
│                                                │
│   ┌─────────────────────────────────────┐      │
│   │      BMv2 simple_switch             │      │
│   │                                     │      │
│   │  port 0 ◄─── basic.p4 ──► port 1    │      │
│   │            (L2 exact fwd)           │      │
│   └─────────┬─────────────────┬─────────┘      │
│             │                 │                │
│          veth-0            veth-1              │
│             │                 │                │
│  ┌──────────▼──┐       ┌──────▼──────┐         │
│  │ netns h1    │       │ netns h2    │         │

│  │ MAC         │       │ MAC         │         │
│  │ 00:...:01   │       │ 00:...:02   │         │
│  └─────────────┘       └─────────────┘         │
└────────────────────────────────────────────────┘
```

**Bước thực hiện:**

**1.** Tạo P4 program `basic.p4` (L2 exact match forwarding):

```p4
#include <core.p4>
#include <v1model.p4>

header ethernet_t {
    bit<48> dstAddr;
    bit<48> srcAddr;
    bit<16> etherType;
}

struct headers { ethernet_t ethernet; }
struct metadata { }

parser MyParser(packet_in pkt, out headers hdr, inout metadata meta, inout standard_metadata_t std) {
    state start { pkt.extract(hdr.ethernet); transition accept; }
}

control MyIngress(inout headers hdr, inout metadata meta, inout standard_metadata_t std) {
    action forward(bit<9> port) { std.egress_spec = port; }
    action drop() { mark_to_drop(std); }
    table l2_fwd {
        key = { hdr.ethernet.dstAddr: exact; }
        actions = { forward; drop; }
        default_action = drop();
    }
    apply { l2_fwd.apply(); }
}

control MyVerifyChecksum(inout headers hdr, inout metadata meta) { apply {} }
control MyComputeChecksum(inout headers hdr, inout metadata meta) { apply {} }
control MyEgress(inout headers hdr, inout metadata meta, inout standard_metadata_t std) { apply {} }
control MyDeparser(packet_out pkt, in headers hdr) { apply { pkt.emit(hdr.ethernet); } }

V1Switch(MyParser(), MyVerifyChecksum(), MyIngress(),
         MyEgress(), MyComputeChecksum(), MyDeparser()) main;
```

**2.** Compile P4 → BMv2 JSON:

```bash
[host]$ p4c --target bmv2 --arch v1model basic.p4
```

Output kỳ vọng: tạo `basic.json` và `basic.p4info.txt` trong thư mục hiện tại, không có error.

**3.** Tạo 2 netns + veth pair:

```bash
[host]# ip netns add h1
[host]# ip netns add h2
[host]# ip link add veth-0 type veth peer name h1-eth0
[host]# ip link add veth-1 type veth peer name h2-eth0
[host]# ip link set h1-eth0 netns h1
[host]# ip link set h2-eth0 netns h2
[host]# ip netns exec h1 ip link set h1-eth0 address 00:00:00:00:00:01
[host]# ip netns exec h2 ip link set h2-eth0 address 00:00:00:00:00:02
[host]# ip netns exec h1 ip addr add 10.0.0.1/24 dev h1-eth0
[host]# ip netns exec h2 ip addr add 10.0.0.2/24 dev h2-eth0
[host]# ip netns exec h1 ip link set h1-eth0 up
[host]# ip netns exec h2 ip link set h2-eth0 up
[host]# ip link set veth-0 up
[host]# ip link set veth-1 up
```

**4.** Chạy simple_switch với basic.json:

```bash
[host]# simple_switch -i 0@veth-0 -i 1@veth-1 basic.json &
```

Output kỳ vọng:

```
Calling target program-options parser
Adding interface veth-0 as port 0
Adding interface veth-1 as port 1
```

**5.** Cài flow entry qua simple_switch_CLI:

```bash
[host]# simple_switch_CLI --thrift-port 9090 << 'EOF'
table_add MyIngress.l2_fwd MyIngress.forward 00:00:00:00:00:02 => 1
table_add MyIngress.l2_fwd MyIngress.forward 00:00:00:00:00:01 => 0
EOF
```

Output kỳ vọng:

```

match key:           EXACT-00:00:00:00:00:02
action:              MyIngress.forward
runtime data:        00:01

RuntimeCmd: Adding entry to exact match table MyIngress.l2_fwd
match key:           EXACT-00:00:00:00:00:01
action:              MyIngress.forward
runtime data:        00:00
Entry has been added with handle 1
```

**6.** Test forwarding từ h1 → h2:

```bash
[host]# ip netns exec h1 arping -c 3 -I h1-eth0 10.0.0.2
```

Output kỳ vọng:

```
ARPING 10.0.0.2 from 10.0.0.1 h1-eth0
Unicast reply from 10.0.0.2 [00:00:00:00:00:02]  0.456ms
Unicast reply from 10.0.0.2 [00:00:00:00:00:02]  0.321ms
Unicast reply from 10.0.0.2 [00:00:00:00:00:02]  0.289ms
Sent 3 probes (1 broadcast, 2 unicast)
Received 3 response(s)
```

**Bài học học được:**

Thứ nhất, P4 tách biệt rõ ràng giữa **data plane behavior** (định nghĩa trong P4 program: parser + tables + actions + deparser) và **control plane runtime** (table entries được cài qua thrift/P4Runtime API). Đây là khác biệt cốt lõi so với OpenFlow, nơi match field và action đã fixed trong spec.

Thứ hai, software simulator (BMv2) có `simple_switch` behavioral model chạy packet qua pipeline chính xác như P4 program định nghĩa. Tốc độ chỉ khoảng 1-5 Gbps, không phải line-rate production, nhưng đủ để verify logic trước khi compile cho Tofino hardware.

Thứ ba, `simple_switch_CLI` là thrift-based interface riêng của BMv2, không phải chuẩn chung. Production P4 deployment dùng P4Runtime (gRPC + protobuf) — xem Part 14.2.

**Cleanup:**

```bash
[host]# pkill -f simple_switch
[host]# ip netns del h1
[host]# ip netns del h2
[host]# ip link del veth-0 2>/dev/null || true
[host]# ip link del veth-1 2>/dev/null || true
```

### ▶ Lab 2: P4 L3 router với LPM table

> **Trạng thái lab verification:** Spec dựa trên P4 tutorials `basic_routing` exercise; chờ verify lab thực.

**Mục đích:** Mở rộng Ex1 từ L2 exact match lên L3 LPM (Longest Prefix Match) routing — TTL decrement, source/dest MAC rewrite, multi-subnet forwarding. Thực hành implement một IPv4 router đơn giản bằng P4.

**Chuẩn bị:**
- Hoàn thành Lab 14.0-Ex1 baseline (p4c + BMv2 installed).
- Clone P4 tutorials: `git clone https://github.com/p4lang/tutorials.git` (reference implementation trong `exercises/basic_routing/`).

**Mô hình lab:**

```
┌──────────────────────────────────────────────────────────┐
│                  Ubuntu 22.04 VM                         │
│                                                          │
│   h1 (10.1.0.1/24)           h2 (10.2.0.1/24)            │
│    MAC 00:00:0a:01:00:01      MAC 00:00:0a:02:00:01      │
│         │                         │                      │
│       veth-0                    veth-1                   │
│         │                         │                      │
│   ┌─────▼─────────────────────────▼─────┐                │
│   │        BMv2 simple_switch            │                │
│   │                                      │                │
│   │   router.p4:                         │                │
│   │   - LPM on dst IP                    │                │
│   │   - TTL-- + checksum update          │                │
│   │   - MAC rewrite src/dst              │                │
│   │                                      │                │
│   │  Router MAC 00:00:00:01:01:01        │                │
│   └──────────────────────────────────────┘                │
└──────────────────────────────────────────────────────────┘
```

**Mục tiêu self-guided (không có step-by-step như Ex1):**

**1.** Đọc `exercises/basic_routing/basic.p4` trong P4 tutorials, hiểu LPM table definition và action `ipv4_forward(macAddr_t dstAddr, egressSpec_t port)`.

**2.** Tạo topology 2-host với IP khác subnet. Gán default gateway cho mỗi host trỏ về router BMv2 IP.

**3.** Compile + deploy. Cài LPM entries cho 10.1.0.0/24 và 10.2.0.0/24.

**4.** Test: `ip netns exec h1 ping -c 3 10.2.0.1`. Phải thành công.

**5.** Verify TTL decrement: từ h1 chạy `tcpdump -nvv -i h1-eth0 icmp`. TTL khi gửi = 64, khi nhận reply = 63 (passed 1 hop = router).

**Output mong muốn:**

```
# Ping output
PING 10.2.0.1 (10.2.0.1) 56(84) bytes of data.
64 bytes from 10.2.0.1: icmp_seq=1 ttl=63 time=0.512 ms
64 bytes from 10.2.0.1: icmp_seq=2 ttl=63 time=0.421 ms
64 bytes from 10.2.0.1: icmp_seq=3 ttl=63 time=0.398 ms

--- 10.2.0.1 ping statistics ---
3 packets transmitted, 3 received, 0% packet loss
```

Quan trọng: `ttl=63` chứng minh packet đã qua router (TTL giảm từ 64 → 63).

**Bài học học được:**

Lab này thể hiện P4's programmability ở mức IPv4 header manipulation — thứ mà OpenFlow 1.0 không làm được native (OpenFlow 1.0 không có SET_NW_TTL action). P4 cho phép programmer chèn action tùy ý vào pipeline: TTL--, checksum recomputation, MAC rewrite, tất cả trong cùng một table apply. Đây là minh chứng **target-independent programmability** — cùng P4 program chạy được trên BMv2, Tofino, eBPF backend.

**Cleanup:** Như Ex1.
