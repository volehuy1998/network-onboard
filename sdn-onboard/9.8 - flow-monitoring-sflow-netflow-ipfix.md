## Dàn bài

## 9.8.1 Ba protocol, một idiom

sFlow (RFC 3176 informational): gửi mẫu packet 1-in-N + periodic counter poll. Header mặc định 128 byte đầu của packet. Collector nhận UDP port 6343.

NetFlow v5/v9 (Cisco origin, v9 RFC 3954): tạo flow records trên active-timeout. Không gửi packet payload, chỉ metadata (5-tuple + byte/packet counter). Collector nhận UDP port 2055 thường dùng, nhưng configurable.

IPFIX (RFC 7011): kế thừa NetFlow v9 dưới dạng IETF standard, thêm template negotiation (client announce schema cho collector). Collector UDP port 4739 mặc định.

## 9.8.2 sFlow: atomic idiom

```
ovs-vsctl -- --id=@s create sFlow agent=eth1 \
 target=\"10.0.0.1:6343\" \
 header=128 sampling=64 polling=10 \
 -- set Bridge br0 sflow=@s
```

`sampling=64`: 1 trong mỗi 64 packet sample (trade-off: thấp hơn = accuracy cao hơn + CPU load cao hơn). `polling=10`: counter poll mỗi 10s.

## 9.8.3 NetFlow atomic idiom

```
ovs-vsctl -- set Bridge br0 netflow=@nf \
 -- --id=@nf create NetFlow targets=\"192.168.0.34:5566\" \
 active-timeout=30
```

`active-timeout=30`: export flow record sau 30s từ packet đầu (kể cả flow chưa kết thúc).

## 9.8.4 IPFIX atomic idiom

```
ovs-vsctl -- set Bridge br0 ipfix=@i \
 -- --id=@i create IPFIX targets=\"192.168.0.34:4739\" \
 obs_domain_id=123 obs_point_id=456 \
 cache_active_timeout=60 cache_max_flows=13
```

`obs_domain_id` / `obs_point_id`: identifier cho collector phân biệt nhiều OVS instance. `cache_max_flows`: giới hạn flow cache internal (default 4096).

## 9.8.5 Verification: tcpdump trên collector

`tcpdump -ni ethX udp port 6343` trên collector để xác nhận agent đang gửi. Rotate target atomically: `ovs-vsctl set sFlow <uuid> target='"new:6343"'`, không drop packet.

## 9.8.6 Open-source collector stack

sFlow: `sflowtool`, `pmacctd`, `ntopng`, `goflow2` (Cloudflare, Go-based, Kafka-integrated).

NetFlow/IPFIX: `nfdump` + `nfcapd`, `goflow2`, `vflow`, Elastic Beats flow module.
