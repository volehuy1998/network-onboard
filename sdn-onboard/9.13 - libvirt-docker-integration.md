## Dàn bài

## 9.13.1 libvirt + OVS: XML `<virtualport type='openvswitch'/>`

Trong VM domain XML:

```
<interface type='bridge'>
  <source bridge='br-int'/>
  <virtualport type='openvswitch'/>
  <model type='virtio'/>
</interface>
```

Khi VM start, libvirt tự:
1. Tạo tap interface `vnetN`.
2. `ovs-vsctl add-port br-int vnetN -- set Interface vnetN external_ids:attached-mac=<vm-mac> external_ids:iface-id=<uuid>`.
3. Set `iface-status=active`.

Khi VM stop, libvirt tự del-port.

## 9.13.2 external_ids:iface-id — contract với orchestrator

Key convention universal: orchestrator (OVN-Kubernetes, OVN standalone script, hoặc OpenStack với chế độ khác curriculum này) set `Interface.external_ids:iface-id=<logical-port-name>`. OVN's `ovn-controller` poll OVSDB, match iface-id với `Logical_Switch_Port.name` trong NBDB, trigger bind port cho chassis hiện tại.

Không có iface-id = port không được OVN claim = traffic không forward through logical pipeline.

## 9.13.3 libvirt troubleshoot: `ovs-vsctl list Interface | grep -A2 vnet`

Debug port attachment khi VM start fail silently:
```
ovs-vsctl list Interface | grep -A5 vnet
```
Check column `error` (nếu nonempty = lỗi kernel), `ofport` (nếu = -1 = chưa attach), `admin_state`/`link_state`.

## 9.13.4 Docker + OVS: ovs-docker helper

`ovs-docker` script ship với ovs-common package — wrapper quanh `ip netns` + `veth` + `ovs-vsctl add-port`:

```
ovs-docker add-port br-int eth0 <container-id> --ipaddress=10.0.0.5/24
ovs-docker set-vlan br-int eth0 <container-id> 100
ovs-docker del-port br-int eth0 <container-id>
```

## 9.13.5 Manual container attach (production-grade)

Production pattern không dùng helper — tự setup cho control:

```
CID=<container-id>
NS=/var/run/netns/$CID
PID=$(docker inspect -f '{{.State.Pid}}' $CID)
ln -s /proc/$PID/ns/net $NS
ip link add veth0 type veth peer name ovsport
ip link set ovsport up
ovs-vsctl add-port br-int ovsport \
  -- set Interface ovsport external_ids:container_id=$CID \
                            external_ids:iface-id=<logical-port-name>
ip link set veth0 netns $CID
ip netns exec $CID ip link set veth0 name eth0
ip netns exec $CID ip addr add 10.0.0.5/24 dev eth0
ip netns exec $CID ip link set eth0 up
```

Giữ control + label external_ids với container_id để garbage-collect sau.

## 9.13.6 CNI plugin pattern (Kubernetes context)

CNI (Container Network Interface) spec: plugin nhận Pod info từ kubelet qua stdin JSON, setup namespace network, return JSON config. OVN-Kubernetes CNI plugin (trong scope curriculum — OVN + K8s là valid combo) wrap pattern 9.13.5 với bookkeeping trong annotation.
