## Dàn bài

## 4.2.1 OF 1.3 release timeline: 1.3.0 (25/04/2012) → 1.3.5 (26/03/2015)

Giới thiệu chuỗi release 1.3.x: 1.3.0 (25/04/2012), 1.3.1 (06/09/2012), 1.3.2 (05/04/2013), 1.3.3 (27/09/2013), 1.3.4 (14/08/2014), 1.3.5 (26/03/2015). Nêu ý nghĩa errata chain: không có breaking change giữa 1.3.x, mỗi release chỉ fix ambiguity và clarification, giữ backward wire compat. Đây là điểm then chốt khiến vendor chọn 1.3 làm baseline long-term: implement một lần, upgrade errata không cần rewrite.

## 4.2.2 Meter table: per-flow QoS rate limiting

Mô tả meter table theo spec 1.3 §5.7: mỗi meter có ID + list of bands; instruction OFPIT_METER đưa packet qua meter trước khi forward. Nêu hai meter band type: **DROP** (drop packet nếu rate vượt threshold), **DSCP_REMARK** (re-mark DSCP xuống precedence thấp hơn). Phân tích token bucket semantic: rate (KB/s hoặc packets/s) + burst size; packet consume tokens, khi bucket cạn → trigger band action. Cross-ref forward đến Part 9.9 (OVS QoS policing/shaping/metering với `linux-htb` recipe) nơi phân tích meter-based policing trong thực hành.

## 4.2.3 Per-connection auxiliary channels (multi-socket)

Mô tả feature mới §6.3.7: switch có thể mở thêm **auxiliary connection** song song với main connection đến cùng controller, dùng cho high-volume message type (PACKET_IN). Nêu motivation: main connection xử lý sequential (FIFO) → head-of-line blocking khi controller chậm reply OFPT_PACKET_OUT; auxiliary connection là UDP hoặc TLS stream riêng cho phép parallel packet_in flow. Thực tế deployment: ít vendor implement vì phức tạp và OVS không support, feature này chủ yếu on-paper.

## 4.2.4 PBB (802.1ah) match + set

Liệt kê PBB fields mới: PBB_ISID (24-bit I-SID, service instance identifier), PBB_UCA (Use Customer Address bit). Nêu context: IEEE 802.1ah "Provider Backbone Bridges" (2008) giải bài toán scale VLAN 4096 cho carrier Ethernet bằng MAC-in-MAC encapsulation. Phân tích use case: 1.3 cho phép OpenFlow switch route traffic trong MEF-certified carrier Ethernet network → mở rộng scope từ campus/DC sang provider edge.

## 4.2.5 Tại sao 1.3 là LTS: vendor alignment, OVS support, Ryu/ODL baseline

Phân tích hệ sinh thái chọn 1.3 làm chuẩn baseline: OVS 2.0+ (2013) hỗ trợ 1.3 feature-complete trong kernel datapath; Ryu Framework mặc định dạy 1.3 trong tutorial; OpenDaylight từ Helium (10/2014) coi 1.3 là default OF version; NEC/HP/Centec/Pica8 hardware commit 1.3 compliance. Nêu data point phản diện: 1.4/1.5 adoption near-zero trên hardware silicon, hai version này chủ yếu là academic exploration (chi tiết 4.3.5 và 4.4.5).
