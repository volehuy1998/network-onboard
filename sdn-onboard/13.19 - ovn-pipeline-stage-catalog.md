## §13.19.1 LS_IN_PORT_SEC_L2 (table 0, cornerstone C6) DEEP-20

**Axis 1 concept.** First stage LS ingress pipeline. Drop packet vi phạm port_security tại L2 layer (eth.src spoofing). Loại: OVN logical pipeline stage, ingress LS table 0.

**Axis 7 importance.** Tier 1 absolute. Mọi packet LS bắt đầu ở stage này.

**Axis 8 mechanism.** Logical_Flow:
```
match: (1)
priority=50: action: drop /* default if eth.src violation */
priority=100: action: next /* if eth.src match port_security */
```

Combined với REGBIT_PORT_SEC_DROP (set bởi LS_IN_PORT_SEC_IP/ND nếu IP spoofing): drop deferred tới stage này.

**Axis 14 version drift.** Stable since OVN 2.4. Behavior change OVN 20.06 (REGBIT deferred drop pattern).

**Axis 17.** 2024-Q1 stale ARP cache post live migration case (cited 13.18 §REGBIT_PORT_SEC_DROP).

**Axis 18.** ovn-trace exercise với eth.src violation observe drop in table 0.

**Axis 20.** Linux iptables `-m mac --mac-source` analog stage. Cisco port-security similar.

## §13.19.2 LS_IN_PRE_ACL (table 3, cornerstone C6) DEEP-20

**Axis 1 concept.** Pre-ACL stage prepare conntrack state cho ACL evaluation. Set ct_state, evaluate ct() for stateful packet. Loại: pipeline stage.

**Axis 7 importance.** Tier 1. Without PRE_ACL, ACL stage không có ct_state populated.

**Axis 8 mechanism.** Logical_Flow:
```
match: (ip)
priority=110: action: reg0[0] = 1; ct_next; /* mark stateful packet */

match: (1)
priority=0: action: next; /* skip if not IP */
```

Sets up ct() invocation for next stage.

**Axis 14.** OVN 2.5 introduce với conntrack integration. Stable since.

**Axis 17.** Production stateful firewall debugging often touches this stage indirectly (via ct_state tracing).

**Axis 18.** ovn-trace với ip packet observe ct_next action invocation table 3.

**Axis 20.** Linux iptables PRE_ROUTING raw chain analog (ct invocation).

---

## §13.19.3 LS_IN_ACL (table 6, cornerstone C6) DEEP-20

**Axis 1 concept.** ACL evaluation stage. Match against compiled ACL rules from NB ACL table. Loại: pipeline stage.

**Axis 7 importance.** Tier 1. Stateful firewall enforcement core.

**Axis 8 mechanism.** Logical_Flow generated từ NB ACL:
```
match: (ip4.src == $set_a && tcp.dst == 80)
priority=2000: action: ct(commit); next; /* allow-related */
```

Conjunction action used cho large Address_Set + Port_Group reduce cartesian.

**Axis 14.** OVN 2.4 basic. 21.06 add tier column. 22.03 add log meter.

**Axis 17.** 2024-Q2 ACL drop case (9.24 ct_state troubleshooting cross-link).

**Axis 18.** `ovn-nbctl acl-add ls-test from-lport 1000 'ip4.src == 10.0.0.5' allow-related` + ovn-trace observe table 6 evaluation.

**Axis 19.** Mode A: Conjunction explosion với misuse. Mode B: ct_state mismatch rule expectation. Mode C: Priority conflict 2 rule cùng priority overlap.

**Axis 20.** Linux iptables FORWARD chain stateful filter analog. NSX-T DFW direct.

## §13.19.4 LS_IN_LB (table 12, cornerstone C6) DEEP-20

**Axis 1 concept.** Load balancer evaluation stage. Match VIP, select backend pool member, ct() with NAT. Loại: pipeline stage.

**Axis 7 importance.** Tier 1. LB foundation cho cloud workload.

**Axis 8 mechanism.** Logical_Flow with ct_lb action:
```
match: (ip4 && ct.new && ip4.dst == 10.0.0.100 && tcp.dst == 80)
priority=110: action: reg0[2] = 1; ct_lb(backends=192.168.1.5:80,192.168.1.6:80, hash_fields="ip_src,tp_src");
```

selection_fields control hash strategy (5-tuple consistent hash for sticky session).

**Axis 14.** OVN 2.4 simple. 22.03 hairpin SNAT integration. 24.03 LB affinity options.

**Axis 17.** 2024-Q1 LB VIP drops new connections threshold case (cited 13.3 LB).

**Axis 18.** `ovn-nbctl lb-add lb-web 10.0.0.100:80 192.168.1.5:80,192.168.1.6:80` + ovn-trace VIP packet observe stage 12.

**Axis 20.** Linux IPVS analog. AWS ALB hashing similar.

## §13.19.5 LS_IN_DHCP_OPTIONS (table 19, cornerstone C6) DEEP-20

**Axis 1 concept.** DHCP option process stage. Generate DHCP reply for VM client. Loại: pipeline stage.

**Axis 7 importance.** Tier 1. VM bootstrap fundamental — without DHCP reply, VM không có IP.

**Axis 8 mechanism.** Logical_Flow match DHCP request packet, set REGBIT_DHCP_OPTS_RESULT, populate reply via put_dhcp_opts action. ovn-controller compile flow set.

**Axis 14.** OVN 2.6 native DHCP introduce. 22.03 IPv6 DHCPv6 mature.

**Axis 17.** 2024 production: VM bootstrap fail. Investigation: DHCP_Options misconfig với wrong server_id. Fix: set explicit server_id matching gateway.

**Axis 18.** `ovn-nbctl dhcp-options-create 10.0.0.0/24` + lsp-set-dhcpv4-options exercise.

**Axis 20.** Linux dnsmasq DHCP server analog. Cisco IOS DHCP server similar.

## §13.19.6 LS_IN_L2_LKUP (table 27, cornerstone C6) DEEP-20

**Axis 1 concept.** L2 destination lookup stage. Match dst MAC → identify outport. Final ingress LS stage before egress. Loại: pipeline stage.

**Axis 7 importance.** Tier 1 absolute. Without L2 lookup, packet cannot reach destination port.

**Axis 8 mechanism.** Logical_Flow generated từ Logical_Switch_Port + MAC_Binding:
```
match: (eth.dst == aa:bb:cc:dd:ee:ff)
priority=50: action: outport = "lsp-vm2"; output;
```

Unknown unicast → flood to all ports (limit OVN 22.03 add MLF_RCV_FROM_RAMP filter).

**Axis 14.** OVN 2.4 basic. 22.03 FDB table improvement (limit broadcast). 24.03 L2 ARP responder integration.

**Axis 17.** 2024 broadcast storm post Logical_Switch misconfig (cited 13.2 LS).

**Axis 18.** `ovn-nbctl ls-add + lsp-add + lsp-set-addresses` exercise + ovn-trace observe table 27 lookup.

**Axis 20.** Linux bridge MAC table analog. Cisco MAC-address-table direct analog.

## §13.19.7 LR_IN_ADMISSION (table 0, cornerstone C6) DEEP-20

**Axis 1 concept.** First stage LR ingress. Validate packet for L3 forwarding (drop loopback, multicast, etc.). Loại: pipeline stage.

**Axis 7 importance.** Tier 1. LR pipeline entry gate.

**Axis 8 mechanism.** Logical_Flow:
```
match: (ip4.mcast || ip4.src == 0.0.0.0/8) /* etc invalid src */
priority=100: action: drop;

match: (1)
priority=0: action: next;
```

**Axis 14.** OVN 2.4 stable. Minor expansion across versions for IPv6 cases.

**Axis 17.** Network anomaly detection case: martian packet drop count spike → check ADMISSION drop counter via lflow stats.

**Axis 18.** ovn-trace với invalid ip.src observe drop table 0.

**Axis 20.** Linux iptables PREROUTING martian filter analog. Cisco IOS RPF similar.

---

## §13.19.8 LR_IN_IP_INPUT (table 4, cornerstone C6) DEEP-20

**Axis 1 concept.** Stage handling packet directed to LR itself (LRP IP). Generate ICMP reply, ARP responder, BFD. Loại: pipeline stage.

**Axis 7 importance.** Tier 1. Critical cho LR self-traffic (ping LR, BFD healthcheck).

**Axis 8 mechanism.** Logical_Flow match packet với eth.dst == LRP MAC + ip.dst == LRP IP. Trigger arp/icmp/bfd handler.

**Axis 14.** OVN 2.4 ICMP/ARP. 22.03 BFD integration.

**Axis 17.** Production: LR not respond ping. Investigation: LRP IP misconfig. Fix: lr-port set ip explicitly.

**Axis 18.** `ovn-nbctl lr-add + lrp-add` + ping LR IP from VM observe table 4.

**Axis 20.** Linux iptables INPUT chain analog. Cisco IOS routes-to-self similar.

---

## §13.19.9 LR_IN_DNAT (table 7, cornerstone C6) DEEP-20

**Axis 1 concept.** DNAT (Destination NAT) stage on LR ingress. Translate dst IP from external → internal. Loại: pipeline stage.

**Axis 7 importance.** Tier 1. North-south DNAT critical cho service publishing.

**Axis 8 mechanism.** Logical_Flow with ct() action commit DNAT mapping:
```
match: (ip4.dst == 1.2.3.4) /* external VIP */
priority=100: action: ct(commit, nat(dst=10.0.0.5)); next;
```

REGBIT_NAT_REDIRECT set if needs gateway chassis.

**Axis 14.** OVN 2.4 basic DNAT. 21.06 distributed DNAT. 22.03 force_snat_for_dnat option.

**Axis 17.** 2024 LB drops new connections case (cited 13.3 LB).

**Axis 18.** `ovn-nbctl lr-nat-add lr-main dnat 1.2.3.4 10.0.0.5` exercise.

**Axis 20.** Linux iptables DNAT in PREROUTING analog. Cisco static NAT similar.

---

## §13.19.10 LR_IN_GW_REDIRECT (table 17, cornerstone C6) DEEP-20

**Axis 1 concept.** Gateway redirect stage. Redirect packet to gateway chassis nếu REGBIT_NAT_REDIRECT set. Loại: pipeline stage.

**Axis 7 importance.** Tier 1. Distributed gateway architecture foundation.

**Axis 8 mechanism.** Logical_Flow:
```
match: (reg9[REGBIT_NAT_REDIRECT] == 1)
priority=100: action: outport = "cr-{LR_NAME}"; next; /* chassis-redirect port */

match: (1)
priority=0: action: next; /* local handle */
```

**Axis 14.** OVN 21.06 introduce với distributed gateway router. Stable.

**Axis 17.** 2024-Q3 gateway chassis CPU 100% case (cited 13.18 REGBIT_NAT_REDIRECT).

**Axis 18.** `ovn-nbctl lr-set-options redirect-chassis=chassis-1` + LR NAT setup exercise.

**Axis 20.** Cisco HSRP gateway redirect similar concept. NSX-T edge cluster centralize NAT analog.
