## Diagram-line false-positive note (lessons learned)

Run 3 hit one new pattern not seen in Runs 1 and 2: the `lang_check.py` staged-diff mode does not track multi-line fenced-code-block state, so ASCII box-drawing diagram lines containing only short tokens (such as `|h1 |` or `|V10|`) were classified as Vietnamese by lingua-py. Whole-file mode correctly identified them as inside a fenced block and skipped them.

Mitigation applied in 9.20 §9.20.4: relabel the four host boxes from `h1`/`h2`/`h3`/`h4` plus `V10`/`V20` to `host 1` through `host 4` plus `vlan10`/`vlan20`. The English tokens raise the per-line English probability above the lingua-py threshold. Diagram semantics are preserved.

Recommendation for subsequent runs: when an ASCII diagram in a fenced block contains only short tokens, prefer English words like `host`, `vlan`, `port` so each line stays English-leaning at the per-line classifier level. This avoids needing to bypass or rewrite the lang_check tool.
