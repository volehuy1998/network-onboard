## Dàn bài

## 8.3.1 tc qdisc taxonomy

Queuing discipline classful (có class hierarchy) hoặc classless. Root qdisc attach vào interface (`tc qdisc add dev eth0 root ...`). Classful qdisc (HTB, HFSC) chứa classes → sub-qdisc hoặc sub-classes, classless qdisc (fq_codel, pfifo_fast) không có hierarchy. Filter attach vào class để classify packet incoming vào class.

## 8.3.2 HTB, HFSC, fq_codel

HTB (Hierarchical Token Bucket): token bucket với borrow mechanism giữa sibling classes. Use cho bandwidth limit và fair sharing. HFSC (Hierarchical Fair Service Curve): real-time + link-share + upper-limit — phức tạp hơn HTB nhưng guarantee latency tốt hơn. fq_codel (Fair Queue + CoDel AQM): default kernel 5.0+, minimal config, giải quyết bufferbloat.

## 8.3.3 conntrack: zones, marks, states

`/proc/net/nf_conntrack` list tất cả entries. Column: protocol, src/dst IP+port, state (ESTABLISHED/ASSURED/TIME_WAIT), mark, zone. Zone (RFC 6599 analog): isolate overlapping address space, multiple tenant cùng conntrack table mà không conflict. connmark: set mark trên packet propagate vào conntrack entry.

## 8.3.4 iptables (nft backend) vs nftables native

Ubuntu 20.04+ ship `iptables-nft` — iptables command với nft kernel backend (alternative để giữ backward compat). Native `nft` syntax ngắn gọn hơn, kernel path nhanh hơn. Migration path: `iptables-save | iptables-restore-translate | nft -f -` để generate nft equivalent. RHEL 9+ default nftables native.

## 8.3.5 Forward reference: OVS ct() tích hợp với conntrack

OVS 2.5+ add `ct()` action: OpenFlow rule có thể đẩy packet qua kernel netfilter conntrack module, reuse code path của iptables/nftables. Flag `commit` install conntrack entry. Flag `zone=N` cho multi-tenant isolation. Xem chi tiết recipe trong Part 4.7 (OF programming) và Part 9.9 (QoS meter integration).
