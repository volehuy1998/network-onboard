## Dàn bài

## 11.4.1 IPsec overview: AH vs ESP, tunnel vs transport mode

Hai giao thức chính trong IPsec:
- **AH (Authentication Header, RFC 4302):** chỉ authentication + integrity, không encrypt. Rarely used trong modern deployment.
- **ESP (Encapsulating Security Payload, RFC 4303):** authentication + integrity + encryption. Default choice.

Hai mode:
- **Transport mode:** encrypt chỉ payload, giữ nguyên IP header. Host-to-host.
- **Tunnel mode:** encrypt toàn bộ original packet, wrap trong IP header mới. Site-to-site hoặc VPN client-gateway.

OVS IPsec tunnel luôn dùng ESP + tunnel mode.

## 11.4.2 IKE Phase 1: ISAKMP SA establishment

IKE = Internet Key Exchange (RFC 7296 for IKEv2, RFC 2409 for IKEv1 legacy). Phase 1 mục đích: thiết lập ISAKMP SA secure channel giữa hai peer, dùng cho Phase 2 negotiate.

Bước Phase 1:
1. **Negotiate algorithms:** encryption (AES/3DES), hash (SHA/MD5), DH group, lifetime.
2. **Diffie-Hellman key exchange:** compute shared secret mà không truyền qua wire.
3. **Identity + authentication:** PSK (Pre-Shared Key), RSA certificate, hoặc EAP.

Packet visible qua Wireshark: UDP port 500 (ISAKMP) + port 4500 (NAT-T). Filter `isakmp`.

## 11.4.3 IKE Phase 2: Child SA cho ESP

Phase 2 mục đích: derive key material cho ESP encryption, thiết lập Child SA (thường 2, inbound + outbound). Bước:
1. Nonce exchange (fresh randomness).
2. Optional DH re-key (PFS, Perfect Forward Secrecy).
3. ESP SA install.

Sau Phase 2: data plane bắt đầu encrypted. Wireshark filter `esp` hiển thị encrypted packet với SPI + sequence number visible nhưng payload opaque.

## 11.4.4 OVS IPsec tunnel: option psk

```
ovs-vsctl add-port br0 vxlan-ipsec -- \
 set Interface vxlan-ipsec type=vxlan \
 options:remote_ip=10.0.0.2 \
 options:key=flow \
 options:psk=mysecret
```

`options:psk` trigger `ovs-monitor-ipsec` daemon detect interface type + program strongSwan (hoặc Libreswan) accordingly. Không cần tự config strongSwan conf file.

Alternative PKI path:
```
options:remote_cert=</path/to/peer.crt>
```

An toàn hơn PSK vì không cần share secret trước.

## 11.4.5 strongSwan daemon và ovs-monitor-ipsec

`systemctl enable --now openvswitch-ipsec`: enable ovs-monitor-ipsec + strongSwan services.

Verify IKE status: `ipsec statusall` trên cả hai peer. Expected output: active SA với algorithm + remaining lifetime.

Debug: `journalctl -u strongswan-starter.service` khi tunnel fail to establish.

## 11.4.6 Topology UofSC Lab 15 absorb: IPsec-over-GRE

Cùng topology Part 11.3 (2 container + 3 ISP router), nhưng tunnel giờ là IPsec-over-GRE:
- GRE xử lý encapsulation + multi-protocol (bridge Ethernet).
- IPsec ESP wrap toàn bộ GRE packet → encrypted.

Outer structure: `[Outer IP] [ESP] [Encrypted GRE [Inner IP [Inner payload]]]`.

## 11.4.7 PSK rotation

Quy tắc: rotate PSK định kỳ (60-90 ngày). Procedure:
1. Add new PSK alongside old on receiver side.
2. Switch sender to new PSK.
3. Remove old PSK sau 24h grace period.

Tốt hơn: migrate sang cert-based auth nếu scale lớn.
