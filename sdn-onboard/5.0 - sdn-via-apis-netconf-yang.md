## Dàn bài

## 5.0.1 Vendor motivation: keep silicon, add API layer

*Sẽ phát triển: vendor preserve existing silicon; expose YANG/NETCONF API thay vì thay thế data plane.*

## 5.0.2 NETCONF (RFC 6241 June 2011)

*Sẽ phát triển: XML-based config protocol; candidate/running/startup datastores; commit/confirm.*

## 5.0.3 YANG data model (RFC 7950 August 2016)

*Sẽ phát triển: YANG 1.1; module/submodule/grouping/augment; IETF models (ietf-interfaces, ietf-routing).*

## 5.0.4 RESTCONF (RFC 8040 January 2017)

*Sẽ phát triển: HTTP/JSON alternative cho NETCONF; same YANG models.*

## 5.0.5 BGP-LS (RFC 7752) cho topology distribution

*Sẽ phát triển: BGP Link-State; carry IGP topology qua BGP; use case SDN WAN controller.*

## 5.0.6 PCE-P (RFC 5440) cho TE path computation

*Sẽ phát triển: Path Computation Element Protocol; stateful PCE (RFC 8231); segment routing integration.*

## 5.0.7 Cisco ACI, Juniper Contrail xuất hiện

*Sẽ phát triển: ACI 2013 launch, APIC northbound REST; Contrail 2013 opensource, BGP EVPN control plane.*
