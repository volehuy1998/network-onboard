## 16.2.1 AF_XDP architecture (4 rings)

*Skeleton — nội dung sẽ bao gồm:* FILL ring (userspace give buffers to kernel), RX ring (kernel push received packets), TX ring (userspace push for transmit), COMPLETION ring (kernel release TX buffers), umem zero-copy memory.

## 16.2.2 Driver vs SKB mode

*Skeleton — nội dung sẽ bao gồm:* XDP_DRIVER (hardware-offload, requires driver support like i40e, ice, mlx5), XDP_SKB (kernel SKB path, slower but universal), XDP_HW (NIC-native offload).

## 16.2.3 XDP program types + actions

*Skeleton — nội dung sẽ bao gồm:* XDP_PASS (let kernel stack process), XDP_DROP (drop at earliest point), XDP_REDIRECT (redirect to AF_XDP socket or another interface), XDP_TX (forward out same interface).

## 16.2.4 libbpf + libxdp userspace

*Skeleton — nội dung sẽ bao gồm:* libbpf program loading + map management, libxdp XDP-specific wrapper, BPF maps shared between XDP program + userspace (PERCPU_ARRAY, HASH).

## 16.2.5 Performance characteristics

*Skeleton — nội dung sẽ bao gồm:* AF_XDP benchmarks (20-50 Gbps on standard NICs), latency (~10μs vs DPDK ~2μs), advantage: keeps kernel features (netfilter, tc qdisc), disadvantage: lower raw PPS than DPDK.
