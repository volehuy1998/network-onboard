## 9.7.1 OVS Mirror table

Mirror là cơ chế copy traffic từ một/nhiều port/VLAN sang destination (port hoặc VLAN). Trong physical switch gọi là SPAN hoặc RSPAN. OVS implement qua `Mirror` table trong OVSDB.

`Mirror` được tham chiếu từ `Bridge.mirrors`. Có:

- **Source selectors**: `select_src_port`, `select_dst_port`, `select_vlan`, `select_all=true`.
- **Destination**: `output_port` (local SPAN) hoặc `output_vlan` (RSPAN — re-tag packet với VLAN này và emit ra trunk).

---

## 9.7.2 Atomic transaction tạo Mirror

Compass Ch G cung cấp idiom chuẩn:

```bash
[host]# ovs-vsctl -- set Bridge br0 mirrors=@m \
         -- --id=@eth0 get Port eth0 \
         -- --id=@eth1 get Port eth1 \
         -- --id=@eth2 get Port eth2 \
         -- --id=@m create Mirror name=mymirror \
                select-dst-port=@eth0,@eth1 \
                select-src-port=@eth0,@eth1 \
                output-port=@eth2
```

Atomic: nếu port eth2 không tồn tại, Mirror row không tạo, Bridge.mirrors không đổi.

Verify:

```bash
[host]# ovs-vsctl list Mirror mymirror
[host]# ovs-vsctl get Mirror mymirror statistics
```

Statistics cột `tx_packets`, `tx_bytes` — số packet đã mirror.

---

## 9.7.3 Cleanup Mirror

Compass Ch G: Mirror tự garbage-collect khi unreferenced, khác với QoS/Queue:

```bash
[host]# ovs-vsctl clear Bridge br0 mirrors
```

Sau lệnh này, Mirror row unreferenced → OVSDB tự xóa. KHÔNG cần explicit `destroy Mirror`. Khác biệt quan trọng với QoS/Queue (Part 9.9) — hai table đó phải `destroy` thủ công.

---

## 9.7.4 Capture traffic từ mirror output

Mirror ra port `tap0` (internal type), packet copy xuất hiện trên netdev `tap0`:

```bash
[host]# ip link set tap0 promisc on
[host]# tcpdump -i tap0 -nn -w mirror.pcap
```

Feed `mirror.pcap` vào Wireshark hoặc Suricata.

RSPAN, analyzer ở remote host:

```bash
[host]# ovs-vsctl -- --id=@m create Mirror name=rspan \
         select-all=true output-vlan=999
[host]# ovs-vsctl set Bridge br0 mirrors=@m
```

Remote analyzer join VLAN 999 trên trunk.

### ▶ Guided Exercise 8: Mirror VM traffic vào tap capture

**Outcomes:** Cấu hình mirror port VM vào tap internal, capture bằng tcpdump.

**Before You Begin:**

```bash
[host]# ovs-vsctl add-br br-mirror
[host]# ovs-vsctl add-port br-mirror vm1 -- set Interface vm1 type=internal
[host]# ovs-vsctl add-port br-mirror analyzer -- set Interface analyzer type=internal
[host]# ip link set vm1 up && ip addr add 10.99.0.1/24 dev vm1
[host]# ip link set analyzer up
```

**Instructions:**

**1.** Tạo mirror:

```bash
[host]# ovs-vsctl -- --id=@m create Mirror name=vm1mirror \
         select-src-port=@p1 select-dst-port=@p1 output-port=@p2 \
         -- --id=@p1 get Port vm1 \
         -- --id=@p2 get Port analyzer \
         -- set Bridge br-mirror mirrors=@m
```

**2.** Start tcpdump: `tcpdump -i analyzer -nn &`

**3.** Generate traffic: `ping -c 5 -I 10.99.0.1 10.99.0.2`

**4.** Quan sát tcpdump output — packet ICMP xuất hiện trên analyzer.

**5.** Cleanup: `ovs-vsctl clear Bridge br-mirror mirrors && ovs-vsctl del-br br-mirror`

## Điểm cốt lõi cần nhớ

Thứ nhất, OVS Mirror table cung cấp SPAN/RSPAN native — không cần tool bên ngoài.

Thứ hai, atomic transaction `-- --id=@...` là cách chuẩn tạo Mirror — đảm bảo Bridge.mirrors reference đúng Mirror row trong một step.

Thứ ba, Mirror tự garbage-collect khi unreferenced — dùng `clear Bridge br0 mirrors` để disable.

Thứ tư, tcpdump trên internal port output + Wireshark/Suricata — foundation cho network visibility.
