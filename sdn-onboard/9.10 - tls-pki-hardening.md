## Dàn bài

## 9.10.1 ptcp: vs ssl:, production chỉ dùng ssl:

OVS support 2 connection mode cho Controller và Manager URIs:
- `ptcp:<port>`: plain TCP, không authentication, không encryption. CHỈ dùng cho lab offline.
- `ssl:<ip>:<port>`: TLS 1.2+ mutual authentication (client + server cert validation).

Trong production, mọi Controller và Manager target phải là `ssl:`. Cho `ptcp:` trên WAN hoặc shared network = credential exposure + MITM risk.

## 9.10.2 ovs-pki workflow: minimal PKI

OVS ship `ovs-pki` helper. CA + cert workflow tối giản, không cần vault hoặc Hashicorp PKI:

```
ovs-pki init # tạo root CA
ovs-pki req+sign ovsclient switch # switch certificate
ovs-pki req+sign ctl controller # controller certificate
ovs-vsctl set-ssl /etc/openvswitch/key.pem \
 /etc/openvswitch/cert.pem \
 /etc/openvswitch/cacert.pem
ovs-vsctl set-controller br0 ssl:10.0.0.10:6653
```

`ovs-pki init` tạo CA tại `/var/lib/openvswitch/pki/` mặc định. `req+sign` combine hai bước thường tách, generate CSR + sign bằng CA, vào một command.

## 9.10.3 TLS handshake verification từ controller

Trước khi tin `set-controller` hoạt động, test handshake bằng `openssl`:

```
openssl s_client -connect 10.0.0.10:6653 -CAfile cacert.pem \
 -cert controller-cert.pem -key controller-key.pem
```

Output "Verify return code: 0 (ok)" = mutual auth thành công. Nếu fail với "unable to get local issuer certificate", thường là CA chain không đúng.

## 9.10.4 CA rotation: add-then-switch

Quy tắc critical: khi rotate CA, add new root vào `cacert.pem` TRƯỚC khi switch client cert sang root mới. Lý do: nếu switch cert trước, mọi client đang hold cert cũ sẽ không verify được, toàn hệ thống mất control channel.

Trình tự:
1. `cat old-ca.pem new-ca.pem > cacert.pem` (bundle cả hai).
2. Reload OVS: `ovs-appctl exit --cleanup` rồi restart.
3. Issue cert mới từ new CA cho từng client, rotate dần.
4. Khi tất cả client đã migrate, remove old-ca từ bundle.

## 9.10.5 Cipher suite và TLS version restriction

Default TLS: v1.2+, modern cipher suites. Override với environment variable `OVS_SSL_PROTOCOLS` nếu cần disable TLS 1.0/1.1 cho compliance:

```
OVS_SSL_PROTOCOLS="tls1.2,tls1.3" systemctl restart openvswitch-switch
```
