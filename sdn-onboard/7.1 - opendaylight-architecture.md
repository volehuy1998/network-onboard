## Dàn bài

## 7.1.1 ODL formation: Linux Foundation, April 2013

12 founding members gồm Cisco, IBM, Citrix, Ericsson, HP, Intel, Juniper, Microsoft, NEC, Red Hat, Brocade, VMware, umbrella project đầu tiên cho SDN multi-vendor. Target: open-source controller tương thích với mọi vendor switch. Funding mô hình premium membership + LF governance.

## 7.1.2 Release train: Hydrogen (2014) → Argon final (2023)

15 releases theo periodic table: Hydrogen, Helium, Lithium, ..., Fluorine, Neon, Sodium, Magnesium, Aluminium, Silicon, Phosphorus, Sulfur, Chlorine, Argon. Fluorine (2019) là LTS cuối được adopt rộng rãi trong production. Argon 2023 là release cuối, project chuyển sang maintenance-only.

## 7.1.3 MD-SAL core

Model-Driven Service Abstraction Layer: toàn bộ ODL core xoay quanh YANG data model. DataBroker API cho applications read/write vào in-memory datastore. Notification bus giữa components. RESTCONF northbound auto-generate từ YANG model. Pattern: định nghĩa YANG schema → ODL tự gen API + storage.

## 7.1.4 Southbound plugins: OpenFlow, NETCONF, BGP-LS

OpenFlow plugin: chính thức, hỗ trợ OF 1.0 + 1.3, cũng có OF 1.5 nhưng ít tested. NETCONF plugin: phổ biến với service provider muốn dùng YANG-driven config. BGP-LS plugin: cho PCEP + segment routing WAN use case. Multi-protocol southbound là differentiation chính của ODL so với ONOS/Ryu.

## 7.1.5 Real deployment: Telefonica, Comcast, AT&T

Telefonica UNICA project: ODL trực tiếp tham gia vào architecture. Comcast CDN over Fiber: ODL controller cho IP backbone slicing. AT&T Network Cloud (ECOMP → ONAP): ODL một trong các core component orchestration. Tất cả đều service provider. DC adoption thấp hơn, đa phần dùng vendor controller.

## 7.1.6 ODL current state 2026, maintenance-only

Commit velocity đã giảm mạnh 2023+. LF Networking umbrella (2018) gom nhiều project SDN+NFV nhưng ODL không tự growth. ONOS chiếm service provider scale mạnh hơn. Outlook 2026: ODL vẫn maintain core cho legacy deployment, không có feature mới lớn.
