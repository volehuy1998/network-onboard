### 2.4. Spot-check (4.9 lines 1747-1752 sample audit per plan §11)

Plan §11 noted line 1751 "OFPAT_COPY_FIELD type 28 (OF 1.5)" claim has a
reference to `enum ofp15_action_type` which is **not** present in OVS
`include/openflow/openflow-1.5.h`. The OVS implementation uses
`OFPAT_RAW15_COPY_FIELD` in `lib/ofp-actions.c`. The numeric type code "28"
is wired through OVS's RAW15 tag rather than a public enum.

Plan §11 also noted line 1752 "OFPAT_PUSH_PBB type 26, POP_PBB type 27"
attribution to `include/openflow/openflow-1.3.h` is wrong: PBB action codes
are not in OVS v2.17.9 OF 1.3 header.

These two §11 findings remain TBD pending wider verification (would require
deeper grep across `lib/ofp-actions.c` and OVS's PBB handling). The audit
log for the broader 4.9 catalog content is satisfactory at the §3.7 sampling
budget; deeper line-by-line per-action audit is out of session capacity for
v3.11. R3 final audit will spot-check 1-2 of these claims.
