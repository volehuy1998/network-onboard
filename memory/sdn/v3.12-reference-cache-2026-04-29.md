# Plan v3.12 Reference Cache (curated, validated 2026-04-29)

> **Status.** Active. Used during plan v3.12 translation passes when an offline-source consultation does not resolve a question. Per the user directive of 2026-04-29 ("search for 30-40 links beforehand to create a quick cache; during execution, only check those links if necessary"), this is the **first place to look** before any free web search.
>
> **Priority order (per user, 2026-04-29):**
>
> 1. **OVS** (Open vSwitch), top priority.
> 2. **OVN**, top priority.
> 3. **OpenFlow**, second priority.
>
> **Lookup discipline (mandatory).**
>
> 1. **Tier 1, offline source.** First, try the local OVS, OVN, or OpenFlow repos pinned at the curriculum baseline tags (OVS `v2.17.9`, OVN `v22.03.8`, OpenFlow `spec1.6_rc1`). The offline source is authoritative for code, headers, schemas, NEWS files, RFC quoted bodies, and CHANGELOG. The offline OVS repo also contains every man page in `Documentation/`.
> 2. **Tier 2, this cache.** If Tier 1 does not cover the need (for example, a community blog, an OpenStack integration note, an IETF specification, an upstream mailing-list discussion), consult **only the URLs in this cache**.
> 3. **Tier 3, free web search.** Only if Tier 1 and Tier 2 are insufficient. Use only large, official, reputable, community-driven sites: openvswitch.org, ovn.org, opennetworking.org, rfc-editor.org, kernel.org, netfilter.org, docs.openstack.org, www.usenix.org, sigcomm.org, github.com (project repos), man7.org, blogs.cisco.com, redhat.com, nvidia.com developer site. **Never** consult anonymous Medium articles, Stack Overflow answers, or unmoderated forum threads as authoritative.
>
> **HTTP validation.** All 36 URLs were validated on 2026-04-29 via `curl -I` (HTTP 200 or HTTP 200 after redirect). If a URL returns 4xx or 5xx during plan execution, halt the lookup, log the dead URL, escalate to the user. Do not silently substitute.
>
> **Scope of trust.** This cache lists URLs by topic. Each URL carries a "level" annotation: `CANONICAL` (project home or official documentation), `SPEC` (RFC or IEEE / ONF specification), `MAN` (Linux man page), `COMMUNITY` (project mailing list or issue tracker), `BLOG` (community-curated blog by a project maintainer), `ACADEMIC` (peer-reviewed paper). Higher-level sources are preferred when they cover the topic.
>
> **HTTP method.** `curl -L -s -o /dev/null -w "%{http_code}" --max-time 12 -A "Mozilla/5.0" <url>`. Followed redirects on 301 / 302.
>
> **Re-validation cadence.** R0 of each future plan (v3.12, v3.13, ...) re-runs HTTP validation on this cache. URLs that fail at R0 are removed or replaced before R1 starts.

## Out of cache by design

The following sources are intentionally not in the cache:

- **Red Hat Developer blog and customer-portal articles.** HTTP `403 Forbidden` for unauthenticated requests; cannot be HTTP-validated reliably. If the curriculum already cites a Red Hat URL (for example, the revalidator-process article cited in 9.26), that citation stays verbatim per Rule 14 §14.5; do not consult the URL during translation.
- **DPDK, BPF, XDP, BGP, K8S official sites.** Permanent ban per CLAUDE.md North Star. Existing curriculum content in those areas is translated as-is, not expanded; the cache must not invite expansion.
- **Personal Medium / dev.to articles.** Not authoritative.
- **Stack Overflow.** Not authoritative; answers are user-generated.
- **Vendor commercial pages without authoritative documentation depth** (for example, generic NSX or ACI marketing pages). Use the official vendor documentation portal (Red Hat, NVIDIA, VMware) only when the curriculum already cites it.

## Maintenance

When a URL fails HTTP validation in a future plan R0, replace it with the most authoritative working alternative from the same priority tier:

| Failed | Replacement strategy |
|---|---|
| `docs.openvswitch.org/en/latest/<page>` 4xx | Fall back to `github.com/openvswitch/ovs/blob/v2.17.9/Documentation/<corresponding-rst>`. |
| `docs.ovn.org/en/latest/<page>` 4xx | Fall back to `github.com/ovn-org/ovn/blob/v22.03.8/Documentation/<corresponding-rst>`. |
| `man7.org/linux/man-pages/...` 4xx | Fall back to `openvswitch.org/support/dist-docs/<man>.html`. |
| `opennetworking.org/wp-content/uploads/...` 4xx | Fall back to the IETF archive or to the offline `openflow` repo (`spec1.6_rc1` for OF 1.6 only). |
| `rfc-editor.org/rfc/<n>` 4xx | Fall back to `tools.ietf.org/html/<n>`. |
| `mail.openvswitch.org/pipermail/...` 4xx | Fall back to `patchwork.ozlabs.org/project/openvswitch/list/`. |
| `github.com/...` 429 | Wait and retry; the 429 is rate-limit, not a dead URL. |

Update this file in the same commit that performs the replacement.
