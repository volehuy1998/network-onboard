### 5.1. Mục tiêu

### 5.2. Deliverables

Cho mỗi axis:

- **Tên axis** (tiếng Việt + tiếng Anh)
- **Định nghĩa formal** (1 câu)
- **Pass criteria** cụ thể: keyword đạt khi có gì
- **Fail example:** đoạn text chỉ là "mention" không đạt

- **Auto-detect signal:** regex pattern + position pattern để script chấm tự động
- **Manual review threshold:** khi nào script không chắc chắn cần manual

Đi qua từng cohort, viết 20-axis cho mọi keyword trong cohort.

### 10.2. Method

1. Pick cohort theo Phase E priority order
2. Cho mỗi keyword trong cohort:
   - Read current state (file mention, score hiện tại từ scorecard Phase D)
   - Identify axis còn thiếu (rubric scorecard)
   - Research từng axis missing:
     - REF entry (offline source)
     - Upstream source via MCP GitHub Rule 14 (file:function:commit)
     - Existing curriculum cross-reference
     - Linux kernel source nếu axis 16 (source code anchor) cần
     - Cross-domain comparison nếu axis 20 cần
   - Write content fill axis missing
   - Update cross-link
3. Sau khi cohort done:
   - Re-run rubric audit (script Phase D)
   - Confirm tất cả keyword cohort đó đạt threshold
   - Commit batch (1 commit per cohort, ~500-2000 dòng tùy cohort size)
   - User review batch (mỗi 5-10 cohort)

### 10.3. Cohort progression

- **Cornerstone cohort first** (~5 cohort, ~30 keyword): cao nhất priority
- **Medium cohort second** (~25 cohort, ~150 keyword)
- **Peripheral cohort last** (~24 cohort, ~140 keyword)

### 10.4. Per-batch deliverables

- 1 commit cho mỗi cohort
- Re-audit scorecard update committed
- User review notes (committed in `memory/sdn/cohort-review-log.md`)

### 10.5. Acceptance Gate per cohort

- [ ] 100% keyword trong cohort đạt minimum threshold per Phase B definition
- [ ] Cross-link integrity 0 broken
- [ ] Quality gate pass (Rule 9, 11, 13, 14)
- [ ] Em-dash density < 0.10/line per file modified
- [ ] Source code citation verified Rule 14
- [ ] User mid-stage review every 5-10 cohort batch
