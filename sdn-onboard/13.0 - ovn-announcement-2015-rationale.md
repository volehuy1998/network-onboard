## 13.0.1 Announcement 13/01/2015

OVN (Open Virtual Network) công bố ngày 13/01/2015 trên Network Heresy blog. Bốn tác giả:

- **Justin Pettit** (co-founder Nicira, OVS architect từ 2007).
- **Ben Pfaff** (co-founder Nicira, NSDI 2015 paper lead).
- **Chris Wright** (Red Hat VP, sau này CTO).
- **Madhu Venugopal** (Red Hat).

Title: "**OVN: Bringing Native Virtual Networking to OVS**".

Key message: OVN là "native" virtual networking cho OVS — built-in, không phải add-on. Trước OVN, OVS dùng control từ third-party (OpenStack Neutron + ML2, OpenDaylight, Contrail). OVN cung cấp control plane native.

## 13.0.2 Động lực thiết kế

Nicira team tại VMware muốn open-source version tương đương NSX cho:

1. **OpenStack** — Neutron ML2 cần SDN controller portable.
2. **Kubernetes** — container networking cần L3/L7 policy + overlay.
3. **Bare-metal OVN** — deployment ngoài cloud (telco).

Red Hat join vì cùng nhu cầu — thay Neutron ML2 + linuxbridge trong RHOSP.

**Nguyên tắc thiết kế**:

- **Reuse OVS + OVSDB**: không viết lại data plane.
- **Portable**: chạy trên mọi orchestrator.
- **Logical abstraction**: admin nghĩ logical switch/router.
- **Distributed control**: mỗi chassis có `ovn-controller` local.

## 13.0.3 3-tier architecture

Compass Ch 17: "OVN sits above OVS as a logical networking controller. Its architecture is three tiers."

```
 ┌────────────────────────────────────┐
 │ Tier 1: ovn-nbctl                  │  ← admin/orchestrator
 │         (writes logical intent)    │
 └───────────────┬────────────────────┘
                 │ JSON-RPC
 ┌───────────────▼────────────────────┐
 │ Tier 2: Northbound DB (NB)         │  ← intent storage
 │    + ovn-northd (translator)       │
 │    → Southbound DB (SB)            │  ← computed logical flows
 └───────────────┬────────────────────┘
                 │ JSON-RPC
 ┌───────────────▼────────────────────┐
 │ Tier 3: ovn-controller (per host)  │  ← per-chassis agent
 │         → OpenFlow rules → br-int  │
 └────────────────────────────────────┘
```

**Tier 1 — Intent** (NBDB):
- Admin write intent qua `ovn-nbctl`.
- NBDB: `Logical_Switch`, `Logical_Router`, `Logical_Switch_Port`, `Logical_Router_Port`, `ACL`, `NAT`, `Load_Balancer`.

**Tier 2 — Translation** (`ovn-northd` + SBDB):
- `ovn-northd` read NBDB, compute logical flows, write SBDB.
- SBDB: `Chassis`, `Port_Binding`, `Logical_Flow`, `Datapath_Binding`, `MAC_Binding`.

**Tier 3 — Data plane** (`ovn-controller`):
- Run trên mỗi chassis.
- Subscribe SBDB, translate logical flow → OpenFlow rule cài vào `br-int`.

## 13.0.4 So sánh competitor

**OVN vs ODL** (Part 7.1):
- ODL: umbrella multi-protocol. OVN: purpose-built cho OVS.
- ODL: monolithic. OVN: distributed.
- ODL: maintenance 2023+. OVN: active.

**OVN vs VMware NSX** (Part 7.3):
- NSX: proprietary, ESXi integration. OVN: open-source, any hypervisor.
- NSX: full stack (DFW, LB). OVN: compose với OVS + external.

**OVN vs Contrail** (Part 7.3):
- Contrail: vRouter + BGP EVPN. OVN: OVS + OVSDB + OpenFlow.

## 13.0.5 Use case

**OpenStack Neutron + OVN ML2**: thay linuxbridge + router/DHCP agent. Red Hat RHOSP 16+ default OVN.

**OVN-Kubernetes**: CNI plugin từ RHEL CoreOS/OKD 4.x. Network Policy + Service.

## Điểm cốt lõi cần nhớ

Thứ nhất, OVN công bố 13/01/2015 như "native virtual networking to OVS" — fill khoảng trống control plane.

Thứ hai, 3-tier architecture (NB intent → northd → SB → ovn-controller → OpenFlow) là đóng góp kiến trúc then chốt.

Thứ ba, OVN reuse OVS + OVSDB hoàn toàn. Trade-off: không hỗ trợ hardware ngoài OVS ecosystem nhưng nhẹ + đơn giản.

Thứ tư, use case: OpenStack + K8s. Standalone OVN cho telco. Block 17-19 đi sâu standalone case study.
