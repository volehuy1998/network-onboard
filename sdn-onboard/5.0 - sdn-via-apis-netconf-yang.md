## Dàn bài

## 5.0.1 Vendor motivation: keep silicon, add API layer

Khác với OpenFlow (thay data plane), "SDN via APIs" giữ nguyên silicon vendor-specific + expose YANG/NETCONF API để automation. Ưu điểm: không cần rewrite firmware, không có performance regression. Nhược điểm: API chỉ là projection của vendor-specific features — không phải true programmability.

## 5.0.2 NETCONF (RFC 6241, June 2011)

XML-based configuration protocol qua SSH (standard port 830) hoặc TLS. Ba datastore: `candidate` (edit-in-progress), `running` (active), `startup` (persistence). Operation chính: `get`, `get-config`, `edit-config`, `commit`/`confirm`. Transaction-oriented: edit candidate → commit để promote sang running, rollback nếu timeout.

## 5.0.3 YANG data model (RFC 7950, August 2016)

YANG 1.1 là schema language mô tả structure của config + state data. Cấu trúc modular: `module` chứa `container` + `list` + `leaf` + `grouping` + `augment`. IETF ship các module chuẩn: `ietf-interfaces`, `ietf-routing`, `ietf-ip`. Vendor extend qua augment thay vì redefine.

## 5.0.4 RESTCONF (RFC 8040, January 2017)

HTTP/JSON alternative cho NETCONF — cùng YANG model, khác transport. URL path mirror YANG tree, method HTTP (GET/PUT/POST/DELETE) map CRUD. Lý do ra đời: NETCONF XML khó integrate với web tooling, RESTCONF phù hợp DevOps workflow.

## 5.0.5 BGP-LS (RFC 7752) cho topology distribution

BGP Link-State: mở rộng BGP để carry IGP topology information (IS-IS/OSPF LSDB) sang external controller. Use case: SDN WAN controller cần biết topology đầy đủ nhưng không muốn tham gia IGP. Controller peer BGP-LS với route reflector, nhận topology updates real-time.

## 5.0.6 PCE-P (RFC 5440) cho TE path computation

Path Computation Element Protocol: separation giữa routing (router quyết định shortest path) và TE computation (PCE quyết định traffic-engineered path). Stateful PCE (RFC 8231): PCE có write authority vào LSP (MPLS tunnel) cho active TE re-optimization. Integration với Segment Routing cho source-routed LSP.

## 5.0.7 Cisco ACI, Juniper Contrail launch 2013

ACI (2013): Cisco APIC controller northbound REST API, southbound tự định nghĩa OpFlex + iVXLAN. Contrail (2013, opensource): Juniper acquired, control plane BGP EVPN + XMPP southbound. Cả hai đều "SDN via APIs" không dùng OpenFlow, đi ngược với ONF positioning.
