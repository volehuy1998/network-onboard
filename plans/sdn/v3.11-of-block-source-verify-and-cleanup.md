### 3.3. Six verification primitives

#### Primitive A: function-name citation

```bash
cd /c/Users/voleh/Documents/ovs
git checkout v2.17.9 -q
grep -nE "^<fn>" <path>
# fallback if empty:
grep -rnE "^<fn>" lib/ ofproto/ utilities/ include/
```

If still empty: category = `FABRICATED`.

#### Primitive B: URL citation

```bash
# Inspect URL pattern: https://github.com/openvswitch/ovs/blob/<ref>/<path>[#<anchor>]
# Acceptable <ref> for OF block: branch-2.17 | v2.17.9 | a 40-char SHA
# Unacceptable <ref>: main | master | branch- without LTS suffix
ls <path>   # at the cited tag; if MISSING, WRONG_PATH
```

For OF spec URLs (`opennetworking.org`, `onf.org/wp-content/uploads/...`): these are external and not verifiable offline. Document as "external; not Rule 14 surface" and skip.

#### Primitive C: free-standing wire-protocol fact

For OFPP_, OFPP_NORMAL, OFP_TCP_PORT (IANA 6653), OFP_VERSION values:

```bash
cd /c/Users/voleh/Documents/ovs
git checkout v2.17.9 -q
grep -nE "#define OFPP_LOCAL|#define OFPP_NORMAL|#define OFPP_CONTROLLER|#define OFPP_TABLE" include/openflow/openflow-1.0.h
grep -nE "#define OFP_TCP_PORT|#define OFP_VERSION" include/openflow/openflow*.h
```

If the cited value differs from `#define`: category = `WRONG_FACT`.

For free-standing OF version introduction claims like "OF 1.5+" for an action:

```bash
cd /c/Users/voleh/Documents/ovs
grep -nE "OFPAT_RAW10_<name>|OFPAT_RAW11_<name>|OFPAT_RAW13_<name>|OFPAT_RAW15_<name>" lib/ofp-actions.c
```

The `RAW10/11/13/15` prefix marks the OF version where the action was introduced. If the curriculum says "OF 1.5+" but the OVS code shows `OFPAT_RAW11_*`: category = `WRONG_FACT` (the action is OF 1.1+, not 1.5+).

#### Primitive D: enum-type code citation

For "OFPAT_COPY_FIELD type 28 (OF 1.5)":

```bash

# Step 1: confirm the action is wired up at OF 1.5 in OVS

# Step 2: confirm the wire numeric type code via the OFPAT type registry

# Step 3: cross-reference the OF 1.5 spec PDF claim (external; document the spec citation but skip

```

If the OFPAT_RAW prefix doesn't match the cited OF version: `WRONG_FACT`. If the type-code numeric doesn't appear: `FABRICATED_NUMERIC` (new category, see §3.4).
