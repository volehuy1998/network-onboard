## 11.3.1 GRE (RFC 2784)

GRE (Generic Routing Encapsulation, RFC 2784 năm 2000) — tunnel protocol đơn giản, predate VXLAN.

Header:

```
 ┌──────┬──────────┬──────────────┐
 │Outer │ GRE (4B) │ Inner frame  │
 │ IP   │ + option │              │
 │(20B) │ (4B key) │              │
 └──────┴──────────┴──────────────┘
```

Overhead: 24 bytes (IP 20 + GRE 4, hoặc +4 cho key extension RFC 2890).

**Khác VXLAN**:

- Không UDP encap — GRE là IP protocol 47, không port.
- Không NAT-friendly (IP protocol 47 khó traverse NAT).
- Không multipath trên underlay (ECMP hash thiếu port).
- Overhead thấp hơn (24B vs 50B VXLAN).

**Khi dùng GRE**: point-to-point trusted tunnel, site-to-site VPN (GRE over IPsec), legacy.

## 11.3.2 OVS GRE port

Compass Ch 11:

```bash
[host]# ovs-vsctl add-port br-tun gre0 -- \
         set Interface gre0 type=gre \
         options:remote_ip=10.0.0.2 \
         options:key=100 \
         options:tos=inherit \
         options:ttl=64
```

- `options:key=100` — GRE key 32-bit (RFC 2890). Phân biệt tunnel cùng endpoint.
- `options:tos=inherit` — copy inner ToS → outer.
- `options:key=flow` — multipoint pattern (Part 11.0).

### ▶ Guided Exercise 10: 3-node OSPF + GRE lab

**Outcomes:** Build 3-node OSPF underlay + GRE overlay, verify tenant traffic.

**Before You Begin:** 3 Ubuntu 22.04 VM với OVS + FRR.

**Instructions:**

**1.** Underlay IP:

```bash
# Host A:
[hostA]# ip addr add 10.0.1.1/24 dev eth0
[hostA]# ip route add 10.0.2.0/24 via 10.0.1.2

# Host B:
[hostB]# ip addr add 10.0.2.1/24 dev eth0
[hostB]# ip route add 10.0.1.0/24 via 10.0.2.2

# Host C (router):
[hostC]# ip addr add 10.0.1.2/24 dev eth0
[hostC]# ip addr add 10.0.2.2/24 dev eth1
[hostC]# sysctl -w net.ipv4.ip_forward=1
```

**2.** OSPF trên Host C (optional — static route cũng đủ):

```bash
[hostC]# cat > /etc/frr/ospfd.conf <<EOF
router ospf
 network 10.0.1.0/24 area 0
 network 10.0.2.0/24 area 0
EOF
[hostC]# systemctl restart frr
```

**3.** GRE tunnel A ↔ B:

```bash
[hostA]# ovs-vsctl add-br br-tun
[hostA]# ovs-vsctl add-port br-tun tap-a -- set Interface tap-a type=internal
[hostA]# ip addr add 192.168.1.1/24 dev tap-a && ip link set tap-a up
[hostA]# ovs-vsctl add-port br-tun gre0 -- set Interface gre0 type=gre \
         options:remote_ip=10.0.2.1 options:key=100

[hostB]# ovs-vsctl add-br br-tun
[hostB]# ovs-vsctl add-port br-tun tap-b -- set Interface tap-b type=internal
[hostB]# ip addr add 192.168.1.2/24 dev tap-b && ip link set tap-b up
[hostB]# ovs-vsctl add-port br-tun gre0 -- set Interface gre0 type=gre \
         options:remote_ip=10.0.1.1 options:key=100
```

**4.** Verify:

```bash
[hostA]# ping 192.168.1.2
[hostA]# ovs-appctl tnl/neigh/show
[hostA]# tcpdump -ni eth0 proto 47       # IP protocol 47 = GRE
```

tcpdump hiện packet GRE từ A → C → B.

**5.** Cleanup: `ovs-vsctl del-br br-tun` trên A, B.

## 11.3.4 Trade-off thực tế

**Khi chọn GRE**:

- Point-to-point trusted tunnel.
- Bandwidth critical (overhead thấp).
- Legacy router không support VXLAN.

**Khi tránh GRE**:

- NAT traversal.
- Multicloud với ECMP underlay (GRE không hash per-flow).
- Scale cloud lớn (không VNI 24-bit native).

OVN **không dùng GRE** — default Geneve. GRE chỉ cho legacy interop.

---

## Điểm cốt lõi cần nhớ

Thứ nhất, GRE (RFC 2784) overhead 24B (thấp hơn VXLAN 50B). Không UDP → không NAT-friendly.

Thứ hai, OVS `options:key=N` phân biệt tenant. `options:key=flow` cho multipoint.

Thứ ba, OSPF underlay + GRE overlay là pattern classic legacy, không phổ biến cho cloud DC mới.

Thứ tư, GRE over IPsec (Part 11.4) là pattern chuẩn site-to-site VPN — encryption + GRE flexibility.
