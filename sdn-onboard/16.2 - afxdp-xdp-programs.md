Sau khi hoàn thành phần này, bạn có thể:

---

## 16.2.1 AF_XDP architecture (4 rings)

*Skeleton — nội dung sẽ bao gồm:* FILL ring (userspace give buffers to kernel), RX ring (kernel push received packets), TX ring (userspace push for transmit), COMPLETION ring (kernel release TX buffers), umem zero-copy memory.

## 16.2.2 Driver vs SKB mode

*Skeleton — nội dung sẽ bao gồm:* XDP_DRIVER (hardware-offload, requires driver support like i40e, ice, mlx5), XDP_SKB (kernel SKB path, slower but universal), XDP_HW (NIC-native offload).

## 16.2.3 XDP program types + actions

*Skeleton — nội dung sẽ bao gồm:* XDP_PASS (let kernel stack process), XDP_DROP (drop at earliest point), XDP_REDIRECT (redirect to AF_XDP socket or another interface), XDP_TX (forward out same interface).

## 16.2.4 libbpf + libxdp userspace

*Skeleton — nội dung sẽ bao gồm:* libbpf program loading + map management, libxdp XDP-specific wrapper, BPF maps shared between XDP program + userspace (PERCPU_ARRAY, HASH).

## 16.2.5 Performance characteristics

*Skeleton — nội dung sẽ bao gồm:* AF_XDP benchmarks (20-50 Gbps on standard NICs), latency (~10μs vs DPDK ~2μs), advantage: keeps kernel features (netfilter, tc qdisc), disadvantage: lower raw PPS than DPDK.

### ▶ Guided Exercise 1: Attach minimal XDP_PASS program với bpftool

> **Trạng thái lab verification:** Output dựa trên kernel XDP docs + bpftool reference 2024; chờ verify lab thực.

**Mục đích:** Entry-level XDP lab — viết một XDP program đơn giản nhất (return XDP_PASS cho mọi packet), compile, load, attach, verify. Baseline cho mọi XDP development later.

**Chuẩn bị:**
- Ubuntu 22.04 VM với kernel 5.15+
- Install toolchain:
  ```bash
  sudo apt install -y clang llvm libbpf-dev libelf-dev gcc-multilib \
                      linux-headers-$(uname -r) linux-tools-$(uname -r)
  ```
- Driver support XDP native: e1000e, virtio_net, mlx5_core, i40e, ice đều support. Check: `ethtool -i eth0 | grep driver`.

**Mô hình lab:**

```
┌────────────────────────────────────────────────┐
│    Ubuntu 22.04 VM                             │
│                                                │
│    Packet inbound ───► NIC (eth0)              │
│                          │                     │
│                          ▼                     │
│                   ┌──────────────────┐         │
│                   │  XDP program     │         │
│                   │  xdp_pass_prog() │         │
│                   │  return XDP_PASS │         │
│                   └──────┬───────────┘         │
│                          │                     │
│                          ▼                     │
│                   Kernel network stack         │
│                   (normal processing)          │
└────────────────────────────────────────────────┘
```

**Bước thực hiện:**

**1.** Viết XDP program `xdp_pass.c`:

```c
#include <linux/bpf.h>
#include <bpf/bpf_helpers.h>

SEC("xdp")
int xdp_pass_prog(struct xdp_md *ctx) {
    return XDP_PASS;
}

char _license[] SEC("license") = "GPL";
```

**2.** Compile:

```bash
[host]$ clang -O2 -g -target bpf -c xdp_pass.c -o xdp_pass.o
[host]$ ls -la xdp_pass.o
-rw-r--r-- 1 user user 1024 Apr 22 12:34 xdp_pass.o
```

**3.** Load program vào kernel:

```bash
[host]# bpftool prog load xdp_pass.o /sys/fs/bpf/xdp_pass
[host]# ls /sys/fs/bpf/
xdp_pass
```

**4.** Get program ID + attach to interface:

```bash
[host]# PROG_ID=$(bpftool prog show pinned /sys/fs/bpf/xdp_pass | awk -F: '/^[0-9]/ {print $1}')
[host]# echo $PROG_ID
42
[host]# bpftool net attach xdp id $PROG_ID dev eth0
```

**5.** Verify attached:

```bash
[host]# ip link show eth0
```

Output kỳ vọng:

```
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 xdp/id:42 qdisc ...
    link/ether aa:bb:cc:dd:ee:ff brd ff:ff:ff:ff:ff:ff
    prog/xdp id 42 tag 123abc jited
```

Key indicators: `xdp/id:42` trong ip link output, `jited` = program was JIT-compiled to native machine code cho performance.

**6.** Measure performance impact (sanity check — should be negligible):

```bash
# Baseline (no XDP)
[host]# bpftool net detach xdp dev eth0
[host]# iperf3 -c 10.0.0.1 -t 10

# With XDP_PASS
[host]# bpftool net attach xdp id $PROG_ID dev eth0
[host]# iperf3 -c 10.0.0.1 -t 10
```

Output kỳ vọng: throughput deltas within ±1% (XDP_PASS is near-zero overhead since it just falls through to kernel stack).

**Bài học học được:**

Thứ nhất, **XDP program architecture**: C source → clang -target bpf → eBPF bytecode → kernel verifier → JIT compile → native execution. Pipeline này ensure safety (verifier reject unbounded loops, invalid memory access) + performance (JIT to native instructions).

Thứ hai, **XDP_PASS is no-op**: packet continues to kernel stack unchanged. Useful cho initial development — attach program without breaking traffic, gradually add logic.

Thứ ba, **bpftool vs ip link**: cả hai có thể attach XDP. `bpftool` advantage: cross-platform, native BPF concepts. `ip link set dev eth0 xdp obj xdp_pass.o` là alternative.

**Cleanup:**

```bash
[host]# bpftool net detach xdp dev eth0
[host]# rm /sys/fs/bpf/xdp_pass
```

### ▶ Lab 2: Packet filter at XDP with AF_XDP redirect

> **Trạng thái lab verification:** Output throughput + CPU stats dựa trên XDP tutorial benchmarks; chờ verify lab thực.

**Mục đích:** Thực hành selective packet processing — XDP program drop non-TCP packets (reduce kernel load), redirect TCP packets vào AF_XDP socket cho userspace processing. So sánh CPU overhead vs equivalent iptables rule.

**Chuẩn bị:**
- Hoàn thành Ex1 baseline (XDP toolchain installed)
- XDP tutorial cloned: `git clone --recurse-submodules https://github.com/xdp-project/xdp-tutorial.git`
- Reference implementation: `xdp-tutorial/advanced03-AF_XDP/`

**Mô hình lab:**

```
┌───────────────────────────────────────────────┐
│    Ubuntu 22.04 VM                            │
│                                               │
│    Incoming packets                           │
│          │                                    │
│          ▼                                    │
│    ┌─────────────────────────────────┐        │
│    │  XDP program (driver mode)      │        │
│    │                                 │        │
│    │  if (is_tcp(pkt))               │        │
│    │      return XDP_REDIRECT →      │─┐      │
│    │           AF_XDP socket         │ │      │
│    │  else if (is_icmp(pkt))         │ │      │
│    │      return XDP_PASS            │ │      │
│    │  else                           │ │      │
│    │      return XDP_DROP            │ │      │
│    └─────────────────────────────────┘ │      │
│                                        │      │
│    ┌───────────────────────────────────▼───┐  │
│    │ AF_XDP socket (userspace)            │  │
│    │  └── Python/C program parse TCP      │  │
│    │      print src/dst + port + len      │  │
│    └──────────────────────────────────────┘  │
└───────────────────────────────────────────────┘

Comparison: iptables equivalent

┌───────────────────────────────────────────────┐
│    Incoming packets                           │
│          │                                    │
│          ▼                                    │
│    NIC → kernel SKB alloc → iptables chain    │
│                                               │
│    iptables -A INPUT -p icmp -j ACCEPT        │
│    iptables -A INPUT -p tcp -j LOG -j DROP    │
│    iptables -A INPUT -j DROP                  │
└───────────────────────────────────────────────┘
```

**Bước thực hiện:**

**1.** Cd to tutorial exercise:

```bash
[host]$ cd xdp-tutorial/advanced03-AF_XDP
```

**2.** Review `af_xdp_kern.c` và `af_xdp_user.c` source.

**3.** Build:

```bash
[host]$ make
```

Output: compile `af_xdp_kern.o` + `af_xdp_user` binary.

**4.** Load + attach:

```bash
[host]# ./af_xdp_user --dev eth0 --poll-mode --filename af_xdp_kern.o
```

Output kỳ vọng:

```
Loading custom XDP program...
XDP program attached to eth0 (id 42, driver mode)
AF_XDP socket created on interface eth0 queue 0
Listening for packets...
```

**5.** Generate mixed traffic (từ machine khác):

```bash
# TCP (should be redirected to AF_XDP)
[remote]$ curl http://<vm-ip>:8080

# ICMP (should PASS to kernel)
[remote]$ ping -c 5 <vm-ip>

# UDP (should be DROPPED)
[remote]$ nc -u <vm-ip> 12345
```

**6.** Quan sát userspace output:

```
[AF_XDP Rx] 1432 bytes from 10.0.0.5:54321 -> 10.0.0.1:8080 (TCP, flags SYN)
[AF_XDP Rx] 66 bytes from 10.0.0.5:54321 -> 10.0.0.1:8080 (TCP, flags ACK)
[AF_XDP Rx] 524 bytes from 10.0.0.5:54321 -> 10.0.0.1:8080 (TCP, flags PSH+ACK)
```

ICMP ping từ host khác thành công (PASSes kernel). UDP packets dropped at XDP, không tới socket.

**7.** Benchmark — so sánh với iptables equivalent:

```bash
# Method A: XDP + AF_XDP (above setup)
# Measure CPU with htop while generating TCP traffic at 1 Gbps
# Expected: ~5-10% CPU cho XDP filter + ~20% cho userspace process

# Method B: iptables equivalent (detach XDP first)
[host]# ./af_xdp_user --dev eth0 --unload
[host]# iptables -A INPUT -p udp -j DROP
[host]# iptables -A INPUT -p tcp -j NFQUEUE --queue-num 0
# User-space NFQUEUE handler parse TCP headers
# Expected: ~30-50% CPU cho iptables + NFQUEUE processing
```

**Output mong muốn:** Bảng so sánh:

| Method | CPU per Gbps TCP | Latency | Notes |
|--------|------------------|---------|-------|
| XDP + AF_XDP | ~25% | ~10 μs | Drop UDP tại driver → zero kernel overhead |
| iptables + NFQUEUE | ~40% | ~30 μs | SKB allocation + netfilter chain traversal |

XDP approach ~37% less CPU overhead và ~3× lower latency.

**Bài học học được:**

Thứ nhất, **XDP dropping UDP tại driver** là massive saving — packet never traverse kernel stack. Compare với iptables DROP (happens after SKB alloc + netfilter chain) → XDP ~10× cheaper cho drops.

Thứ hai, **AF_XDP zero-copy** preserve kernel features cho selected packets. ICMP vẫn reach kernel (ping works), TCP redirect vào userspace (custom processing), UDP dropped silently. Flexibility without full DPDK bypass.

Thứ ba, **Production use case**: DDoS mitigation (XDP drop malicious traffic), custom TCP accelerator (AF_XDP redirect for in-house processing), monitoring tap (XDP mirror to AF_XDP without impact). Cilium + Katran (Facebook LB) real-world examples.

**Cleanup:**

```bash
[host]# ./af_xdp_user --dev eth0 --unload
[host]# iptables -F  # if used comparison method
```
