## 12.2.1 Micro-segmentation: East-West firewall

Legacy DC security: **north-south firewall** giữa Internet và DC. East-West traffic không filter.

Vấn đề: VM compromise → attacker lateral movement giữa VMs không bị chặn.

**Micro-segmentation**: enforce ACL ở tầng hypervisor cho mọi flow giữa VM. Default-deny → chỉ traffic cần thiết được phép.

Benefits:
- **Lateral movement block**.
- **Per-workload policy** granular.
- **Compliance** (PCI-DSS, HIPAA).

---

## 12.2.2 Implementations

**VMware NSX Distributed Firewall**:
- Enforce trong vSwitch của mỗi ESXi host.
- Security Group applied per-VM.
- Stateful conntrack trong hypervisor.
- Multi-cloud: AWS VPC, Azure VNet, ESXi.

**Cisco ACI EPG + Contract**:
- EPG = tập VM có cùng policy.
- Contract = allowed protocol giữa hai EPG.
- Default deny. Explicit allow via Contract.
- Enforce ở TOR Nexus 9000.

**OVN ACL + Port_Group** (Khối XIII):
- ACL row trong NB database.
- Port_Group aggregate port theo tag.
- Stateful via `ct()` action.
- Enforce ở OVS tại mỗi chassis.

## 12.2.3 Ví dụ 3-tier app

Tenant 3-tier web:
- Web: 10.0.1.0/24
- App: 10.0.2.0/24
- DB: 10.0.3.0/24

Micro-segmentation:
- Internet → web: allow 80/443.
- Web → app: allow 8080.
- App → db: allow 5432.
- Bất kỳ khác: DENY.

OVN equivalent:

```bash
[admin]# ovn-nbctl pg-add pg-web web-vm1 web-vm2
[admin]# ovn-nbctl pg-add pg-app app-vm1 app-vm2
[admin]# ovn-nbctl pg-add pg-db db-vm1 db-vm2

[admin]# ovn-nbctl acl-add pg-web from-lport 1000 \
         "outport == @pg-app && tcp.dst == 8080" allow-related
[admin]# ovn-nbctl acl-add pg-app from-lport 1000 \
         "outport == @pg-db && tcp.dst == 5432" allow-related
[admin]# ovn-nbctl acl-add pg-web from-lport 500 "1" drop
```

## 12.2.4 Service chaining với NSH

**Service chaining**: route traffic qua sequence service:

```
VM → [SF1: Firewall] → [SF2: IDS] → [SF3: WAF] → Destination
```

**NSH** (RFC 8300, 01/2018):

```
 ┌──────────────┬──────────────┬────────┐
 │ Outer transp │ NSH (4-byte) │ Payload│
 │ (VXLAN, GRE) │ + context    │        │
 └──────────────┴──────────────┴────────┘
```

NSH chứa:
- **Service Path Identifier (SPI)**: ID của chain.
- **Service Index (SI)**: hop counter (decrement mỗi SF).
- **Context headers**: metadata.

SF inspect SI, thực thi service, decrement SI, forward theo SFF.

---

## 12.2.5 NSH vs OpenFlow-based chaining

NSH đòi hỏi SF hỗ trợ parsing — vendor limited. Alternative: **OpenFlow-based chaining**.

```
table=0, in_port=tenant, actions=output:port-to-FW
table=0, in_port=from-FW, actions=output:port-to-IDS
table=0, in_port=from-IDS, actions=output:destination
```

Service chain encoded trong OpenFlow rules.

OVN 22.x+ có Port_Chain experimental.

---

## 12.2.6 Production deployments

- **VMware NSX**: micro-seg + service insertion qua NSX Edge.
- **Cisco ACI**: EPG/Contract + service graph.
- **OVN**: ACL/Port_Group + OpenFlow-based chaining.
- **Kubernetes**: Network Policy + service mesh (Istio).

---

## Điểm cốt lõi cần nhớ

Thứ nhất, micro-segmentation là East-West firewall — block lateral movement. Required cho modern security.

Thứ hai, 3 implementations: NSX DFW, ACI EPG/Contract, OVN ACL/Port_Group.

Thứ ba, service chaining cần metadata. NSH (RFC 8300) standard; OpenFlow-based alternative.

Thứ tư, micro-seg + service chain + anycast gateway = complete SDN DC stack.
