## Dàn bài

## 9.9.1 OVS configure QoS nhưng không implement

Điểm chốt: OVS KHÔNG tự làm queueing, mà gọi xuống Linux `tc` cho kernel datapath hoặc DPDK `rte_sched` cho userspace datapath. OVS chỉ expose config surface qua OVSDB tables `QoS` + `Queue`. Hiểu điều này tránh hiểu lầm "sửa OVS queue sẽ ảnh hưởng traffic", thực ra bạn đang sửa tc.

## 9.9.2 Ingress policing: token bucket đơn giản

Đơn giản nhất, set 2 column trên Interface row:

```
ovs-vsctl set interface tap0 ingress_policing_rate=1000 \
 ingress_policing_burst=100
```

`ingress_policing_rate` kbps, `ingress_policing_burst` kb. 0 = disable. Packet vượt rate → drop ngay. Không có queue, không có shaping, policing thuần túy.

## 9.9.3 Egress shaping với linux-htb (Hierarchical Token Bucket)

Recipe verbatim từ upstream FAQ:

```
ovs-vsctl -- add-br br0 \
 -- add-port br0 eth0 \
 -- add-port br0 vif1.0 -- set interface vif1.0 ofport_request=5 \
 -- set port eth0 qos=@newqos \
 -- --id=@newqos create qos type=linux-htb \
 other-config:max-rate=1000000000 \
 queues:123=@vif10queue \
 -- --id=@vif10queue create queue other-config:max-rate=10000000

ovs-ofctl add-flow br0 in_port=5,actions=set_queue:123,normal
```

**Foot-gun chính:** queue chỉ hoạt động nếu OpenFlow rule reference `set_queue:` đúng queue ID. Không có rule → packet đi default queue → không shape. Đây là lỗi cấu hình QoS phổ biến nhất ngoài đời.

## 9.9.4 Meter-based policing (OF 1.3+): CIR/PIR color marking

Option mới từ OF 1.3: meter table độc lập với flow table. `add-meter` rồi reference bằng `meter:<id>` action. Recipe Single-Rate Two-Colors:

```
ovs-ofctl -O OpenFlow15 add-meter br0 \
 meter=1,kbps,band=type=drop,rate=300000
ovs-ofctl -O OpenFlow15 add-flow br0 \
 tcp,tp_dst=21,actions=meter:1,output:3
```

Three-Colors (Green < CIR < Yellow < PIR < Red) theo spec OF 1.3 nhưng OVS hiện tại chỉ implement 2-color (Green + Red, xóa Yellow). Lý do: Linux `tc` policer không natively hỗ trợ 3-color mark.

## 9.9.5 Verify + cleanup: destroy QoS và Queue explicit

Verify:
- `ovs-appctl qos/show eth0`. OVS perspective.
- `tc qdisc show dev eth0` + `tc class show dev eth0`, kernel perspective.
- Hai output phải khớp nếu OVS program đúng.

Cleanup (QoS + Queue KHÔNG auto-GC):

```
ovs-vsctl -- destroy QoS eth0 -- clear Port eth0 qos
ovs-vsctl -- --all destroy Queue
```

Quên `destroy` → orphan QoS/Queue row tích tụ qua thời gian, debugging khó.
