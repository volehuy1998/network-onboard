## 9.11.1 Nhóm bridge và FDB

```bash
[host]# ovs-appctl bridge/dump-flows br0        # mọi flow bao gồm hidden
[host]# ovs-appctl fdb/show br0                  # MAC learning table
[host]# ovs-appctl fdb/flush br0                 # buộc relearn MAC
[host]# ovs-appctl mdb/show br0                  # multicast snooping table
```

`fdb/show` hiển thị port + MAC + VLAN + age. Debug "traffic đi sai port" — thường do MAC learning sai hoặc VLAN misconfigured.

---

## 9.11.2 Nhóm bond và LACP

```bash
[host]# ovs-appctl bond/show bond0
[host]# ovs-appctl bond/show-stats
[host]# ovs-appctl bond/migrate bond0 <hash> <slave>
[host]# ovs-appctl lacp/show bond0
```

`bond/migrate` — manually move một hash bucket sang member khác, dùng khi rebalance tự động không tối ưu.

---

## 9.11.3 Nhóm spanning-tree

```bash
[host]# ovs-appctl stp/show                     # 802.1D classic STP
[host]# ovs-appctl rstp/show                    # 802.1w RSTP
```

Output bridge state, port state, port role. Debug loop và convergence issue.

## 9.11.7 Nhóm tunnel

```bash
[host]# ovs-appctl tnl/neigh/show               # tunnel neighbor ARP
[host]# ovs-appctl tnl/arp/show                  # tunnel ARP cache
```

## 9.11.8 Nhóm upcall và revalidator

```bash
[host]# ovs-appctl upcall/show
[host]# ovs-appctl upcall/set-flow-limit 500000
[host]# ovs-appctl revalidator/wait
[host]# ovs-appctl revalidator/purge
```

`upcall/show` xem flow-install rate. `upcall/set-flow-limit` tăng khi chạm ngưỡng. `revalidator/purge` force xóa megaflow (relearn).

---

## 9.11.9 Nhóm telemetry (coverage + memory + vlog)

```bash
[host]# ovs-appctl coverage/show
[host]# ovs-appctl memory/show
[host]# ovs-appctl vlog/list
[host]# ovs-appctl vlog/set dpif:file:dbg
[host]# ovs-appctl vlog/reopen
```

Compass Ch L: vlog subsystem modular (each source file is a module), per-destination (console/syslog/file), per-level (emer/err/warn/info/dbg).

Counter quan trọng:

- `cache-misses` — miss rate megaflow.
- `upcall_flow_limit` — lần hit rate limit.
- `netlink_received/sent` — Netlink I/O.
- `bridge_flush` — bridge reconfigure.
- `rconn_queued` — OpenFlow connection queue buildup.

Run `coverage/show` cron + log delta để detect anomaly.

## 9.11.10 Nhóm cluster (OVSDB clustered)

```bash
[host]# ovs-appctl -t /var/run/openvswitch/ovsdb-server.ctl \
         cluster/status OVN_Southbound
[host]# ovs-appctl -t /var/run/openvswitch/ovsdb-server.ctl \
         cluster/leave OVN_Southbound
[host]# ovs-appctl -t /var/run/openvswitch/ovsdb-server.ctl \
         cluster/kick OVN_Southbound <sid>
```

Dùng cho OVSDB clustered (Part 10.1). Debug split-brain.

### ▶ Guided Exercise 12: Coverage delta analysis

**Outcomes:** Run coverage/show trước và sau sự kiện, tính delta.

**Before You Begin:**

```bash
[host]# ovs-vsctl add-br br-test
[host]# ovs-vsctl add-port br-test tap0 -- set Interface tap0 type=internal
[host]# ip link set tap0 up && ip addr add 10.99.0.1/24 dev tap0
```

**Instructions:**

**1.** Baseline: `ovs-appctl coverage/show > /tmp/cov-before.txt`

**2.** Trigger — tạo 1000 flow:

```bash
[host]# for i in $(seq 1 1000); do
          ovs-ofctl add-flow br-test "cookie=0x$i,priority=100,in_port=tap0,tcp,tp_dst=$i,actions=drop"
        done
```

**3.** Counter sau: `ovs-appctl coverage/show > /tmp/cov-after.txt`

**4.** Diff: `diff /tmp/cov-before.txt /tmp/cov-after.txt`

Sẽ thấy `flow_table_mods` tăng 1000.

**5.** Cleanup: `ovs-vsctl del-br br-test`

## Điểm cốt lõi cần nhớ

Thứ nhất, `ovs-appctl` là RPC Unix socket đến OVS daemon — flexibility cao nhất.

Thứ hai, target tổ chức theo nhóm: bridge, bond, spanning-tree, ofproto, dpctl, dpif, tunnel, upcall, revalidator, coverage, vlog, cluster. Biết group giúp tìm target đúng.

Thứ ba, `coverage/show` + cron + delta logging là foundation cho OVS observability.

Thứ tư, `vlog/set module:destination:level` là kỹ thuật debug sâu — tăng log chỉ cho module quan tâm.
