## Dàn bài

## 8.1.1 brctl legacy vs ip link modern

Package `bridge-utils` deprecated từ Ubuntu 18.04+. Modern syntax dùng iproute2: `ip link add name br0 type bridge`, `ip link set eth0 master br0`. Kernel implementation: `net/bridge/` với FDB (forwarding database) aging time default 5 phút. iproute2 unified cả bridge + VLAN + tunnel command.

## 8.1.2 Bridge datapath trong kernel

`net/bridge/br_forward.c` implement forwarding logic: lookup FDB → forward tới port đích hoặc flood nếu unknown destination. STP optional (disable by default Linux kernel). netfilter bridge-hook (`ebtables`) cho filtering L2. Throughput kernel bridge ~5-10 Gbps trên modern hardware, đủ cho hầu hết container use case không cần SDN.

## 8.1.3 veth pair: net/core/veth.c

veth tạo cặp đối xứng device, packet đưa vào một đầu xuất ra đầu còn lại. Use case chính: nối namespace với bridge (một đầu trong namespace, một đầu trên bridge). Implementation: virtual Ethernet driver với synthetic tx/rx queue. Không có hardware offload, pure CPU work, có overhead ~1-2 μs per packet hop.

## 8.1.4 macvlan modes: private, vepa, bridge, passthru

macvlan tạo virtual interface với MAC address riêng trên cùng physical port.
- **private**: container isolate nhau, không forward giữa macvlan sibling.
- **vepa**: Virtual Ethernet Port Aggregator, traffic giữa sibling đi ra external switch (cần hairpin config).
- **bridge** (default): container giao tiếp trực tiếp trong kernel bridge domain.
- **passthru**: expose 1-to-1 physical port cho container (SR-IOV-like).

## 8.1.5 ipvlan L2 vs L3

ipvlan share một MAC address duy nhất nhưng nhiều IP. L2 mode: L2 switching trong kernel, tương tự macvlan bridge mode nhưng shared MAC. L3 mode: kernel làm L3 forwarding giữa ipvlan children, không broadcast, không ARP qua IPVLAN boundary. Use case container overlay khi cần scale >1000 container/host (MAC table size limit).

## 8.1.6 Tại sao OVS (Block IX) thay thế Linux bridge

Linux bridge limitations cho SDN:
- Không programmable (rule tĩnh theo MAC learning).
- Không có flow statistics.
- Không có per-flow actions (mod_nw_src, set_queue, v.v.).
- VLAN handling đơn giản, không hỗ trợ QinQ native.

OVS giải quyết bằng OpenFlow pipeline + megaflow cache + extensible action set. Part 9.1 sẽ đi sâu kiến trúc OVS giải quyết từng limitation.
