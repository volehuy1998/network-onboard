## Dàn bài

## 11.3.1 GRE header structure và tại sao vẫn dùng

GRE (Generic Routing Encapsulation, RFC 2784, 2000) là tunneling protocol cũ nhất còn active. Header layout:
- Flags + Version (2 byte): C, R, K, S bits + version (thường 0 for GRE, 1 for PPTP).
- Protocol type (2 byte): Ethertype của inner packet (0x0800 IPv4, 0x6558 transparent Ethernet bridging).
- Optional: Key (4 byte, nếu K=1), tenant ID đơn giản.

Overhead: 4 byte (tối thiểu) đến 16 byte (với key + checksum + sequence).

Vẫn còn relevant vì: (1) simplest tunneling, debug dễ nhất; (2) hỗ trợ universal trên mọi router vendor; (3) GRE over IPsec là VPN site-to-site baseline trong enterprise.

## 11.3.2 OVS GRE port configuration

```
ovs-vsctl add-port br-tun gre0 -- \
 set Interface gre0 type=gre \
 options:remote_ip=10.0.0.2 \
 options:key=flow
```

`options:key=flow` = defer key selection đến OpenFlow action `set_tunnel:<N>`. Cho phép một tunnel port reach nhiều tenant qua key khác nhau.

Hoặc key cố định: `options:key=1000`, tất cả traffic qua tunnel này dùng GRE key 1000.

## 11.3.3 Topology UofSC Lab 14 absorb: 3 ISP router + 2 site container

Topology:
```
 Site A container (d1) Site B container (d2)
 - Mininet inside - Mininet inside
 - Hosts h1, h2 - Hosts h3, h4
 - Switch s1 - Switch s2
 | |
 | 192.168.1.10 | 192.168.2.10
 v v
 [r1] --- [r3] --- [r2] (ISP network, OSPF)
```

Docker container chạy Mininet bên trong → simulate 2 customer site. 3 router ISP (r1 → r3 → r2) chạy OSPF để route underlay giữa 192.168.1.0/24 ↔ 192.168.2.0/24.

## 11.3.4 OSPF config với FRR/Quagga

```
router ospf
 network 0.0.0.0/0 area 0
 passive-interface default
 no passive-interface eth1
 no passive-interface eth2
```

Chạy trên r1, r2, r3. Verify: `show ip ospf neighbor` phải thấy adjacency Full/BDR trên mỗi interface inter-router.

## 11.3.5 GRE tunnel giữa s1 và s2 xuyên ISP

Trên site A container:
```
ovs-vsctl add-port s1 tun0 -- \
 set Interface tun0 type=gre \
 options:remote_ip=192.168.2.10 \
 options:key=flow
```

Trên site B container:
```
ovs-vsctl add-port s2 tun0 -- \
 set Interface tun0 type=gre \
 options:remote_ip=192.168.1.10 \
 options:key=flow
```

Add OpenFlow rule forward traffic tenant qua tun0.

## 11.3.6 Wireshark inspection, outer vs inner packet

Capture trên r3 (middle ISP):
- Outer IPv4: src=192.168.1.10, dst=192.168.2.10, proto=47 (GRE).
- GRE header: key 0 (hoặc key bạn set).
- Inner IPv4: src=10.0.0.1 (h1), dst=10.0.0.3 (h3).

Wireshark auto-decode "Generic Routing Encapsulation" và hiển thị nested.

## 11.3.7 MTU math và common pitfall

IEEE 1500 MTU underlay → inner MTU tối đa = 1500 - 24 (IP outer) - 4 (GRE min) = 1472 byte.

Nếu inner ping `-s 1473` → fragment hoặc drop (nếu DF set). Đây là lý do hầu hết production setup tweak inner MTU = 1420-1450 để có buffer.
