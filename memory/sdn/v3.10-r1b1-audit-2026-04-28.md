## 4. Empirical observation

13.1 contradicts the §11 sample-audit prediction of 30-45 percent for R1.B. The actual rate (1.9 percent) shows that schema-anchored content is far more accurate than pipeline-table-anchored content (18.0 was 44 percent, dominated by `main`-branch authoring bias). The schema parser anchors verification at row-of-truth precision; the pipeline-stage block in `northd/northd.c` is a single source of truth that the 13.1 author cross-referenced correctly.

This suggests R1.B.2 (13.2 + 13.5) and R1.D (services cluster) may also run at low single-digit error rates if their authors used the same offline-repo discipline. R1.C (controller / northd cluster) and R1.E (Block 17 / 18 / 19) likely run higher because pipeline / table-id claims dominate.

The 45 COMMIT_SHA candidates are uniformly false positives from the inventory regex (CLI output UUIDs). The 172 SCHEMA_TABLE candidates are uniformly PASS (every cited table name is real). The inventory's wider regex coverage produced high recall but low precision; per-row inspection still completes faster than expected because most rows close in 5-10 seconds (visual scan plus PASS).
