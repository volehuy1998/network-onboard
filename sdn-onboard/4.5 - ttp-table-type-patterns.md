## Dàn bài

## 4.5.1 Vấn đề: mỗi vendor silicon support khác subset của OF spec

Phân tích gốc rễ: OpenFlow spec 1.3 mô tả đầy đủ match field, action, group, meter nhưng silicon thực tế chỉ hỗ trợ subset. Nêu ví dụ cụ thể: Broadcom Trident2 hỗ trợ ingress ACL 8K entry TCAM nhưng không có egress ACL; Intel FM6000 hỗ trợ 16-table pipeline nhưng mỗi table có match field constraint khác nhau. Kết quả: controller viết flow_mod "theoretically valid" nhưng switch reject với OFPERR_BAD_MATCH, bug chỉ lộ ở runtime, không ở compile/validate time.

## 4.5.2 TTP khái niệm: pre-agreed pattern

Mô tả TTP approach: thay vì controller query từng feature cờ, ONF định nghĩa các "pattern" pre-agreed, mỗi TTP specify cụ thể number of tables, match fields per table, instruction set per table, group types supported. Nêu flow: switch advertise list of supported TTP IDs ở OFPT_FEATURES_REPLY; controller chọn TTP phù hợp rồi chỉ install flow_mod tuân theo pattern đó. Analogy với HTTP Accept/Content-Type negotiation, client-server thoả thuận common ground trước khi trao đổi payload.

## 4.5.3 ONF TS-017: TTP spec

Giới thiệu ONF Technical Specification 017 "OpenFlow Table Type Patterns" (phiên bản 1.0 công bố 15/08/2014). Nêu format: YANG-based data model define schema cho TTP; reference TTPs bao gồm L2-L3 Edge TTP, Aggregation-Core TTP, WAN PE TTP. Phân tích mục đích: TTP không thay thế OpenFlow spec, nó là layer bổ sung above spec để quản lý heterogeneity. Pointer đến URL opennetworking.org/wp-content/uploads/2014/10/TR_TTP_v1.0.pdf (archive nếu dead link).

## 4.5.4 Alternative: Flow Objectives abstraction (xem Block VI)

So sánh TTP với approach khác giải cùng bài toán, **Flow Objectives** của ONOS (Brigadier Boerner et al. 2015): thay vì controller generate match/action theo TTP pattern, ứng dụng express intent ở abstraction cao hơn ("forward traffic matching X to port Y with priority Z"), driver layer dịch sang flow_mod tuỳ switch capability. Nêu lý do ONOS chọn Flow Objectives thay TTP: (a) TTP yêu cầu application phải aware switch capability; Flow Objectives ẩn toàn bộ heterogeneity khỏi app layer; (b) TTP fixed catalog không extend runtime được; Flow Objectives có driver SDK. Forward reference đến 6.1 (Flow Objectives abstraction deep-dive).
