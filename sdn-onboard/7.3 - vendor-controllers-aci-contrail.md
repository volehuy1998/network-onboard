## Dàn bài

## 7.3.1 Cisco APIC + ACI

APIC (Application Policy Infrastructure Controller) launch 2013 cùng Nexus 9000 leaf-spine. Southbound: OpFlex protocol (Cisco-proprietary) + iVXLAN encap. Policy model: Tenants → Application Network Profiles → Endpoint Groups (EPGs) + Contracts. Khác biệt chính với OpenFlow: không có flow programming, controller push intent/policy, silicon decide forwarding.

## 7.3.2 Juniper Contrail — formerly OpenContrail

OpenContrail opensource 2014 (Juniper donate từ Contrail Systems acquisition 2012). Rebrand Tungsten Fabric 2018 khi move sang LFN. Architecture: vRouter agent trên mỗi compute, BGP EVPN control plane + XMPP cho controller-agent. Multi-hypervisor (KVM/ESXi/Hyper-V) + container (Kubernetes CNI). VMware acquired Juniper 2022 — ảnh hưởng scope chưa rõ.

## 7.3.3 VMware NSX-T (merged với NSX-V 2021)

NSX-T và NSX-V hai codebase độc lập 2016-2021. NSX 4.0 (2022) merger chính thức, NSX-V EOL. NSX-T features: multi-cloud (AWS/Azure/GCP native integration), Antrea CNI cho Kubernetes, micro-segmentation built-in. Architecture giữ hypervisor-edge pattern từ NVP (Part 5.1).

## 7.3.4 Arista CloudVision

Arista CloudVision eXchange (CVX): khác biệt lớn — không OpenFlow, không overlay controller. Focus: state streaming telemetry + config push + change management. API-driven (eAPI, OpenConfig, gNMI). Triết lý: "SDN" của Arista là consistent network state management chứ không phải centralized control plane đặc trưng.

## 7.3.5 So sánh: open ecosystem vs vendor

Lock-in cost: vendor controller require vendor switch hardware → switching cost cao 5-year. Feature velocity: vendor push features nhanh hơn open (6-tháng release cycle vs 12-tháng). Support model: vendor có 24/7 enterprise support, open cần in-house expertise hoặc third-party. TCO 5-year analysis thường vendor win nhỏ do support + training cost, open win lớn khi scale 500+ switch.
