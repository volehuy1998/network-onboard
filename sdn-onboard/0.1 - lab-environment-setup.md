## Mục tiêu bài học

Sau khi hoàn thành phần này, bạn có thể:

1. **Cài đặt (Apply)** môi trường lab: Ubuntu 22.04, OVS 2.17+, OVN 22.03+, Mininet, kolla-ansible 17.x.
2. **Verify (Apply)** ba cấp kiểm tra: OS version, package version, runtime health.
3. **Phân biệt (Analyze)** three lab modes: single-node all-in-one, two-node chassis pair, multi-node Kolla deployment.

---

## Dàn bài

## 0.1.1 Hardware/VM requirements

*Sẽ phát triển: baseline CPU/RAM/disk cho từng lab mode; khuyến nghị dùng Multipass hoặc VirtualBox cho lab đơn.*

## 0.1.2 Ubuntu 22.04 baseline + kernel modules

*Sẽ phát triển: verify kernel version ≥ 5.15; `openvswitch` module load; nested virtualization nếu lab KVM trong VM.*

## 0.1.3 OVS + OVN package installation

*Sẽ phát triển: `apt install openvswitch-switch ovn-central ovn-host`; service enable; verify `ovs-vsctl show` và `ovn-nbctl show`.*

## 0.1.4 Mininet cho OpenFlow labs (Block III-IV)

*Sẽ phát triển: Mininet 2.3 trên Ubuntu 22.04; Ryu controller cho Block III-IV; custom topology script.*

## 0.1.5 Kolla-ansible cho OpenStack labs (Block XIV)

*Sẽ phát triển: kolla-ansible 17.x baseline tương ứng OpenStack Antelope/Bobcat; globals.yml với `neutron_plugin_agent: ovn`.*

## 0.1.6 Health check playbook

*Sẽ phát triển: checklist systemctl status, ovn-sbctl show, ovs-appctl bond/show, ping xuyên chassis.*

## 0.1.7 Teardown/reset procedure

*Sẽ phát triển: cleanup sạch giữa các lab — `kolla-ansible destroy`, purge OVS DB, restart host.*

### ▶ Guided Exercise 1: Verify OVS 2.17 + OVN 22.03 với lệnh ovn-sbctl show

*Sẽ phát triển: step-by-step verify installation + output kỳ vọng.*
