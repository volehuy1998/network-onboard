## Dàn bài

## 12.1.1 Underlay BGP: iBGP vs eBGP route reflector patterns

iBGP full-mesh scale không tốt (N² session). Route reflector mitigate. eBGP per-switch ASN pattern phổ biến modern DC: mỗi leaf + spine có ASN riêng, eBGP peering đơn giản loop prevention. RFC 5549 + RFC 8950 (BGP unnumbered qua IPv6 link-local): không cần assign IP underlay cho BGP peering, đơn giản hoá config. FRR/Cumulus/Arista đều support.

## 12.1.2 Overlay VXLAN-EVPN integration

Underlay routes carry VTEP IP (loopback của mỗi leaf). Overlay BGP EVPN (RFC 7432) distribute MAC/IP reachability qua Type-2 routes, broadcast domain qua Type-3 inclusive multicast route. Head-end replication: leaf clone BUM (broadcast/unknown/multicast) ra mọi VTEP trong VNI. Ingress replication: mỗi VTEP source forward BUM copy, scale O(N × M).

## 12.1.3 Vendor fabric comparison

Cisco ACI: proprietary APIC controller, tight Nexus 9000 integration, contracts policy model. Arista CloudVision: API-driven, không centralized controller, prefer OpenConfig/gNMI. Juniper Apstra (acquired 2021): intent-based, graph datastore, blueprint templates. Comparison matrix theo dimension: control plane architecture, vendor lock-in, scale, multi-vendor support.

## 12.1.4 VNI scaling limit

24-bit VNI theoretical = 16.7M tenant. Practical limit thấp hơn nhiều do:
- Hardware TCAM size (Broadcom Trident2 ~8K entries per slice).
- MAC table entries total trong switch (~500K Tomahawk 3).
- Control plane BGP EVPN convergence với >10K VNI chậm đáng kể.

Mitigation: VNI re-use với separate VTEP groups, hoặc migrate Geneve (RFC 8926 với variable-length options, 24-bit VNI nhưng mở rộng được metadata).
