## 9.13.1 Libvirt + OVS

Compass Ch S pattern: trong domain XML của VM, dùng `<virtualport type='openvswitch'/>`:

```xml
<interface type='bridge'>
  <mac address='52:54:00:aa:bb:01'/>
  <source bridge='br-int'/>
  <virtualport type='openvswitch'/>
  <model type='virtio'/>
</interface>
```

Libvirt tự động:

1. Tạo tap interface (ví dụ `vnet0`).
2. Gọi `ovs-vsctl add-port br-int vnet0`.
3. Gọi `ovs-vsctl set Interface vnet0 external_ids:attached-mac=... external_ids:iface-id=<uuid>`.

`iface-id` là cách OVN (và OpenStack Neutron) discover port của VM cụ thể.

Khắc phục sự cố VIF plug:

```bash
[host]# ovs-vsctl list Interface | grep -A2 vnet
[host]# virsh domiflist <vm-name>
```

## 9.13.2 Docker + OVS

Compass Ch T hai pattern:

**Pattern 1 — ovs-docker helper** (đơn giản):

```bash
[host]# ovs-docker add-port br0 eth0 <container-id> \
         --ipaddress=10.0.0.5/24 --gateway=10.0.0.1
[host]# ovs-docker del-port br0 eth0 <container-id>
[host]# ovs-docker set-vlan br0 eth0 <container-id> 100
```

Script tự tạo veth pair, move vào container namespace, plug vào OVS.

**Pattern 2 — Manual veth** (production):

```bash
[host]# ip link add veth0 type veth peer name veth0-br
[host]# ip link set veth0-br up
[host]# ovs-vsctl add-port br0 veth0-br -- \
         set Interface veth0-br external_ids:container_id=<container-id>
[host]# PID=$(docker inspect -f '{{.State.Pid}}' <container-id>)
[host]# ip link set veth0 netns $PID
[host]# nsenter -t $PID -n ip addr add 10.0.0.5/24 dev veth0
[host]# nsenter -t $PID -n ip link set veth0 up
```

Manual cho control chi tiết (QoS, Mirror, OpenFlow per-veth).

## 9.13.3 Security baseline (compass Ch U)

**File permissions**:

```bash
[host]# chown openvswitch:openvswitch /etc/openvswitch
[host]# chmod 0750 /etc/openvswitch
[host]# chmod 0755 /var/run/openvswitch
[host]# chmod 0400 /etc/openvswitch/key.pem
```

**TLS mandatory**: mọi Controller/Manager target dùng `ssl:` (Part 9.10). Không bao giờ `ptcp:` trong production.

**IPsec cho tunnel** (compass Ch 12):

```bash
[host]# ovs-vsctl add-port br0 vxlan-ipsec -- \
         set Interface vxlan-ipsec type=vxlan \
         options:remote_ip=10.0.0.2 options:key=flow \
         options:psk=mysecret
[host]# systemctl enable --now openvswitch-ipsec
```

Verify `ipsec statusall` (strongSwan/Libreswan).

**SELinux/AppArmor**: distro ship default profile. KHÔNG disable.

## 9.13.4 ovs-sandbox cho testing (compass Ch V)

Source tree có `tutorial/ovs-sandbox` — boot transient `ovsdb-server` + `ovs-vswitchd` trong `/tmp` với `datapath_type=dummy`:

```bash
[dev]# cd /path/to/ovs-source/tutorial
[dev]# ./ovs-sandbox
```

Trong sandbox: add-br, add-port, add-flow, ofproto/trace mà không cần kernel module. Useful cho develop OpenFlow app, reproduce bug, CI test.

`ovs-testcontroller` pair với sandbox cho quick test.

## 9.13.5 BSD/Windows ports (compass Ch W)

- **Windows**: Hyper-V extensible-switch forwarding extension. Same `ovs-vsctl`/`ovs-ofctl`.
- **OpenBSD/NetBSD**: userspace datapath only, không DPDK/AF_XDP.

Scope khuyến nghị: production Linux. Windows/BSD cho use case đặc biệt.

### ▶ Guided Exercise 14: Plug container vào OVS qua veth manual

**Outcomes:** Tạo container, plug veth vào OVS, traffic qua OVS.

**Before You Begin:**

```bash
[host]# ovs-vsctl add-br br-docker
[host]# docker run -dit --name c1 --network none ubuntu:22.04 sleep 3600
```

**Instructions:**

**1.** Tạo veth pair:

```bash
[host]# ip link add c1-veth type veth peer name c1-br
[host]# ip link set c1-br up
```

**2.** Plug vào OVS:

```bash
[host]# ovs-vsctl add-port br-docker c1-br -- \
         set Interface c1-br external_ids:container_id=c1
```

**3.** Move c1-veth vào container namespace:

```bash
[host]# PID=$(docker inspect -f '{{.State.Pid}}' c1)
[host]# ip link set c1-veth netns $PID
[host]# nsenter -t $PID -n ip addr add 10.99.0.1/24 dev c1-veth
[host]# nsenter -t $PID -n ip link set c1-veth up
```

**4.** Kiểm chứng: `docker exec c1 ip addr show c1-veth`

**5.** Cleanup: `docker rm -f c1 && ovs-vsctl del-br br-docker`

## Điểm cốt lõi cần nhớ

Thứ nhất, libvirt + OVS qua `<virtualport type='openvswitch'/>` là pattern chuẩn — libvirt tự plug VM với external_ids metadata.

Thứ hai, Docker + OVS có hai pattern: `ovs-docker` (dev) và manual veth (production).

Thứ ba, security baseline non-negotiable: file permissions strict, TLS, IPsec, SELinux/AppArmor enabled.

Thứ tư, `ovs-sandbox` là công cụ development quan trọng — test mà không cần kernel module hay physical NIC.
