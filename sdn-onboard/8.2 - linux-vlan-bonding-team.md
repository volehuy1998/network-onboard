## 8.2.1 VLAN stacking: single-tagged, QinQ, triple-tagged

**Single-tagged 802.1Q (1998)**: 4-byte VLAN tag chèn sau MAC. 12-bit VLAN ID → 4096 VLAN tối đa.

```
+--------+--------+----------+----------+--------+------+
| DstMAC | SrcMAC | TPID=8100| VID+PCP  | EthType| Data |
| 6B     | 6B     | 2B       | 2B       | 2B     | ...  |
+--------+--------+----------+----------+--------+------+
```

**QinQ 802.1ad (2011)**: S-VLAN (service provider) + C-VLAN (customer) stacked. Service provider transport customer VLAN trong own VLAN space — mở rộng 4096 VLAN tenant trên hạ tầng shared.

**Triple-tagged**: deployment niche carrier, 3 tag lồng nhau cho MEF service hierarchy. Ít gặp ngoài telco.

Cấu hình VLAN trên Linux:

```bash
[host]# ip link add link eth0 name eth0.100 type vlan id 100
[host]# ip addr add 192.168.100.1/24 dev eth0.100
[host]# ip link set eth0.100 up
```

USC Day 5 Lab 6 hands-on VLAN trunking — dùng OVS `vlan_mode=trunk` + `trunks=[10,20,30]` thay cho Linux VLAN subinterface. Khác biệt: Linux VLAN subinterface chỉ handle VLAN theo per-interface; OVS trunk port handle nhiều VLAN với programmable policy.

## 8.2.2 Bonding: 7 modes

Linux kernel bonding driver (`drivers/net/bonding/`) implement 7 mode:

| Mode | Tên | Hoạt động | Peer switch requirement |
|---|---|---|---|
| 0 | `balance-rr` | Round-robin packet giữa slaves | Static LAG (Etherchannel on mode) |
| 1 | `active-backup` | Một slave active, còn lại standby | Không yêu cầu |
| 2 | `balance-xor` | Hash (MAC src XOR MAC dst) modulo slave count | Static LAG |
| 3 | `broadcast` | Replicate packet ra mọi slave | Redundancy niche |
| 4 | `802.3ad` (LACP) | Dynamic link aggregation per IEEE 802.3ad | LACP-capable peer |
| 5 | `balance-tlb` | Transmit LB, receive dedicated | Không yêu cầu |
| 6 | `balance-alb` | Adaptive LB TX + ARP-based RX | Không yêu cầu |

Cấu hình:

```bash
[host]# ip link add bond0 type bond mode 802.3ad miimon 100 lacp_rate fast
[host]# ip link set eth0 master bond0
[host]# ip link set eth1 master bond0
[host]# ip link set bond0 up
```

Mode 4 (LACP) là standard cho production multi-link — cần switch peer có LACP-capable LAG configuration tương ứng.

## 8.2.3 LACP specifics

IEEE 802.3ad-2020 định nghĩa LACPDU exchange giữa Actor (our side) và Partner (peer):

- **System ID**: MAC address + priority (ưu tiên peer với system ID thấp hơn làm negotiator).
- **Key**: identify aggregation group. Actor và Partner phải có key match để aggregate.
- **Port priority**: ưu tiên port trong group.
- **State bits**: Activity, Timeout, Aggregable, Synchronizing, Collecting, Distributing, Defaulted, Expired.

Timer:

- **Fast (1 second)**: LACPDU mỗi 1s, peer timeout 3s. Failover sub-100ms. CPU overhead cao hơn.
- **Slow (30 second)**: LACPDU mỗi 30s, peer timeout 90s. Default.

Fast-timer cho production datacenter, slow-timer cho WAN uplink.

Switch-side config phải match:
- Cùng port list.
- Cùng LACP mode (active-active hoặc active-passive).
- Cùng timer.

OVS bonding (Part 9.6) có fallback mode `lacp-fallback-ab=true` — fallback sang active-backup nếu LACP negotiation fail. Linux kernel bonding không có fallback tương đương.

## 8.2.4 Team daemon: deprecate trong RHEL 9

Team (`libteam`) ra đời 2012 thay thế bonding với:

- Userspace daemon `teamd` handle logic.
- D-Bus API cho programmatic control.
- Extensible runner (activebackup, roundrobin, loadbalance, random, lacp).

RHEL 7-8 ship team như alternative cho bonding. Tuy nhiên, **RHEL 9 (2022) deprecate team**, khuyến cáo dùng bonding.

Lý do deprecate:

1. **Tighter kernel integration**: bonding hoàn toàn trong kernel, team có userspace dependency.
2. **Team daemon crash** có thể disrupt networking — bonding không phụ thuộc userspace process.
3. **Duplicate functionality**: mọi feature team cung cấp đã có trong bonding (LACP mode 4 thay thế teamd lacp runner).
4. **Maintenance burden**: hai driver cùng làm việc tương đương, team contributions giảm.

Cho deployment mới, dùng bonding mode 4 (LACP) — không dùng team.

### ▶ Guided Exercise 3: Bonding LACP lab

**Outcomes:** Cấu hình Linux bond mode 4 với LACP fast-timer, kiểm chứng qua `/proc/net/bonding/bond0`.

**Before You Begin:** Host có 2 interface (eth1, eth2).

**Instructions:**

**1.** Tạo bond:

```bash
[host]# ip link add bond0 type bond mode 802.3ad miimon 100 lacp_rate fast
[host]# ip link set eth1 master bond0
[host]# ip link set eth2 master bond0
[host]# ip link set bond0 up
```

**2.** Verify:

```bash
[host]# cat /proc/net/bonding/bond0
```

Output cho thấy `Bonding Mode: IEEE 802.3ad Dynamic link aggregation`, `LACP rate: fast`, và state của từng slave.

**3.** Check LACP state (nếu có peer):

```bash
[host]# cat /sys/class/net/bond0/bonding/ad_actor_system
[host]# cat /sys/class/net/bond0/bonding/ad_partner_key
```

**4.** Cleanup: `ip link delete bond0`

## Điểm cốt lõi cần nhớ

Thứ nhất, VLAN 802.1Q single-tagged đủ cho 4096 VLAN — QinQ cần thiết khi service provider transport customer VLAN.

Thứ hai, Linux bonding có 7 mode. Mode 4 (802.3ad LACP) là standard production. Mode 1 (active-backup) cho scenario đơn giản.

Thứ ba, LACP fast-timer (1s) cho datacenter production, slow-timer (30s) cho WAN. OVS có `lacp-fallback-ab` — kernel bonding không có.

Thứ tư, team daemon deprecated từ RHEL 9 (2022). Mọi deployment mới dùng bonding.
