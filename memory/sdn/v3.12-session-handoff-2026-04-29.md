## Stop instruction to in-flight agents

Agents are autonomous sub-processes. There is no clean way to inject a "stop now" message to running agents from the orchestrator. The pragmatic actions are:

1. **Let any agent finish that is close to commit**, they self-validate and self-commit, so the worst case is one more commit lands within the next 15-30 minutes after pause.
2. **At wake-up, check `git status` first.** Any uncommitted file is a partial-state agent that did not finish. Apply rule A above (clean slate) by default.
